If you're utilizing Barracuda's Email Security Gateway, you can whitelist KnowBe4 to allow our simulated phishing test emails and training notifications through to your end-users.\u00a0\u00a0For the most up-to-date list of our IP addresses, please see the KnowBe4's IP Addresses, Hostnames, and Headers section of our Whitelisting Data and Anti-Spam Filtering Information article.\u00a0
\n
The instructions below are for a third-party software. If you run into issues whitelisting KnowBe4 in Barracuda,\u00a0we recommend reaching out to Barracuda for specific instructions. You can also\u00a0contact our support team\u00a0whenever you need assistance.
\n\n
Whitelisting by IP in Barracuda in Email Security Gateway
\n
These instructions were gathered and summarized\u2013based on Barracuda's knowledgebase. For more information on how to whitelist in Barracuda, check out the Barracuda Email Security Service - Configuring Inbound Email video.
\n
If you are using Barracuda's Email Security Service (cloud), follow these steps to whitelist\u00a0Barracuda by IP address:
\n
    \n
  1. Log in to your Barracuda Cloud Control.
    2. \n
  2. Go to Email Security > Inbound Settings > IP Address Policies.
    3. \n
  3. In the IP Blocking / Exemption section, use the top line to enter one of our IP addresses. This process will need to be repeated for each IP address. For the most up-to-date list of our IP addresses, please see the KnowBe4's IP Addresses, Hostnames, and Headers section of our Whitelisting Data and Anti-Spam Filtering Information article.
    4. \n
  4. In the\u00a0Netmask\u00a0field, type\u00a0255.255.255.255.\n
    \nNote: Barracuda does not allow for IP address ranges, so the first IP range should be 147.160.167.0 with a netmask of 255.255.255.192. Entering the IP range this way will cover the entire range. The single addresses will have a netmask of 255.255.255.255.
    \n
    5. \n
  5. Set the Policy field to Exempt.
    6. \n
  6. If you'd like, add a note in the\u00a0Comment\u00a0field. For example, KnowBe4 Simulated Phishing IP Address.
    7. \n
  7. Click Add to whitelist the IP address.
    8. \n
  8. Repeat steps 2 through 7 for each of the KnowBe4 IP addresses.

Barracuda Intent Analysis
\n
You may need to whitelist us in Barracuda's Intent Analysis feature to prevent the URLs in simulated phishing tests from being altered, and potentially resulting in skewed phishing test results. See the Intent Analysis - Inbound Mail article from Barracuda explaining this process.
\n
If you are using Barracuda's Email Security Service (cloud), follow these steps to whitelist\u00a0Barracuda's Intent Analysis:
\n
    \n
  1. Log in to your Barracuda Cloud Control.
    2. \n
  2. Navigate to\u00a0Email Security > Inbound Settings > Anti-Phishing.\n
    3. \n
  3. Under the Intent section, add KnowBe4's hostnames. Make sure the Policy drop-down menu is set to Ignore.
    4. \n
\n
If you are using Barracuda's Email Security Gateway (on-premises), follow these steps to whitelist Barracuda's Intent Analysis:
\n
    \n
  1. Log in to your Barracuda Email Security Gateway web interface.
    2. \n
  2. Navigate to\u00a0Email Security Gateway\u00a0> Basic > Spam Checking.
    3. \n
  3. Under the Intent Analysis section, add KnowBe4's hostnames to the URI Exemptions: text box field.
    4. \n

Barracuda Sender Authentication
\n
If you'd like to spoof your own domain in simulated phishing tests, you can exempt Trusted Forwarder IP addresses from SPF checks. See the How to Configure Sender Policy Framework article from Barracuda for more information.
\n
If you are using Barracuda's Email Security Service (cloud), follow these steps to whitelist\u00a0Barracuda's Sender Authentication:
\n
    \n
  1. Log in to your Barracuda Cloud Control.
    2. \n
  2. Navigate to Email Security > Inbound Settings > Sender Authentication.\n
    3. \n
  3. In the Use Sender Policy Framework center enter our IP addresses in the SPF exemptions table.
    4. \n
\n
If you are using Barracuda's Email Security Gateway (on-premises), follow these steps to whitelist Barracuda's Sender Authentication:
\n
    \n
  1. Log in to your Barracuda Email Security Gateway web interface.
    2. \n
  2. Navigate to\u00a0Email Security\u00a0> Block/Accept tab and select Sender Authentication.
    3. \n
  3. Under Sender Policy Framework (SPF) Configuration\u00a0section, select Yes.
    4. \n
  4. Add the KnowBe4 IP addresses to the exemption list.
    5. \n
\n
\nNote: After following this article, we recommend setting up a test phishing campaign for 1-2 users to ensure your whitelisting was successful. As a last resource, we suggest reaching out to your service provider for assistance. Barracuda Advanced Threat Protection (ATP)\u00a0
\n
If you are using Barracuda's Advanced Threat Protection (ATP) and have experienced false clicks or false attachment opens, you can set up exemptions. Setting up exemptions allows you to bypass PDF scanning for phishing test emails from KnowBe4's IP addresses.\u00a0
\n
To set up exemption addresses to bypass\u00a0ATP PDF Scanning:
\n
    \n
  1. Log in to your Barracuda Email Security Gateway web interface.
    2. \n
  2. Select the ATP Settings\u00a0tab.
    3. \n
  3. Enter the IP address(es) and Subnet Mask(s). For the most up-to-date list of our IP addresses, please see the KnowBe4's IP Addresses, Hostnames, and Headers section of our Whitelisting Data and Anti-Spam information article.\u00a0
    4. \n
  4. Click Add.
    5. \n

Barracuda Sentinel Allow Senders
\n
Using Barracuda Sentinel\u2019s Allow Senders list allows KnowBe4 emails to bypass your organization\u2019s current whitelisting rules. See Barracuda\u2019s How to Allow Senders article for more information.
\n
To add specific senders to your Allow Senders list:
\n
    \n
  1. Log in to your Barracuda admin console.
    2. \n
  2. Click Dashboard in your console menu.
    3. \n
  3. Click the Settings icon, which should appear as a gear.
    4. \n
  4. Click Allowed Senders.
    5. \n
  5. Enter one email address or domain name into the Sender Email or Domain field.\n
    \nNote: Barracuda Sentinel will only allow you to enter one email address or domain name at a time for security reasons. You can repeat steps 5 through 7 for as many email addresses or domains you wish to add.
    \n
    6. \n
  6. You can add a comment to your email address or domain name if you wish.
    7. \n
  7. Click Save.
    8. \n
\n
You can also delete or edit the email addresses and domain names from the Allowed Senders page using their respective icons.
\n
\nTip: To ensure that you have whitelisted correctly, see our How to Verify You Have Whitelisted KnowBe4 Correctly article.
