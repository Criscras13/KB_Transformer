In your KCM GRC platform, you can create user groups that will be responsible for completing control tasks, in order to satisfy your organization's compliance requirements or risk management objectives.
\n
\n
\nNote:\u00a0The user groups discussed in this article are different from the user groups that you create for your end users under the Policy Management module. See our Policy Management Module Guide guide for more information.
\n
\n
This article explains how to manage the user groups that will complete tasks for your risk management or compliance management controls. Before you begin, please see the How Do I Use User Groups? section below.
\n
\n
How Do I Use User Groups?
\n
The first step is to create a user group and assign a user as the Group Lead. Then, you'll assign the user group to one or more controls. Assigning a user group to a control allows you to delegate task assignments and task evidence approvals to the Group Lead. Users\u00a0who are assigned as a Group Lead have special permissions allowing them to reassign tasks to another user in their group. To learn more about user permissions and user roles, see our User Roles Guide.
\n
Here are a few things you should know about assigning a user group to a control:\u00a0
\n
    \n
  • When evidence is required for a task under this control, the Group Lead must approve the evidence before the task can be closed.\u00a0\n
      \n
    • The Group Lead is notified once evidence has been submitted for a task. To learn more, see Control Task Notifications.\u00a0
      • \n
    \n
    • \n
  • When creating task schedules or one-time tasks for the control, you must select one of the users from that group to be the User Assigned for the task. This means users who are not in the group cannot be assigned to task schedules for that control.\n
      \n
    • The user who is assigned to the control's task schedule(s) will receive task reminder notifications from KCM GRC To learn more, see Control Task Notifications.
      • \n
    \n
    • \n
  • Tasks must be completed by the User Assigned for the task.\n
      \n
    • Group Leads or other group members cannot complete a task that is assigned to someone else unless they have a user role that allows them to do so, such as Scope Administrator. To learn more about user permissions, see our User Roles Guide.\u00a0\u00a0
      • \n
    \n
    • \n
  • A Group Lead can reassign tasks for this control to another member in their user group.\u00a0
    • \n
  • Group members can view all tasks that are created for the control that their group is assigned to. This includes tasks that may have been created before you assigned a group to the associated control.\u00a0
    • \n
\n
\n
Creating a User Group
\n
You will start by creating one or more user groups. To create user groups you must have Account Administrator privileges. See our User Roles Guide\u00a0to learn more about user permissions.
\n
Follow the steps below to create a new user group:
\n
    \n
  1. From the upper right-hand corner of your account, click Settings > Manage Users. \n
    2. \n
  2. Click the User Groups tab. \n
    3. \n
  3. Click the Create Group button. Then, add the details outlined below:\n
    \n
    \nNote:We recommend that you avoid including the < and > special characters in these fields.
    \n
    \n
      \n
    1. \nGroup Name: Create a title for the user group. This group name will be visible to group members, and to other users who can view tasks in your account.\u00a0
      2. \n
    2. \nGroup Description: Describe the purpose of the group. You can include details about the scopes and/or controls that this user group will be responsible for.
      3. \n
    3. \nSelect Users:\u00a0Select a user from the drop-down menu. You can also type a user's name or email address into this field to search for a user.\u00a0
      4. \n
    4. \nSelect Group Lead: Select a user from the list of users that you have added to the group.\u00a0
      5. \n
    \n\n
    4. \n
  4. Click Create to save the group.
    5. \n
\n
Once you've created the group, you will assign the group to one or more controls. Proceed to the next section for instructions.\u00a0
\n
\n
Assigning a User Group to a Control
\n
Once you've created a user group, you can assign the group to one or more controls that the group will be responsible for. Once the group is assigned to a control, you can assign one of the group members to the control's task schedule or schedules.\u00a0
\n
Follow the steps below to assign a user group to a control:\u00a0
\n
    \n
  1. Navigate to the desired control using one of the following options:\n
      \n
    1. From the Controls Library:\n
        \n
      1. From the navigation menu on the left, click Controls.
        2. \n
      2. From the Controls Library, use the search bar under the Name or Description columns to find the appropriate control.
        3. \n
      3. Click a control name to open the control
        4. \n
      \n
      2. \n
    2. From a specific scope (to view controls that are mapped to the requirements under that scope):\n
        \n
      1. From the navigation menu on the left side, click Compliance > Scopes.
        2. \n
      2. From the View All Scopes page, click a scope name to open the scope.
        3. \n
      3. From the View Scope page, click the Controls tab.
        4. \n
      4. Click a control name to open the control.\n
        \n
        \nTip: If you will be assigning user groups to multiple controls in this scope, right-click the control name and open the control in a new tab, instead.
        \n
        \n
        5. \n
      \n
      3. \n
    \n
    2. \n
  2. From the View Control page, click the Update button. \n
    3. \n
  3. Click the User or Group Assigned drop-down menu, and search for the name of the desired user group. \n
    4. \n
  4. Click Save to save your changes.
    5. \n
\n
After you've assigned your group to a control, you can create one or more task schedules for this control and assign the group lead or another group member to the task schedule. For more information about control tasks, see our How to Work with Tasks and Task Schedules for Controls article.
\n
\n
Managing Group Memberships\n
\n
See the following two subsections to add additional users to a group or to remove users from a group.\u00a0
\n
Adding Users to a Group
\n
Follow the steps below to add users to a group:\u00a0
\n
    \n
  1. From the top-right, click Settings > Manage Users.
    2. \n
  2. Use the checkboxes in the far-left column to select the user or users that you would like to add to a group.
    3. \n
  3. Click the Select a Group drop-down and select the desired group. You can also click this field and type to search for the desired group.\u00a0
    4. \n
  4. Then, click the Add Selected to Group button to add the selected users to this group.\n
    5. \n
\n
\n
\nTip: Alternatively, you can add users to a group from the Update User Group window. For instructions, see the Updating a User Group section below.\u00a0
\n
\n
Removing Users from a Group
\n
Follow the steps below to remove users from a group:
\n
    \n
  1. From the top-right, click Settings > Manage Users.
    2. \n
  2. Click the User Groups tab.
    3. \n
  3. Click a group name to open the group.
    4. \n
  4. Use the checkboxes in the far-left column to select the user or users that you would like to remove from the group.\n
    \n
    \nNote: If you would like to remove the Group Lead from the group, first click the Update Group button to reassign the Group Lead role to a different user.
    \n
    \n
    5. \n
  5. Click the Remove Selected From Group button. Then, when prompted, click the Remove button.\n
    \n
    \nImportant: If the removed users are assigned to tasks, their tasks will be reassigned to the group member that is specified as the Group Lead. This only applies to tasks that are under a control that this user group is assigned to.\n
    \n
    \n
    6. \n
\n
Updating a User Group
\n
You can make the following changes to a user group: Change the Group Name or Group Description, add or remove users from a group, and reassign the Group Lead role to a different user.
\n
Follow the steps below to update a user group:\u00a0
\n
    \n
  1. From the top-right, click Settings > Manage Users.
    2. \n
  2. Click the User Groups tab.
    3. \n
  3. Click a group name to open the desired group.
    4. \n
  4. Click the Update Group button toward the top-right area of the page.\u00a0
    5. \n
  5. Make the desired changes, then click the Save button.\n
    6. \n
\n
Deleting a User Group
\n
Follow the steps below to delete a user group:\u00a0
\n
\n
\nImportant: You cannot delete a user group if the group is assigned to a control.
\n
\n
    \n
  1. From the top-right, click Settings > Manage Users.
    2. \n
  2. Click the User Groups tab.
    3. \n
  3. Find the group you wish to delete from the list. Then, under the\u00a0Actions column, click the trash can icon for that group.
    4. \n
  4. When prompted, click the Delete button to delete the group.
    5. \n
