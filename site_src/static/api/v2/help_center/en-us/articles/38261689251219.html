This article guides you through creating a mail flow rule in Microsoft 365 to bypass Defend scanning for Secure/Multipurpose Internet Mail Extensions (S/MIME) encrypted emails. This rule is required because S/MIME emails are already encrypted. As a result, our Defend system cannot scan or process the contents of these emails for security purposes. The preexisting encryption prevents any further analysis or modification by Defend.
\n
Prerequisites
\n
The only prerequisite for bypassing Defend scanning is to have admin access to Microsoft 365.
\n
Instructions
\n
Follow the steps below to bypass Defend scanning:
\n
    \n
  1. Log in to the Microsoft Admin Center.
    2. \n
  2. In the left sidebar, navigate to\u00a0Show All\u00a0>\u00a0Admin Centers\u00a0>\u00a0Exchange.
    3. \n
  3. In the Exchange admin center, navigate to\u00a0Mail flow >\u00a0Rules\u00a0>\u00a0+ >\u00a0Create a new rule.\u00a0
    4. \n
  4. Configure the rule by filling out the fields as shown in the screenshot and list below:\n
      \n
    1. \nRule name: Enter a descriptive name, such as \"Skip S/MIME Emails for Defend\".\u00a0
      2. \n
    2. \nRule conditions: Under\u00a0Apply this rule if..., select\u00a0The message properties >\u00a0Include any of these words in the message header. In the\u00a0Specify words or phrases\u00a0dialog box, enter\"Words=Content-Type\" and \"matches=application/pkcs7-mime\". Under\u00a0Do the following... select Modify the message properties > Set a message header. Enter the header name as \"X-Egress-Defend-Bypass-SMIME\" and set the header value to true.
      3. \n
    3. \nExcept if: Under Except if..., add any necessary exceptions. This step is optional.
      4. \n
    4. \nRule settings: Set the rule priority to run before the Defend mail flow rules and select Stop processing more rules. \u00a0
      5. \n
