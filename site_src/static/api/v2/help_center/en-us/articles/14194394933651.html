From the Syslog subtab of your PhishER Settings, you can configure the Syslog servers connected to your PhishER platform. System Logging Protocol, or Syslog, is a protocol that generates logs for network devices or servers. You can integrate Syslog servers with your PhishER platform to log when PhishER actions are triggered.
\n
Connecting a New Syslog Server
\n
To connect a Syslog server to your PhishER platform, follow the steps below:
\n
    \n
  1. Log in to your PhishER platform.
    2. \n
  2. Navigate to Settings > Syslog.
    3. \n
  3. Click the New Syslog button in the top-right corner of the page.
    4. \n
  4. In the Add Syslog Settings pop-up window that opens, configure your Syslog server settings. To learn more about these settings, see the screenshot and list below:\n
      \n
    1. \nName: In this field, enter a custom name for your Syslog server.
      2. \n
    2. \nProtocol: From the drop-down menu, select one of the following protocols: TCP, UDP, TLS, or TLS_INSECURE.
      3. \n
    3. \nHost: In this field, enter the host IP address of your Syslog server.
      4. \n
    4. \nPort: In this field, enter the port number of your Syslog server.
      5. \n
    5. \nFormat: From the drop-down menu, select one of the following Syslog output formats: JSON, CEF, or LEEF. To learn more about these formats, see the Syslog Output Formats section of this article.
      6. \n
    \n\n
    5. \n
  5. Click Create.
    6. \n
\n
Once your Syslog server is integrated with your PhishER platform, your organization can track and log events that occur in your platform. You can connect as many Syslog servers as you would like. If you create a PhishER action, you can select the Send to Syslog option under the Choose how you would like to report this action step.
\n
Then, you can use the drop-down menu to select one of your Syslog servers. When your action is run, your Syslog server will log the event.
\n
Managing Your Connections
\n
From the Syslog subtab of your PhishER Settings, you can view all of your Syslog server connections. You can click a Syslog server\u2019s name to open the Update Syslog Settings pop-up window. Then, you can update the information for that Syslog server.
\n
For more information about the Syslog settings, see the screenshot and list below:
\n
    \n
  1. \nNew Syslog: Click this button to create a new Syslog server connection to your PhishER account.
    2. \n
  2. \nName: This column displays the name of the Syslog server.
    3. \n
  3. \nHost: This column displays the host IP address of the Syslog server.
    4. \n
  4. \nPort: This column displays the port number of the Syslog server.
    5. \n
  5. \nProtocol: This column displays the protocol that the Syslog server uses to transport messages from the client to the server.
    6. \n
  6. \nFormat: This column displays the output format for the Syslog server\u2019s reports.
    7. \n
  7. \nActions: This column displays the actions available to run on a Syslog entry. You can click the trash can icon to delete the entry.
    8. \n
\n
Syslog Output Formats
\n
You can select a JSON, CEF, or LEEF output format for your Syslog reports. For examples of each output format, see the subsections below.
\n
JSONCEFLEF
\n
JSON
\n
For an example of a JSON output format, see below:
\n
\n\t\t{ \"receivedAt\":\"2019-05-20T17:39:43.351851Z\",\n\t\t\"reportedAt\":\"2019-05-20T17:39:39Z\", \"sender\":\"sender@example.com\",\n\t\t\"reporter\":\"reporter@example.com\", \"subject\":\"JSON Syslog Example\",\n\t\t\"priority\":\"medium\", \"category\":\"spam\", \"status\":\"received\",\n\t\t\"action\":\"Action 1\", \"tags\":\"TagSet\" \"permalink\":\"[[Unique URL]]\" }\n\t

AttributeDescription
receivedAt\n

\n

This attribute indicates the date and time of when a user reported the message.

statusThis attribute indicates the message\u2019s current state of PhishER analysis. PhishER uses the following statuses: Received, In Review, and Resolved.
actionThis attribute indicates the name of the PhishER action that triggered this report.
tagsThis attribute indicates the label attached to the message based on the message's attributes.
permalinkThis attribute indicates the URL specific to the message in your PhishER Inbox.
