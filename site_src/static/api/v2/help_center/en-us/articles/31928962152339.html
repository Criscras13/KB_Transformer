In this article, you will learn how to integrate Palo Alto Next-Generation Firewall (NGFW) with SecurityCoach. Once the integration is complete, data provided by Palo Alto NGFW will be available in the SecurityCoach tab of your KnowBe4 Security Awareness Training (KSAT) console. This data can be viewed in SecurityCoach reports and can be used to create detection rules for real-time coaching campaigns. For more information about SecurityCoach, see our SecurityCoach Product Manual.
\n
\n
\nTip:SecurityCoach also supports integrating Palo Alto NGFW as a Splunk vendor. If your organization uses Splunk for security information and event management, we recommend integrating through Splunk instead by following our Splunk Integrations Guide for SecurityCoach.
\n
\n
Setting Up the Integration in Palo Alto NGFW
\n
    \n
  1. Log in to your Palo Alto NGFW admin account.
    2. \n
  2. Select Device at the top of the page.\n
      
    \n
    3. \n
  3. Select Admin Roles in the left sidebar.\n
      
    \n
    4. \n
  4. Select Add near the bottom of the page.\n
      
    \n
    5. \n
  5. A pop-up window will open. In the Name and Description fields, enter your desired name and description.\n
      
    \n
    6. \n
  6. Disable all access options in the Web UI and REST API tabs.\n
      
    \n
      
    \n
    7. \n
  7. Enable Log access in the XML API tab.\n
     
    \n
    8. \n
  8. Select OK in the bottom-right corner to save your changes.\n
     
    \n
    9. \n
  9. Select Device at the top of the page.\n
      
    \n
    10. \n
  10. In the left sidebar, select Administrators.\n
      
    \n
    11. \n
  11. Select Add near the bottom of the page.\n
      
    \n
    12. \n
  12. A pop-up window will open. See the screenshot below to fill in the required information:\n
      
    \n
      \n
    1. \nName: Enter your desired name.
      2. \n
    2. \nDescription: Enter your desired description.
      3. \n
    3. \nPassword: Enter your desired password.
      4. \n
    4. \nPassword: Enter your password again.
      5. \n
    5. \nAdministrator Type: Select Role Based.\n
      6. \n
    6. \nProfile: Select the drop-down menu and choose the name of the admin role you created earlier in Steps 5-7 of Setting Up the Integration in Palo Alto NGFW.
      7. \n
    \n
    13. \n
  13. In the bottom-right corner, select OK to save your changes.\n
      
    \n
    14. \n
  14. In the top-right corner of the page, select Commit.\n
      
    \n
    15. \n
  15. A new pop-up window will open. At the bottom of the window, select Commit to start committing your changes.\n
      
    \n
    16. \n
  16. Wait until the Progress bar reaches 100% to indicate completion.\n
      
    \n
    Once the Progress bar reaches 100%, \u201cConfiguration committed successfully\u201d will display next to Details.
    \n
    17. \n
  17. You can now find the Firewall Domain name in the URL of the webpage. For example, https://[your firewall domain name]/\n
      
    \n
    18. \n
\n
Setting Up the Integration in SecurityCoach
\n
To set up the Palo Alto NGFW integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to SecurityCoach > Setup > Security Coach Vendor Integrations.
    3. \n
  3. Locate the Palo Alto vendor tile, select Configure, then select Direct Integration. This will open the Palo Alto Next-Generation Firewall (NGFW) integration page.\n
      
    \n
    4. \n
  4. To complete the integration, see the screenshot below and fill in the required information:\n
      
    \n
      \n
    1. \nFirewall Domain: Copy and paste your Firewall Domain from Step 17 of the Setting Up the Integration in Palo Alto NGFW section.
      2. \n
    2. \nUsername: Enter the admin username that you used in the Setting Up the Integration in Palo Alto NGFW section.
      3. \n
    3. \nPassword: Enter the admin password that you used in the Setting Up the Integration in Palo Alto NGFW section.
      4. \n
    \n
    5. \n
  5. Select Connect to complete the integration.
    6. \n
\n
\n
\nNote:Events from Palo Alto NGFW are automatically mapped to users using their usernames.
\n
\n
Deleting the Integration in SecurityCoach
\n
If you want to delete the Palo Alto NGFW integration from SecurityCoach, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to SecurityCoach > Setup > Security Vendor Integrations.
    3. \n
  3. Locate the Palo Alto vendor tile and select Edit.
    4. \n
  4. Select Delete Integration near the bottom of the page. A new pop-up window will open.
    5. \n
  5. If you are sure you want to delete the integration, click Confirm.
    6. \n
