The PhishER Inbox displays emails that your users report manually or by using the Phish Alert Button (PAB). Once your users start reporting emails, you can view and manage them from your Inbox.
\n
See the sections below to learn how to use your Inbox.
\n
Navigating the Inbox
\n
If you would like to show or hide specific Inbox columns, select the gear icon at the top-right corner of the Inbox page. When you click this icon, the Inbox Table Settings pop-up window will open. You can show or hide columns by selecting or deselecting the check box to the left side of the column name.
\n
\n
For more information about the Inbox page, see the screenshot and list below:
\n
\n
    \n
  1. \nCategory: This column displays the grouping of messages based on disposition. A message can be categorized or dispositioned in one of three ways: Clean, Spam, or Threat. Each message will be dispositioned as Unknown until a different disposition is determined.
    2. \n
  2. \nFrom: This column displays the sender name associated with the original message.
    3. \n
  3. \nFrom Email: This column displays the email address associated with the original message.
    4. \n
  4. \nSubject: This column displays the text found in the subject line of the original message.
    5. \n
  5. \nFirst Disposition Date: This column displays the timestamp of when the message was first dispositioned.
    6. \n
  6. \nCurrent Disposition Date: This column displays the timestamp of when the message's current disposition was applied.
    7. \n
  7. \nReported At: This column displays the date and time of when the message was received by the Inbox.
    8. \n
  8. \nReported By: This column displays the name of the user who reported the message.
    9. \n
  9. \nReported By (Email): This column displays the email address of the user who reported the message.
    10. \n
  10. \nStatus: This column displays a message's analysis status. A message can have a status of Received, In Review, or Resolved.
    11. \n
  11. \nTags: This column displays a label attached to a message based on the message's attributes. There is no limit to the number of tags a message can have. All tags are clickable. By clicking a tag, you can create a filtered view of the PhishER Inbox based on that tag. Tags can be automatically or manually assigned to a message. To learn about the ways tags can be assigned, see the list below:
    12. \n
  12. Users can suggest a tag for a message with the Phish Alert Button (PAB) disposition feature. For more information, see our Adding User Comments and Email Disposition to the Phish Alert Button article.
    13. \n
  13. An admin can manually add a tag to a message from the Inbox or Message Details page of the PhishER Inbox. Admins can also add tags to multiple messages by selecting multiple messages in the Inbox.
    14. \n
  14. Tags are automatically assigned to a message when the message matches a rule containing that specific tag.
    15. \n
  15. \nPriority: This column indicates if a message needs urgent review due to potential malicious content. A message can have a Low, Medium, High, Critical, or Unknown priority. The priority of a message is determined initially by your organization's rules and actions. However, admins can change the priority at any time.
    16. \n
  16. \nPhishRIP: This column will be available if PhishRIP is enabled in your PhishER platform. You can click on the plus icon to initiate PhishRIP. If PhishRIP was already initiated on the message, a fishbone icon will display instead.\n
    \n
    17. \n

Downloading the Inbox CSV
\n
You can download a CSV file of your PhishER Inbox by clicking the Download CSV button in the top-right corner of the Inbox page. This CSV file will include all visible and hidden columns in the Inbox.
\n
If the CSV file does not exceed the maximum number of items, it will automatically download directly through your browser. The maximum number of CSV file items for direct download is set to 5,000 by default. You can set the item limit in the Preferences subtab of your PhishER Settings. If the CSV file includes more than the maximum number of items, it will be generated first in your PhishER console before you can download it. You will receive an alert in your console when the file is fully generated and available to download. The file will also be displayed in the Downloads subtab of your PhishER Settings. For more information, see our PhishER Settings: Account article.

Filtering the Inbox
\n
You have the ability to filter your Inbox messages. To learn about the filters you can select, see the screenshot and list below:
\n
\n
    \n
  • \nFilter by Category: This section allows you to filter messages based on disposition. For more information, see the options listed below:\n
      \n
    • \nClean: Select this option to display all messages dispositioned as clean.
      • \n
    • \nSpam: Select this option to display all messages dispositioned as spam.
      • \n
    • \nThreat: Select this option to display all messages dispositioned as threat.
      • \n
    • \nUnknown: Select this option to display all messages that are not dispositioned as one of the three main dispositions.
      • \n
    \n
    • \n
  • \nFilter by Status: This section allows you to filter messages based on their analysis status in PhishER. For more information, see the options listed below:\n
      \n
    • \nReceived: Select this option to display all messages that have not been reviewed in PhishER.
      • \n
    • \nIn Review: Select this option to display all messages that are in review.
      • \n
    • \nResolved: Select this option to display all messages that have been reviewed and resolved.
      • \n
    \n
    • \n
  • \nFilter by Priority: This section allows you to filter messages based on their assigned priority. For more information, see the options listed below:\n
      \n
    • \nUnknown: Select this option to display all messages that have not been prioritized.
      • \n
    • \nLow: Select this option to display all messages marked as low priority.
      • \n
    • \nMedium: Select this option to display all messages marked as medium priority.
      • \n
    • \nHigh: Select this option to display all messages marked as high priority.
      • \n
    • \nCritical: Select this option to display all messages marked as critical priority.
      • \n
    \n
    • \n
\n
For more filter options, you can use the Search... box to filter your Inbox using Lucene queries. You can also save a customized filter by clicking the Save query as room button. For more information, see our How to Use Lucene Query Syntax and How to Create and Manage PhishER Rooms articles.

Running Actions on Selected Messages
\n
To select a message, select the check box to the left of the message. When a message is selected, the Run drop-down menu will display in the top-left corner of the Inbox page. From this drop-down menu, you can select an action listed under Replay, PhishRIP, Email, or Actions.
\n
For more information about these options, see the screenshot and list below:
\n
\n
    \n
  1. \nReplay: This subsection allows you to rerun your rules and actions against selected messages. For more information about the available actions, see the list below:
    2. \n
  2. \nAll Rules: This action allows you to run all of your custom rules against any selected messages. If manually selected, this will also run for emails that do not match the criteria of your tags.
    3. \n
  3. \nAll Rules and All Actions: This action allows you to run all of your custom rules and actions against any selected messages. If your organization has enabled VirusTotal or PhishML, they will run against any selected messages as well.
    4. \n
  4. \nPhishRIP: If your organization has enabled PhishRIP, this subsection allows you to run PhishRIP actions against any selected messages. For more information about the available actions, see the list below:
    5. \n
  5. \nFind Similar Messages: This action will open the Find Similar Messages pop-up window, which allows you to select the match criteria of your PhishRIP query.
    6. \n
  6. \nCreate KSAT Template: This action will send the selected email to your KSAT console to create a new phishing template using a clean version of the email.
    7. \n
  7. \nEmail: This subsection allows you to send an email. You can click Send Custom Email to open the template editor in a pop-up window. When your email is ready to be sent, click the Send button.
    8. \n
  8. \nActions: This subsection allows you to run custom actions against any selected messages.\n
    \n
    \nTip:Alternatively, you can select a custom quick action from the QuickActions bar on the top-right side of the Inbox page to run against your selected messages. Both options may be helpful if a message was received before an action was created.
    \n
    \n
    \n
    9. \n
\n
From the top of the Inbox page, you can also set a Category, Status, and Priority for the selected messages and add multiple tags.
\n
\n
If you would like to clear the selected messages, you can click the Clear Selection button.

Viewing Message Details
\n
If you click an individual message in your PhishER Inbox, the Message Details page will open. For more information about the top of the Message Details page, see the screenshot and list below:
\n
\n
    \n
  1. \nFrom: This field displays the name or email address of the person who sent the original message.
    2. \n
  2. \nReply-to: This field displays the name or email address that will populate the to field of an email if a user replies to the message.
    3. \n
  3. \nTo: This field displays the name or email address of the original message recipient.
    4. \n
  4. \nCC: This field displays any of the email addresses copied on the original message.
    5. \n
  5. \nReported: This field displays the date and time of when the message was reported by a user.
    6. \n
  6. \nReported by: This field displays the name or email address of the user who reported the message.
    7. \n
\n
All email addresses in this section of the Message Details page are clickable. Clicking an email address will take you to a filtered view of your Inbox and display all of the messages tied to the specific email address.
\n
\n
\nNote:The timestamp in the top-right corner will display when the message was received by the reporter. A phish hook icon will display next to the date and time if the message was reported using the Phish Alert Button (PAB). A forwarding arrow icon will display next to the date and time if the message was not reported using the PAB.

\n
On the right side of the Message Details page, you can view the Actions and Discussion sidebar. From this sidebar, you can open the Actions and Discussion subtabs.
\n
For information about these subtabs, see the subsections below:

Actions
\n
From the Actions subtab, you can update the message's dispositioning and tags, download the original reported email, and run actions against the message. If PhishRIP is enabled, you can create and view PhishRIP queries, and you can view your PhishML confidence results. For more information, see the screenshot and list below:

Discussion
\n
From the Discussion subtab, you can communicate with other admins about a message. Similar to a chat window, this method of communication may be useful for organizations with multiple admins managing the PhishER Inbox.
\n
Users can communicate with admins about specific messages when they use the PAB. When users leave a comment while using the PAB, the comment will appear in the Discussion subtab. For more information, see our Phish Alert Button (PAB) User Comments and Email Disposition Guide.

Admins can communicate with users who report emails using the PAB. From the Message Details page of your PhishER Inbox, you can send emails to users who report messages with the PAB. The Discussion subtab displays the full conversation and allows you to send an external message from the PhishER console when the user responds. User responses will generate an Event under Alerts, which you can click to navigate directly to their comment. For more information, see the Viewing Alerts for Downloads and Comments section of our PhishER Settings: Account article.
