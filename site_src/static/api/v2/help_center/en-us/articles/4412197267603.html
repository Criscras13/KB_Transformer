In the Policy Management module of your KCM GRC platform, you can create policy campaigns to send policies to your users. When your users receive the policies in their email, they can read and acknowledge them. Then, as your users acknowledge the policies, you can monitor your campaign progress.\u00a0
\n
See the sections below to learn how to create and manage policy campaigns. For general information about the Policy Management module, see our Policy Management Module Guide.\u00a0
\n
Creating Campaigns
\n
When you create a campaign, you can select the policy or policies that you would like to assign to your users. You can also set campaign notifications to prompt your users to acknowledge the policies.\u00a0
\n
\n
\nImportant:Before you create a campaign, you will need to add policy management users to policy management groups. To learn how to create policy management users and groups, see our How to Add Users and Groups to the Policy Management Module.\u00a0
\n
\n
To create a policy campaign, follow the steps below:
\n
    \n
  1. From the navigation panel, navigate to Policy Management\u00a0> Campaigns.
    2. \n
  2. In the top-right corner of the page, click the Create Campaign button.
    3. \n
  3. Fill out the fields on the Create a New Campaign page. For more information about these fields, see the list below:\n
    \n
    \nNote:We recommend that you avoid including the < and > special characters in these fields.
    \n
    \n
    \n
      \n
    1. \nName: Enter a name for your campaign. This name will be included in any email notifications that you set up for this campaign.
      2. \n
    2. \nStart Date: Click this field to open a calendar where you can select a start date for your campaign.\u00a0
      3. \n
    3. \nStart Time: Enter a start time into the field, or click the clock icon to select a start time for your campaign.\n
      \n
      \nNote:You must select a start time that is at least 15 minutes after the time that you create the campaign.
      \n
      \n
      4. \n
    4. \nTime Zone: From the drop-down menu, select a time zone for your campaign's start time.
      5. \n
    5. \nEnd Campaign At: Select a type of end date for the campaign. To learn about each type of end date, see the list below:\n
        \n
      • \nSpecify Date: Select this option to set a specific end date for the campaign. When you select this option, End Date and\u00a0End Time\u00a0fields will appear. In these fields, you can select the date and time that you would like for your campaign to end on.\u00a0
        • \n
      • \nRelative Duration: Select this option to set a deadline based on when users are enrolled in the campaign. When you select this option, a Relative Duration field will appear. In this field, you can select the amount of time that users will have to acknowledge the policy or policies after you add the users to the campaign. The campaign notifications that you set up will be sent to users based on when they are enrolled in the campaign.
        • \n
      • \nNo End Date: Select this option to create the campaign without an end date. If you would like to encourage your users to promptly acknowledge your organization's policies, we do not recommend that you select this option.
        • \n
      \n
      6. \n
    6. \nLink Expiration: Select the amount of time that users can click the policy link before the link expires. We recommend that you select a time length that corresponds with the type of end date you selected for this campaign. For more information about the recommended link expirations, see the list below:\n
        \n
      • \nSpecify Date: We recommend that you select a link expiration that comes before the campaign's end date because policy acknowledgments will not be recorded after the campaign ends.
        • \n
      • \nRelative Duration: We recommend that you select a link expiration that is the same length of time as the relative duration. When users receive campaign notifications for campaigns with Relative Duration selected, the notifications will instruct the users to acknowledge the policy before the link expiration date.
        • \n
      • \nNo End Date: Because this campaign does not have an end date, you can select any link expiration. When your users receive campaign notifications for the campaign, the notifications will instruct the users to acknowledge the policy before the link expiration date.
        • \n
      \n
      7. \n
    7. \nSelect Campaign Policy(ies): From the drop-down menu, select the policy or policies that you would like for your users to acknowledge.
      8. \n
    8. \nSelect Campaign Group(s): From the drop-down menu, select the user groups that you would like to include in the campaign. For example, if your organization uses policies that only apply to specific job roles, you may want to create role-based user groups and policy campaigns. To learn more about user groups, see the Creating Policy Management Groups section of our How to Add Users and Groups to the Policy Management Module article.\u00a0
      9. \n
    9. \nAutomatically enroll users that are added in the future: Select this check box if you would like to automatically enroll users as they are added to the groups that are included in the campaign.
      10. \n
    10. \nNotifications: Click the + button to expand this menu. Then, select the types of notifications that you would like to include in the campaign. To learn about the types of campaign notifications that you can select, see the Selecting Policy Campaign Notifications section below.\n
      \n
      \nImportant:We recommend that you always include campaign notifications in a campaign. Without campaign notifications, your users will not receive the campaign emails that allow them to access your policies.\u00a0
      \n
      \n
      \n
      \nNote: If a user is enrolled in multiple campaigns at one time, they may receive two or more campaign notifications in one KCM GRC email. For more information, see the Policy Campaign Notifications section of our Email Notifications and the Email Digest article.
      \n
      \n
      11. \n
    \n
    4. \n
\n
\n
Selecting Policy Campaign Notifications
\n
When you create a campaign, you can select from six types of campaign notifications to send to your users or the campaign owner.\u00a0
\n
To learn about the types of campaign notifications you can include in a campaign, see the list below:\u00a0 
\n
    \n
  • \nOn Campaign Start (User): When the campaign starts, an email will be sent to all users who are included in the campaign.
    • \n
  • \nBefore Acknowledgement Due (User): Before the campaign's due date, an email will be sent to all users who are included in the campaign that have not acknowledged the policies yet. If you select this option, a Number of Days field will appear. In this field, you can enter the number of days before the due date that you would like for the notification email to send. You can add multiple instances of this type of notification by entering a different number in the Number of Days field for each instance.
    • \n
  • \nOn Campaign Start (Campaign Owner): When the campaign starts, an email will be sent to notify the creator of the campaign that the campaign has started. This notification will include the campaign's start date and start time.
    • \n
  • \nBefore Campaign End (Campaign Owner): Before the campaign's end date, an email will be sent to notify the creator of the campaign that the campaign is ending in a number of days. If you select this option, a Number of Days field will appear. In this field, you can enter the number of days before the due date that you would like for the notification email to send.
    • \n
  • \nOn Campaign Completion (Campaign Owner): When the campaign ends, an email will be sent to notify the creator of the campaign that the campaign has ended. This notification will include a link to sign in to KCM GRC and view the campaign.\n
    \n
    \nNote: If the campaign has a Relative Duration end date, you will not be able to select this type of notification.
    \n
    \n
    • \n
  • \nOn Campaign Enrollment (User): After the campaign starts, an email will be sent to any users that are added to the groups that are included in the campaign. This notification will include links to all of the policies that are included in the campaign.\n
    \n
    \nNote:You will only be able to select this type of notification for campaigns with Relative Duration and No End Date end dates.\u00a0
    \n
    \n
    • \n
\n
\n
Monitoring and Managing Policy Campaigns
\n
After your policy campaign starts, you can monitor the campaign's progress to see which users have acknowledged your policies.
\n
To view a campaign's progress, follow the instructions below:
\n
    \n
  1. Navigate to Policy Management > Campaigns.
    2. \n
  2. Select a campaign's name from the list of campaigns. When you select a campaign, you will be taken to the campaign's View Campaign page.
    3. \n
  3. Select the Overview, Users, and Members tabs to view details about the campaign.
    4. \n
\n
To learn about the Overview, Users, and Members tabs of the View Campaign page, see the subsections below.\u00a0
\n
\n
Overview Tab
\n
From the Overview tab, you can view information about the campaign\u2019s progress. For more information about this tab, see the list below:
\n
    \n
  1. \nProgress: View the number of policies that users who are included in the campaign have acknowledged.\u00a0
    2. \n
  2. \nPercentage: View the percentage of policies that users who are included in the campaign have acknowledged.
    3. \n
  3. \nUser Acknowledgement Activity: View the progress of your users' policy acknowledgments. This graph will show a daily, weekly, or monthly view, depending on how far apart the campaign's start date and end date are.\u00a0
    4. \n
  4. \nNudge Users: Click this button to automatically send a reminder email to all enrolled users who have not acknowledged the required policies. After you click the Nudge Users button, an alert will appear to report the number of users that were sent the reminder email.\u00a0 \n
    5. \n
  5. \nUpdate Campaign: Click this button to edit the campaign's details.\u00a0
    6. \n
  6. \nCampaign Details: View details about the campaign, such as the status, enrolled groups, and scheduled notifications.
    7. \n
\n
\n
Users Tab
\n
From the Users tab, you can view a breakdown of the users who are enrolled in the campaign. For more information about this tab, see the list below:\u00a0
\n
    \n
  1. \nSearch: Use this search bar to search for users who have been enrolled in the campaign.\u00a0
    2. \n
  2. \nFirst Name, Last Name, Email: View each users first name, last name, and email address.
    3. \n
  3. \nPlus or minus: Click this button to expand or minimize a user's progress for each policy in the campaign.\u00a0
    4. \n
  4. \nForce Acknowledgement for Selected: Click this button to mark the policy as acknowledged for any users that you have selected on this page.\u00a0
    5. \n
\n
\n
Members Tab
\n
From the Members tab, you can view a list of the users in your account who currently have access to view or edit the policy campaign. This tab displays each user's name, email address, and user role.
