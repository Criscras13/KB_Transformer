This article provides a comprehensive guide on disabling Prevent, including the necessary steps to modify Microsoft 365 settings, on-premise Microsoft Exchange configurations, and other secure email gateways. Whether you're transitioning to a different email security solution or need to disable Prevent temporarily, this step-by-step guide will walk you through the process, ensuring a smooth and secure transition for your organization's email communications.
\n
You will first need to complete the following steps:
\n
    \n
  • Disable any Egress mail flow rules within Microsoft 365 and any other secure email gateway.
    • \n
  • Disable the Egress Transport Agent if used in combination with on-premise Microsoft Exchange.
    • \n
  • Remove any Egress send and receive connectors in Microsoft 365 or on-premise Microsoft Exchange.
    • \n
  • Remove any Egress connectors or accepted relays at any other secure email gateway.
    • \n
\n
If you wish to continue using the Microsoft Outlook add-in without the gateway, you should remove the following registry keys which will have been used to enable gateway mode:
\n
    \n
  • [HKEY_LOCAL_MACHINE\\SOFTWARE\\Egress\\Switch] \"Forcegatewaymode\" \"Gatewayaddress\" \"MinimumSizeKB\"
    • \n
  • [HKEY_LOCAL_MACHINE\\SOFTWARE\\Wow6432Node\\Egress\\Switch] \"Forcegatewaymode\" \"Gatewayaddress\" \"MinimumSizeKB\"
    • \n
\n
If you do not wish to continue using our Outlook add-in, you should also remove this. At the end of your contract, any customized policies, including branding, will be reverted to the default configuration.
