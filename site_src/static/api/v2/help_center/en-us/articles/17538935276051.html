You can integrate the Phish Alert Button (PAB) with Microsoft Defender for Office to send emails to Microsoft\u2019s Submissions page for analysis. In the Microsoft 365 Defender platform, the Submissions page receives emails that users report with Microsoft\u2019s Report button. With this integration, you can allow your users to report suspicious emails to your organization and your Submissions page with the PAB. When users report KnowBe4\u2019s simulated phishing emails with the Report button, Microsoft can mistakenly classify these emails as threats. This integration also helps Microsoft recognize KnowBe4\u2019s simulated phishing emails and avoid false positives for users\u2019 reported emails.
\n
Prerequisites
\n
Before you configure this integration, you'll need to enable and configure the PAB in your KSAT Account Settings. For more information, see the Enable and Configure section of our Phish Alert Button (PAB) Product Manual.
\n
You'll also need to enable the Microsoft 365 Defender for Office service for your organization\u2019s mail server. Then, configure a security operations (SecOps) mailbox for your Microsoft account so that emails can be sent to your Submissions page. For more information, see Microsoft's Use the Microsoft 365 Defender portal to configure SecOps mailboxes in the advanced delivery policy article.
\n
\n
\nImportant:Microsoft Defender requires that emails be in EML format.
\n
\n
Configuring the Integration
\n
To configure this integration, you\u2019ll need to allow third-party reporting tools for your Microsoft 365 Defender platform. For more information, see Microsoft\u2019s Options for third-party reporting tools article. To allow the third-party reporting tools, follow the steps below:
\n
    \n
  1. Log in to your Microsoft 365 account using your admin credentials.
    2. \n
  2. Navigate to the Microsoft 365 Defender portal > Settings > Email & collaboration. \n
    3. \n
  3. \u00a0Click User reported settings. Or, you can access the User reported settings page by navigating to https://security.microsoft.com/securitysettings/userSubmission. \n
    4. \n
  4. On the User reported settings page, select the Monitor reported messages in Outlook check box to enable the reporting options.
    5. \n
  5. Under the Select an Outlook report button configuration section, select Use a non-Microsoft add-in button.\n
    \n
    \nImportant:If you select this option, Microsoft's Report button will be disabled automatically.
    \n
    \n\n
    6. \n
  6. Under the Send reported messages to: drop-down menu, select My reporting mailbox only or Microsoft and My reporting mailbox.\n
      
    \n
    \n
    \nNote:If you select Microsoft and my reporting mailbox, any reported messages will be automatically sent to Microsoft for analysis. For more information, see Microsoft's Options for third-party reporting tools section of their User reported settings article.
    \n
    \n
    7. \n
  7. In the Add an exchange online mailbox to send reported messages to: field, enter the email address associated with your Microsoft account\u2019s Submissions page and SecOps mailbox.\n
    \n
    \nNote:Enter the same email address you used to configure your SecOps mailbox. For more information, see Microsoft's Use the Microsoft 365 Defender portal to configure SecOps mailboxes in the advanced delivery policy article.
    \n
    \n\n
    8. \n
  8. (Optional) You can select the Allow reporting for quarantined messages. Only admins can report quarantined Teams messages. check box to allow your users to report emails from their quarantine folder.
    9. \n
\n
After you have configured these settings in Microsoft 365 Defender, you\u2019ll need to configure the integration in your KSAT console. To configure the integration in KSAT, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Click your email address in the top-right corner of the page. From the drop-down menu that opens, select Account Settings.
    3. \n
  3. Navigate to Account Integrations > Phish Alert.
    4. \n
  4. Click the plus icon next to the PAB instance to view your settings.
    5. \n
  5. Select the Enable Microsoft 365 Defender Integration check box.
    6. \n
  6. In the Submit Reported Emails to: field, enter the email address associated with your Microsoft account\u2019s Submissions page and SecOps mailbox.\n
    \n
    \nImportant:In addition to entering the email address associated with your Microsoft account, you can enter a different email address in the Send Non-Simulated Emails to: field to provide a copy of users\u2019 reported emails to your organization\u2019s admins. If you only want to send reported emails directly to your SecOps mailbox, leave the Send Non-Simulated Emails to: field empty.
    \n
    \n
    7. \n
  7. (Optional) You can select the Save a copy of reported emails check box if you would like the PAB to save a copy of reported emails in the Sent folder of the user who reported them. \n
    8. \n
  8. Click the Save Phish Alert Settings button.
    9. \n
  9. At the bottom of the page, click the Save Changes button.
    10. \n
\n
User Experience
\n
Once you've configured the integration, your users can use the PAB to report suspicious emails to both KnowBe4 and Microsoft.
\n
When a user reports an email, they must select a disposition. By default, they can select the Phishing/Suspicious or Spam/Junk disposition. If you have enabled the User Comments and Email Disposition feature, your users can also add comments and select the Unknown disposition.
\n
If a user selects the Unknown disposition, the reported email is initially labeled as Phishing in Microsoft\u2019s Submissions page. For more information, see our Phish Alert Button (PAB) User Comments and Email Disposition Guide. 
\n
\n
\nNote:Microsoft\u2019s Submissions page cannot receive emails that contain multiple attachments or files with comments. If a user adds a comment when they report an email, the comment file will not be included in the copy of the reported email sent to the Submissions page.
\n
\n
After a user reports an email, the email will be deleted from their inbox, and two copies of the reported emails will be sent. The first copy will be sent to the email address entered in the Send Non-Simulated Emails to: field in your KSAT Account Settings. The second copy will be sent to the email address in the Submit Reported Emails to: field in your KSAT Account Settings, associated with Microsoft Defender\u2019s Submissions page. If the Save a copy of reported emails setting is enabled, a copy of the reported email is also saved in the user\u2019s Sent folder.
\n
Once an email is submitted to Microsoft, it will display in the User reported tab on the Submissions page of your Microsoft 365 Defender portal within a few minutes.
