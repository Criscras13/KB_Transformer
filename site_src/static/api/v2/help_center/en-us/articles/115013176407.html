In this article, you can learn how to configure single sign-on (SSO) for your KSAT console with Okta. If your organization uses Okta, configuring SSO will allow your users to automatically sign in to the Learner Experience by authenticating with Okta.
\n
\n
\nImportant:For SSO to work properly, the email address that your users authenticate with must be entered in the Email or Email Aliases field of their User Profile. However, only the email address listed in the Email field will receive training notification emails. For more information, see our User Profile Guide.
\n
\n
\n
\nNote: You'll need an Okta subscription to configure single sign-on with Okta. For additional assistance, we recommend that you see Okta's documentation.
\n
\n
Configuring SSO with the Default Entity ID
\n
To configure SSO by using the default entity ID, follow the steps below:
\n
\n
\nImportant:If you're using a custom entity ID instead, follow the steps in the Configuring SSO for Okta Using a Custom Entity ID section below.
\n
\n
    \n
  1. Log in to your Okta account.
    2. \n
  2. Navigate to the Applications tab.
    3. \n
  3. Select Browse App Catalog.\n
    \n
    4. \n
  4. In the search box, type in KnowBe4 SAML, then select the KnowBe4 SAML application.
    5. \n
  5. Select + Add Integration.\n
    \n
    6. \n
  6. In the Application label field, enter a name for the integration.\n
    \n
    7. \n
  7. Select Done.
    8. \n
\n
In a separate browser tab or window, you'll need to copy your SAML ID from KSAT. To copy your SAML ID, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Select your email address in the top-right corner, then select Account Settings.
    3. \n
  3. In your Account Settings, navigate to SAML > SAML Settings.
    4. \n
  4. Copy the text in the SAML ID field.
    5. \n
\n
Finally, you'll need to return to Okta to finish configuring SSO.
\n
    \n
  1. Navigate to the KnowBe4 SAML application again.
    2. \n
  2. Select the Sign On subtab.
    3. \n
  3. Select Edit.\n
    \n
    4. \n
  4. Go to the Advanced Sign-on Settings section.
    5. \n
  5. In the SAML ID field, paste the SAML ID you copied above.
    6. \n
  6. In the Base URL field, enter the URL for the KSAT instance you use. The instances are listed below:\n
      \n
    • \nUnited States: https://training.knowbe4.com
      • \n
    • \nEuropean Union: https://eu.knowbe4.com
      • \n
    • \nCanada: https://ca.knowbe4.com
      • \n
    • \nGermany: https://de.knowbe4.com
      • \n
    • \nUnited Kingdom: https://uk.knowbe4.com
      • \n
    \n
    7. \n
  7. Select Save.
    8. \n
  8. Select More details.\n
    \n
    \nNote:For SSO to work properly, you may need to change the Application username format field to Email. To edit the settings on this page, select the Edit button.
    \n
    \n
    \n
    9. \n
  9. Select Certificate fingerprint. \n
    10. \n
  10. Copy the IDP SSO Target URL and IDP Cert fingerprints and save them. You'll need these fingerprints to finish configuring SSO.
    11. \n
  11. Finish setting up SAML in your KSAT console. For more information, see our Set Up SAML Single Sign-on (SSO) for the Security Awareness Training Console article.
    12. \n
\n
\n
\nImportant:Before your users can start authenticating using Okta, you'll need to assign them to the KnowBe4 app in Okta. You can assign users and groups from the Assignments subtab.
\n
\n
Configuring SSO with a Custom Entity ID
\n
To configure SSO by using a custom entity ID, follow the steps below:
\n
\n
\nImportant:If you're using the default entity ID instead, follow the steps in the Configuring SSO for Okta Using a Default Entity ID section above.
\n
\n
    \n
  1. Log in to your Okta account.
    2. \n
  2. Navigate to the Applications tab.
    3. \n
  3. Select Browse App Catalog.\n
    4. \n
  4. In the search box, type in KnowBe4 SAML, then click the KnowBe4 SAML application.
    5. \n
  5. Select + Add Integration.
    6. \n
  6. In the Application label field, enter a name for the integration.
    7. \n
  7. Select Done.
    8. \n
\n
In a separate browser window, you'll need to create your Unique Entity ID from KSAT.
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Select your email address at the top-right corner, then select Account Settings.
    3. \n
  3. In your Account Settings, navigate to SAML > SAML Settings.
    4. \n
  4. Select on Generate Unique Entity ID.
    5. \n
\n
Finally, return to Okta to continue configuring SSO.
\n
    \n
  1. Navigate to the KnowBe4 SAML application again.
    2. \n
  2. Select the Sign On subtab.
    3. \n
  3. Select Edit.
    4. \n
  4. Go to the Advanced Sign-on Settings section.
    5. \n
  5. In the SAML ID field, paste the SAML ID from KSAT.
    6. \n
  6. In the Entity ID field, paste the Entity ID from KSAT.
    7. \n
  7. In the Base URL field, enter the URL for the KSAT instance you use. The instances are listed below:\n
      \n
    • United States: https://training.knowbe4.com\n
      • \n
    • European Union: https://eu.knowbe4.com\n
      • \n
    • Canada: https://ca.knowbe4.com\n
      • \n
    • Germany: https://de.knowbe4.com\n
      • \n
    • United Kingdom: https://uk.knowbe4.com\n
      • \n
    \n
    8. \n
  8. Select Save.
    9. \n
  9. Select More details.
    10. \n
  10. Select Certificate fingerprint.\n
    11. \n
  11. Copy the IDP SSO Target URL and IDP Cert fingerprints and save them. You'll need these values to finish configuring SSO.
    12. \n
  12. Finish setting up SAML in your KSAT console. For more information, see our Set Up SAML Single Sign-on (SSO) for the Security Awareness Training Console article.
    13. \n
\n
\n
\nImportant:Before your users can start authenticating using Okta, you'll need to assign them to the KnowBe4 app in Okta. You can assign users and groups from the Assignments subtab.
\n
