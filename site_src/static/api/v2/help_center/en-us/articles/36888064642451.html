In this article, you will learn how to integrate Splunk vendors with SecurityCoach. Once you set up this integration, data provided by the Splunk vendors will be available under the SecurityCoach tab of your KnowBe4 console. This data can be viewed in SecurityCoach reports and used in your real-time coaching campaigns. For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n

\n
The Splunk integration for SecurityCoach currently supports the following vendors. These vendors can also be integrated directly with SecurityCoach by following their linked direct integration guides:
\n

  • Bitdefender GravityZone

Whitelisting Splunk
\n
Before setting up any integrations in your Splunk console, you must ensure your Splunk IP address is whitelisted for your SecurityCoach console. To ensure Splunk is whitelisted, submit a support ticket to KnowBe4 that indicates if you are using Splunk Cloud or Splunk Enterprise and includes the public static IP address associated with your Splunk instance. A member of our support team will whitelist your IP address and ensure that Splunk has been successfully integrated.
\n
If you are using Splunk Cloud, follow these steps to locate your public static Splunk Cloud IP address:
\n
    \n
  1. Open the Command Prompt (Windows) or Terminal (MacOS or Linux) program on your device.
    2. \n
  2. Enter \u201cnslookup\u201d in the command line text field. A new text field will appear.
    3. \n
  3. In the new text field, enter the domain for your Splunk Cloud instance.\n
    \n
    \nTip:The start of your Splunk Cloud domain is unique to your organization, but always ends with: \".splunkcloud.com\"
    \n
    \n
    4. \n
  4. Your Splunk Cloud IP address will be shown in the search results.\n
      
    \n
    5. \n
  5. Copy or screenshot the IP address to share with KnowBe4 support.
    6. \n
\n
\n
\nImportant:If your Splunk IP address ever changes, reach out to our support again so that they can whitelist your new IP address.
\n

Setting Up the Integration in SecurityCoach
\n
\n
\nImportant:You will need to complete the steps in both the Setting Up the Integration in SecurityCoach and the Setting Up the Integration in Splunk sections for each vendor you wish to integrate.
\n
\n
To set up the Splunk vendor integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to SecurityCoach > Setup > Security Vendor Integrations.
    3. \n
  3. Navigate to your desired Splunk vendor.
    4. \n
  4. Click Configure on the vendor\u2019s card and then click Splunk Integration from the drop-down menu to navigate to the integration setup page.\n
      
    \n
    5. \n
  5. Under Step 1, click Enable Splunk for [[vendor]] to generate a unique Splunk Alert Title value.\n
      
    \n
    \n
    \nWarning:Each time Splunk is enabled, a new Splunk Alert Title value is generated, and previously generated values will no longer be connected to your SecurityCoach console.
    \n
    \n
    6. \n
  6. Under Step 2, copy the Splunk Alert Title value by clicking the copy icon to the right of the value. You will need this value for the Setting Up the Integration in Splunk section.\n
      
    \n
    7. \n
  7. For Step 3, proceed to the steps in the Setting Up the Integration in Splunk section.\n
      
    \n
    8. \n
\n
Setting Up the Integration in Splunk
\n
\n
\nRemember:Complete the Setting Up the Integration in SecurityCoach section before starting this section.
\n
\n
To set up the SecurityCoach integration for a supported vendor in your Splunk console, create an associated vendor index alert by following the steps below:
\n
    \n
  1. Log in to your Splunk admin portal.
    2. \n
  2. From the Splunk homepage, click Search & Reporting from the Apps menu on the left side of the screen.\n
      
    \n
    3. \n
  3. Search for the name of the vendor you set up in the Setting Up the Integration in SecurityCoach section of this article. To search for a vendor, enter \u201cindex = [[vendor name]]\u201d in the Search text field.\n
      
    \n
    \n
    \nTip:To send only specific event data from the vendor to SecurityCoach, see the Limiting Events Sent to SecurityCoach section below to create a custom index and enter that custom index into the Search field instead.
    \n
    \n
    4. \n
  4. Verify that the index search for the vendor has incoming data in the Event column and that the data is accurate.\n
      
    \n
    5. \n
  5. Create an alert from the search results. To create an alert, click Save As near the top-right corner of the screen, then click Alert from the drop-down menu.\n
      
    \n
    6. \n
  6. To configure and save the alert, see the screenshot and list below:\n
      
    \n
      \n
    1. Enter the Splunk Alert Title token you copied earlier in Step 7 of the Setting Up the Integration in SecurityCoach section.
      2. \n
    2. Set the top Alert Type selection to Scheduled.
      3. \n
    3. Set the bottom Alert Type selection to Run on Cron Schedule.\n
        
      \n
      4. \n
    4. Click the Time Range selection to open a new window. From there, click Advanced, set the Earliest field to \u201c-6m\u201d and the Latest field to \u201c-1m\u201d, then click Apply.\n
        
      \n
      5. \n
    5. Set Cron Expression to \u201c*/5 * * * *\u201d.
      6. \n
    6. \n
      Set Expires to the length of time that you want this alert to show on your Triggered Alert page in Splunk.
      \n
      \n
      \nTip:We recommend setting this value to 365 days.
      \n
      \n
      7. \n
    7. Set Trigger alert when to Number of Results, is greater than, and \u201c0\u201d.
      8. \n
    8. Click Add Actions to open a drop-down menu. Click Webhook to add the When triggered > Webhook > URL field to your configuration.\n
        
      \n
      9. \n
    9. In the URL field, enter your KnowBe4 instance\u2019s Webhook URL from the table below:\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
      KnowBe4 InstanceWebhook URL

      European Unionhttps://splunk.vendor.eu.knowbe4.com/v1
      Canadahttps://splunk.vendor.ca.knowbe4.com/v1
      United Kingdomhttps://splunk.vendor.uk.knowbe4.com/v1
      Germanyhttps://splunk.vendor.de.knowbe4.com/v1
      \n
      10. \n
    \n
    7. \n
  7. Click Save to save the vendor index alert.
    8. \n

Limiting Events Sent to SecurityCoach
\n
To send only specific events from a Splunk vendor to SecurityCoach, create a custom index in Splunk to filter which events are sent. For more information, see Splunk\u2019s Create Custom Indexes article.
\n
Mapping Your Users
\n
After you\u2019ve finished integrating your Splunk vendor, the final step is to map your users. Some vendors map your users automatically, which will be indicated on the vendor\u2019s Setup page.
\n
  
\n
Other vendors need to be mapped to your users, which will also be indicated on the vendor\u2019s Setup page. For these vendors, map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Map Users in SecurityCoach article.
\n
Managing System Detection Rules
\n
Once you\u2019ve successfully integrated the Spunk vendor, use the Detection Rules subtab to manage detection rules for the integration data. For more information, see our Detection Rules Guide. For a full list of available system detection rules available for Splunk vendors, see our System Detection Rules by Vendor article.
\n
Deleting a Splunk Integration
\n
To delete a Splunk vendor integration with SecurityCoach, we recommend first deleting the integration in your Splunk console, and then you must delete the integration in your SecurityCoach console.
\n
Deleting the Vendor Index Alert in Splunk
\n
To delete the associated vendor index alert from Splunk, follow the steps below:
\n
    \n
  1. Log in to your Splunk admin console.
    2. \n
  2. From the Splunk homepage, click Search & Reporting from the Apps menu on the left side of the screen.
    3. \n
  3. Click the Alerts subtab.
    4. \n
  4. Locate your Splunk Alert Title in the Title column.\n
    \n
    \nNote:The Splunk Alert Title can be found in your SecurityCoach console on the associated vendor\u2019s Setup page. For more information, see Step 7 of the Setting Up the Integration in SecurityCoach section.
    \n
    \n
    5. \n
  5. In the Actions column to the right of your Splunk Alert Title, click Edit.
    6. \n
  6. From the drop-down menu, click Delete to open a pop-up confirmation window. Click Delete again to confirm deletion.
    7. \n
\n
Deleting the Integration in SecurityCoach
\n
To delete the Splunk vendor integration from SecurityCoach, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to SecurityCoach > Setup > Security Coach Vendor Integrations.
    3. \n
  3. Locate the Splunk vendor tile and click View.
    4. \n
  4. Click Delete Integration near the bottom of the page.
    5. \n
  5. A new pop-up window will open. If you are sure you want to delete the integration, click Confirm.
    6. \n
