The Recent Emails page in the Defend admin console allows admins to view and analyze all emails processed by Defend. This article explains how to use the filtering options, view email details, and take necessary actions.
\n
Date Selection\u00a0
\n
At the top of the Recent Emails page, use the date picker to specify the time period for which you want to view emails.
\n
\n
Available Filters\u00a0
\n
By default, the Dangerous and Exclude Phishing Simulation Tests filters are applied. The following filters can be used individually or in combination to refine your email search.
\n
Basic Filters\u00a0
\n
The fields you can use for these filters are listed below:
\n
    \n
  • Internet Message ID: Search for a specific email using its unique identifier
    • \n
  • Subject: Filter by email subject line
    • \n
  • From: Filter by the sender's display name or email address
    • \n
  • Mail From: Filter by the sender's envelope address
    • \n
  • To: Filter by the recipient's display name or email address
    • \n
  • Recipient/Rcpt To: Filter by recipient's envelope address
    • \n
\n
\n
User Interactions Filters
\n
The options you can select for these filters are listed below:
\n
    \n
  • Link clicked
    • \n
  • Email response
    • \n
  • Phish reported
    • \n
  • Not phish reported
    • \n
\n
\n
Threat Classification\u00a0
\n
The options you can select for these filters are listed below:\u00a0
\n
    \n
  • Dangerous\u00a0
    • \n
  • Suspicious\u00a0
    • \n
  • Dangerous & Suspicious\u00a0
    • \n
  • All\u00a0
    • \n
\n
Please note that selecting the\u00a0All\u00a0option will include benign emails.\u00a0
\n
\n
Email Location\u00a0
\n
The options you can select for these filters are listed below:
\n
    \n
  • Inbox
    • \n
  • Junk
    • \n
  • Archive
    • \n
  • Deleted Items
    • \n
  • M365 Quarantine
    • \n
  • Other
    • \n
\n
\n
Phishing Simulation Tests\u00a0
\n
There is only one option in this filter that is selected by default:
\n
    \n
  • Exclude Phishing Simulation Tests\u00a0
    • \n
\n
Advanced Filters\u00a0
\n
The options you can select for these filters are listed below:
\n
    \n
  • Remediation Status
    • \n
  • Payload Type
    • \n
  • Microsoft SCL
    • \n
  • Email Productivity
    • \n
  • Phish Type
    • \n
  • Email Authentication
    • \n
  • From Exclusions
    • \n
  • High Confidence Phish
    • \n
\n
\n
Wildcards
\n
Wildcarding can be used within the filter text fields to offer more granular searching. Use an asterisk before or after your text. For example, to return all results from example.com addresses, search for \"*@example.com\" in the\u00a0From\u00a0text box.\u00a0
\n
Exporting Search Results\u00a0
\n
The\u00a0Recent Emails\u00a0page offers multiple options for exporting filtered email data:
\n
    \n
  • Export Page As CSV\n
      \n
    • Exports only the emails displayed on the current page
      • \n
    • Data is exported in CSV format\u00a0
      • \n
    \n
    • \n
  • Export Current Filter As CSV\n
      \n
    • Exports all emails matching your current filter criteria\u00a0
      • \n
    • Data is exported in CSV format
      • \n
    \n
    • \n
  • Export Page As JSON\u00a0\n
      \n
    • Exports only the emails displayed on the current page\u00a0
      • \n
    • Data is exported in JSON format
      • \n
    \n
    • \n
  • Export Current Filter As JSON\n
      \n
    • Exports all emails matching your current filter criteria\u00a0
      • \n
    • Data is exported in JSON format
      • \n
    \n
    • \n
\n
\n
Column Options
\n
The\u00a0Recent Emails page displays four columns by default:
\n
    \n
  • Time\u00a0
    • \n
  • From\u00a0
    • \n
  • To
    • \n
  • Threat
    • \n
\n
You can customize which columns appear using the\u00a0Column Options drop-down menu. Your selected preferences will be saved and applied each time you return to this page.\u00a0
\n
Email Details Pane\u00a0
\n
When you select an email from the list, a detailed pane opens on the right side of the screen. This pane contains five tabs, which are discussed below.
\n
Basic Tab\u00a0
\n
The\u00a0Basic tab displays key information such as sender and recipient information, email location, authentication results, and user interactions.\u00a0
\n
The available actions in this tab are:
\n
    \n
  • View the Email Summary page
    • \n
  • Set Investigation Status
    • \n
  • Add email to deny list
    • \n
  • Add email to allow list
    • \n
\n
\n
Advanced Tab
\n
The Advanced tab displays detailed information such as extended sender and recipient details, comprehensive email location data, complete authentication results and details, email attachment information, and email link analysis.
\n
The available actions in this tab are:
\n
    \n
  • Set Investigation Status
    • \n
  • Add email to deny list
    • \n
  • Add email to allow list
    • \n
\n
\n
Intelligence Tab
\n
The Intelligence tab displays the information gathered by Defend for this email, such as phish classification, the payload detected, the Defend banners added to the email, and sender location details. Communication history between the sender and recipient is also displayed. The user interactions and user report sections show how the user has interacted with the email and whether they have reported the email in any way.
\n
The available actions in this tab are:
\n
    \n
  • Set Investigation Status
    • \n
  • Add email to deny list
    • \n
  • Add email to allow list
    • \n
\n
\n
Operations Tab
\n
This tab displays information retrieved using the integration between Microsoft 365 and Defend. All recipients who have received this email will be displayed, along with the email\u2019s location. The Threat Remediation section allows you to view and remediate exact and similar emails sent to other users.
\n
The available actions in this tab are:
\n
    \n
  • Adjust message property filters for broader remediation
    • \n
  • Remediate Selected Emails
    • \n
  • Remediate All Emails
    • \n
\n
\n
Comments Tab
\n
Defend admins can add comments to an email to collaborate and discuss an investigation or email status. Investigation status changes can be viewed live in the comments feed.
\n
The available actions in this tab are:
\n
    \n
  • Set Investigation Status
    • \n
  • Add email to deny list
    • \n
  • Add email to allow list
    • \n
