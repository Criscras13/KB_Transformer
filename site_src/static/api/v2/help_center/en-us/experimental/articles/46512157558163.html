You can integrate your Workato platform into your KSAT console. Workato allows you to utilize Workbots for conversational process automation, allowing users to initiate KnowBe4 actions, view statuses, or receive context-aware notifications directly within Slack or Microsoft Teams. Workato utilizes KnowBe4\u2019s Graph APIs to enable automated security awareness training workflows and data management in areas including groups, users, phishing, and training.
\n
Before You Connect Workato to Your KSAT Console
\n
Before integrating Workato into your KSAT console, you must find your API address and create a Product API token. Follow the steps below to do both actions.
\n
\n
\nNote:KnowBe4's Graph APIs are available to Diamond-level customers. For more information about Graph APIs, please see our KnowBe4 Graph API Overview article.
\n
\n
    \n
  1. Your API address depends on where your KnowBe4 account is located. Please see the table below to find the API address for your region.\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
    RegionBase URL
    UShttps://us.api.knowbe4.com
    CAhttps://ca.api.knowbe4.com
    EUhttps://eu.api.knowbe4.com
    UKhttps://uk.api.knowbe4.com
    DEhttps://de.api.knowbe4.com
    \n
    2. \n
  2. Log in to your KSAT console as an admin.
    3. \n
  3. Navigate to Account Settings in the top-right corner under your account\u2019s email address.
    4. \n
  4. In Account Settings, navigate to Account Integrations.
    5. \n
  5. Under Product API, select Product API. The Product API subtab will open.
    6. \n
  6. At the top-right corner of the Product API subtab, select + Create New API Token.
    7. \n
  7. Fill out the fields on the Create New API Token page.
    8. \n
  8. Select Create Token. The Product API Token pop-up window will open.
    9. \n
  9. Select the Product API token to copy the token. Be sure to save your copied API token in a location that you can easily access. After you close this window, you will not be able to view this token again.
    10. \n
\n
Connecting Workato to Your KSAT Console
\n
You can connect your Workato platform to your KSAT console by following the steps below.
\n
    \n
  1. Log in to your Workato platform as an admin.
    2. \n
  2. In your workspace, navigate to Projects > Create project.\n
      
    \n
    3. \n
  3. Provide the Project name (required) and Project description, then select Create project. It is suggested that you create a Project name for testing purposes only, as seen in the screenshot below.\n
      
    \n
    4. \n
  4. In your newly created project, navigate to Create > Connection.\n
      
    \n
    5. \n
  5. Search for and select KnowBe4 to establish a connection.\n
      
    \n
    6. \n
  6. Provide the Connection name, Location, API Key, and API Region of your KnowBe4 account. You should have the Product API token and API address from the previous steps.\n
      
    \n
    7. \n
  7. Select Connect.
    8. \n
\n
Setting Up KnowBe4 Workflows in Workato
\n
You can set up KSAT workflows in your Workato platform by following the steps below.
\n
    \n
  1. In your project, navigate to Create > Recipe.\n
      
    \n
    2. \n
  2. Provide a Name and Location for your new recipe.
    3. \n
  3. Select a starting point for your recipe. In this example, we will select Run on a schedule. Then, select the Start building button.\n
      
    \n
    4. \n
  4. Next, you should see an Actions page. In this example, our Action in app will be the Trigger every 5 minutes option.\n
      
    \n
    5. \n
  5. Next, you will be taken to the Choose an app section. Search for and select the KnowBe4 app.\n
      
    \n
    6. \n
  6. Under Actions, choose the action you want to perform. In this example, you want to select the Archive one or more users by ID option.\n
      
    \n
    7. \n
  7. You should already have an existing connection to KnowBe4. If you haven't yet connected, you will be prompted to enter your connection name, Location, API Key, and API Region to establish a connection.
    8. \n
  8. Next, provide the required information for the action you want to perform. In this case, you want to select the Archive one or more users by ID option, so you would select one or more users from the drop-down menu to archive them.\n
      
    \n
    9. \n
  9. Select Save at the top of this page.
    10. \n
  10. Select the Test recipe option, located next to the Save button, to test your action. You should be able to view Output, which will display the action you have taken.\n
      
    \n
    11. \n
  11. Verify your action against what is displayed in the KSAT console. In this example, the archived user\u2019s email address should be displayed in the Users tab.\n
      
    \n
    12. \n
  12. You can also create multiple actions by using the recipes workflow, as shown in the screenshot below:\n
      
    \n
    13. \n
\n
If you experience issues integrating Workato with your KSAT console, please contact Workato\u2019s technical support.
