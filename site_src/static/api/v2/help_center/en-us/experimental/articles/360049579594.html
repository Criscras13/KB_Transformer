In KCM GRC, a control is a process, technical implementation, or other action that demonstrates how you are meeting your compliance requirements or objectives. In your KCM GRC platform, you can create a control individually by creating a custom control or by creating a control from a requirement. You can also create controls in bulk by using a CSV file or by creating controls from requirements. We recommend that you create controls individually so you can focus on one compliance objective at a time.
\n
\n
\nTip: We include guidance for the requirements under many of the managed templates that KCM GRC offers. Guidance provides information that can help you create controls to satisfy requirements. To learn more about the guidance under our managed templates, see our How to Work with Control Guidance article.
\n
\n
See the sections below to learn how to create controls individually and create controls in bulk.
\n
\n
\nNote: After you've created and mapped your controls to requirements, we recommend that you create recurring tasks for your controls. Recurring tasks can help your organization demonstrate that it is consistently meeting its compliance requirements. For more information, see our How to Work with Tasks for Controls article.\n
\n
\n
Creating Controls Individually
\n
Creating controls individually may be the best option for your organization if controls will vary for your scopes or if your organization has not established a process for documenting compliance requirements. Once you've created one or more scopes, you can navigate to the View Scoped Requirement page to create individual controls for scoped requirements. You can either create custom controls or create controls from the scoped requirements. When you create a control from the View Scoped Requirements page, the control will automatically be mapped to the scoped requirement.\u00a0
\n
To navigate to a View Scoped Requirement page, follow the steps below.
\n
    \n
  1. From the navigation panel, select the\u00a0Scopes tab (Compliance > Scopes).
    2. \n
  2. From the View All Scopes page, select a scope under the Name column. When you select a scope, you'll be taken to the\u00a0View Scope page. \n
    3. \n
  3. From the View Scope page, click the Requirements tab.
    4. \n
  4. Select a requirement under the\u00a0Name column. \n
    5. \n
\n
See the subsections below to learn how to create custom controls and create controls from scoped requirements.
\n
\n
Creating Custom Controls
\n
Follow the steps below to create a custom control.
\n
    \n
  1. From the Controls section of the View Scoped Requirement page, click the Create Control button. \n
    2. \n
  2. Fill out the fields on the Create Control for Requirement page. For more information about these fields, see the list below.\n
    \n
    \nNote:We recommend that you avoid including the < and > special characters in these fields.
    \n
    \n
      \n
    1. \nName: Enter a name that represents the purpose of the control. The name can be up to 255 characters, including spaces.
      2. \n
    2. \nControl Description: Enter a detailed description of the control. The description can be up to 10,000 characters, including spaces.\n
        \n
      • The description should include what the control is, how to review and assess the control, and what type of evidence is expected to satisfy the control. See our Glossary of Compliance Terms to learn more about control descriptions.
        • \n
      \n
      3. \n
    3. \nTags: (Optional) You can add one or more tags to your control. Tags allow you to group similar controls together in your platform.To add an existing tag, click the drop-down menu and select the tag you'd like to add. To create a new tag, enter one or more words into the field, and press Enter on your keyboard. Tags can be up to 25 characters long, including spaces.
      4. \n
    \n\n
    3. \n
  3. (Optional) If you'd like to create an additional control for this requirement, select the Create Another Control check box.
    4. \n
  4. Click the Create button.
    5. \n
  5. Repeat step 1 through step 4 for the remaining requirements in your scope. To navigate to the next requirement in your scope, click the Next Requirement button in the top-right corner of the View Scoped Requirement page. \n
    6. \n
\n
After you create the control, the control will display in the Controls area of the View Scoped Requirement page. In the\u00a0Controls area, you can view a table that displays all controls that are mapped to the scoped requirement.
\n
\n
\nNote: If the control applies to additional scoped requirements, you can map the control to the scoped requirements. For more information, see our How to Map Requirements and Controls article.
\n
\n
\n
Creating a Control from a Requirement
\n
In some cases, you may want to create controls that have the same name and description as the scoped requirement. For example, the requirements in our FedRAMP managed templates contain verbiage from NIST 800-53. Therefore, these requirements provide actionable controls that your organization should have in place to pass a FedRAMP authorization assessment.\u00a0
\n
To create a control from a requirement, navigate to the\u00a0View Scoped Requirement page. Then, click the Create Control from\u00a0Requirement\u00a0button.
\n
KCM GRC will use the requirement name and description to automatically create a control that is mapped to this requirement. Then, you can view the control in the Controls area of the View Scoped Requirement page.\u00a0
\n
\n
Creating Controls in Bulk
\n
Creating controls in bulk may be the best option for your organization if controls are applicable to multiple scoped requirements or if your organization has already established a process for documenting compliance requirements. You can create controls in bulk by importing controls with a CSV file or by creating controls from requirements.\u00a0
\n
See the subsections below to learn about these methods.\u00a0
\n
Importing Controls with a CSV file
\n
\n
\nImportant: When you import controls with a CSV file, the controls will be created separately from scoped requirements. After you import controls, we recommend that you map the controls to scoped requirements. To learn how to map controls to scoped requirements, see our How to Map Requirements and Controls article.
\n
\n
To import controls in bulk, begin by creating a CSV file. When you create your CSV file, make sure the file meets the specifications listed below.
\n
    \n
  • The separator should be a comma.
    • \n
  • The following header line is required, and it is case-sensitive.\n
    name, description
    \n
    • \n
  • All fields are mandatory.\n
    \n
    \nNote:We recommend that you avoid including the < and > special characters in your CSV file. When you import the CSV file into your account, these special characters can cause fields to import unsuccessfully.\u00a0
    \n
    \n
    • \n
  • The name field has a 255 character limit.
    • \n
  • The description field has a 10,000 character limit.
    • \n
\n
\n
\nImportant: If you are using Excel, your file must be saved in the CSV UTF-8 (Comma-delimited) format.
\n
\n
Once you've created your CSV file, follow the steps below to import the controls into your account.
\n
\n
\nNote:When you import a CSV file into your platform, KCM GRC automatically removes duplicate entries. For example, if you enter two controls with the same name and description, only one of the controls will import.
\n
\n
    \n
  1. From the navigation panel, select the Controls tab.
    2. \n
  2. Click the Upload CSV button in the top-right corner of the Controls Library page. \n
    3. \n
  3. From the Import Items window, click the Click to Upload button and select your CSV file.
    4. \n
  4. (Optional) If you'd like to remove a control from the list of controls, click the trash can icon next to the control's Description.
    5. \n
  5. Click the Import Items button to import the controls.\u00a0 \n
    6. \n
\n
After you've imported your controls, you can view the controls from the\u00a0Controls tab of your platform.\u00a0
\n
\n
Creating Controls from Requirements
\n
You can create controls from requirements in bulk, which will automatically map the controls to requirements.\u00a0
\n
To create controls from requirements, follow the instructions below.\u00a0
\n
    \n
  1. From your navigation panel, select the\u00a0Scopes\u00a0tab\u00a0(Compliance\u00a0>\u00a0Scopes).
    2. \n
  2. Select a scope under the\u00a0Name\u00a0column.
    3. \n
  3. From the\u00a0View Scope\u00a0page, select the\u00a0Requirements\u00a0tab.
    4. \n
  4. Click the check box next to each requirement that you want to create a control from.
    5. \n
  5. Click the\u00a0Create Controls from Requirements\u00a0button. \u00a0\u00a0 \n
    6. \n
  6. In the pop-up window, click the\u00a0Accept button.\u00a0
    7. \n
