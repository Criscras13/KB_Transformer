In this article, you\u2019ll learn how to integrate Cloudflare Zero Trust with SecurityCoach. Once the integration is complete, data provided by Cloudflare Zero Trust will be available for use under the SecurityCoach tab of your KSAT console. This data can be viewed in SecurityCoach reports and used to create real-time coaching campaigns. For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n\n
Set Up the Integration in Your Cloudflare Zero Trust Account
\n
Before you can set up this integration in your KSAT console, you will need to create an API Token and copy your Account ID from your Cloudflare Zero Trust platform.
\n
    \n
  1. Log in to your Cloudflare dashboard, then click My Profile from the navigation menu on the left side of the screen.\n\n
    2. \n
  2. Under the My Profile drop-down menu, click API Tokens > Create Token. \n\n
    3. \n
  3. Click Get started to create an API Token with custom permissions. \n\n
    4. \n
  4. In the Token name field, enter a name for your new token. Then, assign permissions as shown in the screenshot:\n\n
    5. \n
  5. Click Continue to summary. You will see a summary of the permission you\u2019ve assigned to your API token. Ensure your permissions match the screenshot below: \n\n
    6. \n
  6. Click Create Token.
    7. \n
  7. Click Copy to copy your API Token. Make sure to save your copied API Token somewhere that you can easily access. You will need this token when you complete the integration in the Set Up the Integration in Your KSAT Console section below. \n\n
    8. \n
  8. Locate your Account ID in your Cloudflare dashboard. For example, in the image below, the Account ID is daa2f528f306548dcdd584886da4b318 \n\n
    9. \n
\n\n\n
Set Up the Integration in Your KSAT Console
\n
Once you have created your API Token and copied your Account ID from your Cloudflare Zero Trust account, you can set up the integration in your KSAT console. To register Cloudflare Zero Trust in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and navigate to SecurityCoach > Setup > Security Vendor Integrations.
    2. \n
  2. Locate Cloudflare Zero Trust and click Configure.
    3. \n
  3. Enter the API Token and Account ID that you saved in your Set Up the Integration in Your Cloudflare Zero Trust Account section of this article. \n\n
    4. \n
  4. Click Authorize.
    5. \n
\n\n\n
Map Your Users
\n
After you\u2019ve finished integrating Cloudflare Zero Trust, you can map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Mapping Users in SecurityCoach article.
\n
Once you\u2019ve successfully authorized this integration, you can manage detection rules for Cloudflare Zero Trust on the Detection Rules subtab of SecurityCoach. For a full list of available system detection rules for this vendor, see our Which Detection Rules Can I Use with My Vendors? article.
