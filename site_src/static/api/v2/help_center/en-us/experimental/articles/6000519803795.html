If your organization uses Atlassian Jira Software to manage your tasks, you can integrate your Jira account with your KCM GRC platform. By using this integration, you can send copies of your KCM GRC tasks to Jira and assign Jira users to the tasks. Then, the task assignees can submit evidence to the tasks and complete the tasks in Jira, which will automate the task completion process in your KCM GRC platform.\u00a0
\n
\n
\nNote:This integration is only compatible with the Jira Cloud version of Jira Software. You're unable to integrate your platform with Jira Work Management or Jira Service Management.
\n
\n
To set up the integration, you'll need to create a Jira API token and then configure the integration in your KCM GRC platform. To complete this process, see the sections below.
\n
\n
\nImportant: The API token that you enter for this integration will be encrypted and used to authenticate your Jira account. When communicating data from your KCM GRC platform to your Jira account, we use a Secure Sockets Layer (SSL) to encrypt and secure your data.
\n
\n
\n
Prerequisites\u00a0
\n
Before you create your Jira API token, you'll need access to a Jira user account that meets the requirements listed below:
\n
\n
\nNote:To set up this Jira user account, you'll need to create a permission scheme that has the permissions listed below. Then, add the user to the permission scheme. For more information, see Jira's Managing project permissions article.
\n
\n
    \n
  • The user has the Administer Jira global permission.
    • \n
  • The user has access to the project that KCM GRC tasks will be sent to.
    • \n
  • The user can create and update issues in the project that KCM GRC tasks will be sent to.
    • \n
  • The user can update comments in the project that KCM GRC tasks will be sent to.
    • \n
\n
\n
Creating a Jira API Token
\n
To connect your Jira account with your KCM GRC platform, you'll need to create a Jira API token.
\n
\n
\nImportant:You'll need to create an API token from a Jira account that meets the requirements listed in the Prerequisites\u00a0section above.
\n
\n
To create a Jira API token, follow the steps below:
\n
    \n
  1. Log in to a Jira admin account.
    2. \n
  2. At the top-right corner of the page, click the gear icon.
    3. \n
  3. Select\u00a0Atlassian account settings.
    4. \n
  4. From the menu on the left side of the page, select the Security tab. \n
    5. \n
  5. In the\u00a0API token of the\u00a0Security\u00a0page, click Create and manage API tokens. \n
    6. \n
  6. On the API Tokens page, click the Create API token button. \n
    7. \n
  7. In the\u00a0Label field, enter a name for your API token. For example, you could enter \"KCM GRC Integration.\"
    8. \n
  8. Click the\u00a0Create button. When you click this button, a new window will display to provide your API token.
    9. \n
  9. Click the\u00a0Copy button to copy your API token. You'll need this token to configure the integration. \n
    10. \n
\n
After you copy your new Jira API token, continue to the Configuring the Integration section below to configure the integration in your KCM GRC platform.
\n
\n
Configuring the Integration
\n
After you copy your API token, you can configure the integration from your KCM GRC platform.
\n
To configure the integration, follow the steps below:
\n
    \n
  1. Log in to your KCM GRC platform.\u00a0
    2. \n
  2. At the top-right corner of the page, select Settings >\u00a0Account Settings.\u00a0\n
    3. \n
  3. From your Account Settings, select the Integrations subtab. \n
    4. \n
  4. In the\u00a0Atlassian Jira Software section of the page, click the Connect button. When you click this button, the\u00a0Atlassian Jira Software page will open. \n
    5. \n
  5. \n Fill out the fields on the\u00a0Atlassian Jira Software\u00a0page. For more information, see the screenshot and list below:\n
      \n
    1. \nInstance URL: Enter the full URL for your organization's Jira instance, including the protocol and the domain. For example, enter \"https://yourorganizationsdomain.atlassian.net\u201d.
      2. \n
    2. \nUsername: Enter the email address of the same Jira admin account that you used to create the API token.
      3. \n
    3. \nAPI Token: Enter the API token that the Jira admin created in their Atlassian account settings. For more information, see the Creating the Jira API Token section above.\n
      \n
      \nNote:After you enter the API token, make sure to click the Connect button.
      \n
      \n
      4. \n
    4. \nProject: Select the Jira project that you would like to send KCM GRC tasks to. This field affects the Issue Type, Issue Status, and Jira Assignee fields.\n
      \n
      \nNote: Any changes you make to the selected project will affect any task schedules that you create.\u00a0
      \n
      \n
      \n
      \nTip:If you don't see your project listed in the\u00a0Project drop-down menu, verify that the user entered into the Username field created the API token entered into the API Token field.\u00a0Then, verify that the user has the Browse Projects permission in Jira.
      \n
      \n
      5. \n
    5. \nIssue Type: Select the Jira issue type for the tasks that are sent to Jira from KCM GRC. For example, if you select Issue\u00a0in this field and then create a task schedule, the Issue Type field will default to\u00a0Issue for that task schedule.
      6. \n
    6. \nIssue Status: Select the Jira issue status that will prompt the task approval workflow to start. For example, if you select Done in this field, the Group Lead or Approving Manager will be prompted to approve the task when the User Assigned marks the task as Done in Jira.\n
      \n
      \nImportant:We recommend that you don't select the status that your Jira issues default to when they are created. For example, if your Jira issues default to the To Do status when you create them, don't select the To Do status in this field.
      \n
      \n
      7. \n
    7. \nJira Assignee: Select the default Jira user to assign tasks to, or select Unassigned. You can change the\u00a0Jira Assignee for individual task schedules when you create each task schedule.
      8. \n
    \n\n
    6. \n
  6. Click the\u00a0Save button.
    7. \n
\n
After you click the Save button, the configuration will be complete and you can begin to create task schedules for Jira. For more information about creating task schedules for Jira, see our Jira Integration: How to Work with Task Schedules article.
