SecurityCoach offers a range of reports that provide insight into your organization\u2019s security risks, detection rules, and real-time coaching campaigns. These reports can help you understand where you may need to improve your security awareness program. You can also use these reports to track trends in your users' risky activity over time.
\n
To view SecurityCoach reports in your KSAT console, navigate to SecurityCoach > Reports. To view a SecurityCoach: Behaviors and Coaching report, click the Report button for the specific report. To view a Risk Overview for Vendor Type report, click the name of the report.
\n
\n
\nNote:By default, each report displays data and activity from the last 90 days. However, you can adjust the report to reflect a different date range by clicking the drop-down menu at the top-right corner of the report.
\n
\n
In the SecurityCoach: Behaviors and Coaching section, you can view an overview of your real-time coaching campaign, an overview of your detection rules, or an overview of your vendor events. To learn more about each report type, see the sections below.
\n
Real-Time Coaching Report
\n
The Real-Time Coaching Report provides an overview of your real-time coaching campaigns.
\n
To learn more about the data in this overview, see below:
\n
\n
    \n
  1. \nReal-Time Coaching Overview: This section provides information about your real-time coaching campaigns.\n
      \n
    1. \nTotal Users Coached: This number displays the number of users that received SecurityTips during the selected date range.
      2. \n
    2. \nUsers Coached Multiple Times: This number displays the number of users that received more than one SecurityTip during the selected date range.
      3. \n
    3. \nNew Users Coached: This number displays the number of users that received a SecurityTip during the selected date range that had never received a SecurityTip before.
      4. \n
    4. \nSecurityTips Delivered: This number displays the total number of SecurityTips delivered to users during the selected date range.
      5. \n
    \n
    2. \n
  2. \nTop Real-Time Coaching Campaigns by SecurityTips Delivered: This chart displays your real-time coaching campaigns with the most detections of risky activity during the selected date range. This report will include up to ten real-time coaching campaigns.
    3. \n
  3. \nTop 10 Coached Groups: This table displays a list of your Smart Groups that received the most SecurityTips during the selected date range.
    4. \n
  4. \nSecurityTips Delivered Over Time: This graph displays the changes in the number of SecurityTips delivered during the selected date range.
    5. \n
  5. \nTop 10 Coached Users: This table displays a list of your users that received the most SecurityTips during the selected date range. This report will include up to ten users.
    6. \n
\n
Detection Rules Report
\n
The Detection Rules Report provides an overview of your active detection rules.
\n
To learn more about the data in this overview, see below:
\n
\n
    \n
  1. \nDetections Rule Overview: This section provides information about your detection rules.\n
      \n
    1. \nTotal Events: This number displays the total number of events during the selected date range.
      2. \n
    2. \nRule Detections: This number displays the number of events that triggered a detection rule during the selected date range.
      3. \n
    3. \nUsers Involved: This number displays the number of mapped users that triggered a detection rule during the selected date range.
      4. \n
    4. \nUsers with Multiple Rule Detections: This number displays the number of users that triggered two or more detection rules during the selected date range.
      5. \n
    \n
    2. \n
  2. \nTop Detection Rules: This graph displays the number of users that had detections for your organization\u2019s most detected rules during the selected date range. This graph will show up to ten detection rules.
    3. \n
  3. \nTop Users by Rule Detections: This table displays the number of rule detections that your organization\u2019s riskiest users had during the selected date range. This table will show up to ten users.
    4. \n
\n
Vendor Events Report
\n
The Vendor Events Report displays the events from your integrated vendors that are linked to detection rules. You can use this information to ensure that you have real-time coaching campaigns set up for your most common events. To include more events in your report, visit the Creating a Custom Detection Rule section of our Detection Rules Guide to create and link detection rules to the events.
\n
\n
\nNote:The Vendor Events Report includes only events from the last 30 days. If you had more than 10,000 events during that time, only the most recent 10,000 events will display.
\n
\n
To generate the report, select one or more vendors from the Vendor drop-down menu at the top-left corner of the report and then click Show Report. For more information about additional report filter options, see the screenshot and list below:
\n
\n
\n
    \n
  1. \nVendor: Use this filter to select one or more vendors to display event data for.
    2. \n
  2. \nDate Range: Use this filter to select the date range to display data for.
    3. \n
  3. \nDetection Rules Status: Use this filter to display only events that have associated detection rules, or only events that do not have any associated detection rules.
    4. \n
  4. \nUser Mapping Status: Use this filter to display either all events, only events that are mapped to a user, or only events that are not mapped to a user.
    5. \n
  5. \nThreat Category: Use this filter to select one or more vendor threat categories to display data for.

    \n
    \n
    \nNote:You must choose a vendor using the Vendor filter before you can select specific threat categories for that vendor.
    \n
    \n
    6. \n
  6. \nThreat Severity: Use this filter to select one or more vendor threat severities to display data for.

    \n
    \n
    \nNote:You must choose a vendor using the Vendor filter before you can select specific threat severities for that vendor.
    \n
    \n
    7. \n
  7. \nShow Report: Use this button to save your filters and generate the new report data.
    8. \n
  8. \nLive View: Click this toggle to view the latest events detected by your integrated vendors.
    9. \n
\n
\n
Once you generate the report, the Event Field Summary displays a count of events for your selected Event Field. You can view events by Vendor, Threat Type, Threat Category, Threat Severity, or Event Source.
\n
\n
Below the Event Field Summary, you can view a table that displays information about each event. Click the Add or Remove Columns button to customize which columns display in the table.
\n
\n
To learn more information about each event, select the arrow icon.
\n
\n
In the Risk Overview for Vendor Type section, you can view an overview of your organization's events related to endpoint security, email security, or web security. For more information about these reports, see the sections below.
\n
\n
\nNote:For a list of vendors by security type, see our Vendor Security Types article.
\n
\n
