\n
\nNote:As of April 2023, Microsoft no longer supports Exchange 2013. For more information, see the Exchange 2013 end of support roadmap article from Microsoft.
\n
\n
In this article, you'll learn how to whitelist by email header in Microsoft 365, Exchange 2016, and Exchange 2019. This method ensures your Phishing Security Tests (PSTs) bypass your spam filters and reach your users\u2019 inboxes. For more information on how to whitelist our training emails, see our Whitelist Training Notifications in Microsoft Defender for Office 365.
\n
\n
\nNote:We typically recommend whitelisting by IP address or hostname. However, if you use Exchange or Microsoft 365 without Defender for Office 365, you may need to whitelist by email headers to ensure PSTs are delivered. For more information about whitelisting best practices, see our Whitelisting Guide.
\n
\n
If you don't have a spam filter or if you use Microsoft Defender for Office 365, Microsoft requires that you use advanced delivery policies to ensure email delivery.\u00a0
\n
\n
\nImportant:Some whitelisting configurations may not be available in environments outside of the US.
\n
\n
Bypass Clutter and Spam Filtering
\n
To bypass clutter and spam filtering by email header, follow the steps below.
\n
    \n
  1. Log into your Microsoft 365 Exchange admin center. \n
    2. \n
  2. Navigate to Mail flow\u00a0> Rules.
    3. \n
  3. Click Add a rule > Create a new rule. \u00a0 \n
    4. \n
  4. On the Set rule conditions page, give the rule a name, such as \"Bypass Clutter and Spam Filtering by Email Header\".
    5. \n
  5. In the Apply this rule if fields, select The message headers... and includes any of these words. \n
    6. \n
  6. Click Enter text, then\u00a0enter the header name.\n
    \n
    \nNote:By default, the header for KnowBe4 mail is X-PHISHTEST. We recommend changing the default header to a custom header or header token for enhanced security. You can change the header settings for your account from the KSAT Account Settings page. For more information, see our Account Settings Guide.
    \n
    \n
    7. \n
  7. Click Save.\n
    8. \n
  8. Click Enter words and enter \"KnowBe4\".
    9. \n
  9. Click\u00a0Add, then click Save. \n
    10. \n
  10. In the\u00a0Do the following fields, select Modify the message properties and set the spam confidence level (SCL). \n
    11. \n
  11. In the specify SCL pop-up window, select\u00a0Bypass spam filtering. Then, click\u00a0Save.\u00a0 \n
    12. \n
  12. To the right of the\u00a0Do the following\u00a0fields, click\u00a0the plus icon. \n
    13. \n
  13. In the And fields, select Modify the message properties and\u00a0set a message header. \n
    14. \n
  14. Click the first\u00a0Enter text on the left and enter \"X-MS-Exchange-Organization-BypassClutter\".\u00a0
    15. \n
  15. Click\u00a0Save. \n
    16. \n
  16. Click the second Enter text on the right, then enter \"true\".
    17. \n
  17. Click\u00a0Save. \n
    18. \n
  18. Click\u00a0Next.\u00a0
    19. \n
  19. On the Set rule settings page, click Next. As a best practice, we recommend leaving these options at their default settings.
    20. \n
  20. On the\u00a0Review and finish\u00a0page, click\u00a0Finish. \n
    21. \n
\n
Bypass the Junk Folder in Microsoft 365
\n
To bypass your Junk folder in Microsoft 365, follow the steps below.
\n
    \n
  1. Log into your Microsoft 365 Exchange admin center.
    2. \n
  2. Navigate to Mail flow > Rules.
    3. \n
  3. Click\u00a0Add a rule > Create a new rule. \u00a0 \n
    4. \n
  4. Give the rule a name, such as \"KnowBe4 - Skip Junk Filtering\".
    5. \n
  5. In the Apply this rule if fields, select\u00a0The message headers... and\u00a0includes any of these words.\u00a0\n
    \n
    \nNote:Make sure that you add a condition for each header you need to whitelist. For more information on phishing email headers settings, see our Account Settings Guide.
    \n
    \n\n
    6. \n
  6. Click Enter text and enter the header name.\n
    \n
    \nNote:By default, the header for KnowBe4 mail is X-PHISHTEST. We recommend changing the default header to a custom header or header token for enhanced security. You can change the header settings for your account from the KnowBe4 console Account Settings page. For more information, see our Account Settings Guide.
    \n
    \n
    7. \n
  7. Click Save. \n
    8. \n
  8. Click Enter words\u00a0and enter \"KnowBe4\".
    9. \n
  9. Click Add, then click Save.\n
    10. \n
  10. In the\u00a0Do the following fields, select Modify the message properties and set the spam confidence level (SCL). \n
    11. \n
  11. In the specify SCL pop-up window, select\u00a0Bypass spam filtering. Then, click\u00a0Save. \n
    12. \n
  12. To the right of the Do the following fields, click the plus icon. \n
    13. \n
  13. In the And fields, select Modify the message properties and set a message header.\u00a0
    14. \n
  14. Click the first Enter text on the left and enter \"X-Forefront-Antispam-Report\". This value is case-sensitive.
    15. \n
  15. Click Save. \n
    16. \n
  16. Click the second Enter text on the right and enter \"SFV:SKI;CAT:NONE;\". To learn more about this header, see Microsoft's Anti-spam message headers in Microsoft 365 article. This field is case-sensitive.
    17. \n
  17. Click Save.\n
    18. \n
  18. Click\u00a0Next.\u00a0\u00a0
    19. \n
  19. On the\u00a0Set rule settings\u00a0page, click\u00a0Next.\u00a0As a best practice, we recommend leaving the other options at their default settings.
    20. \n
  20. On the Review and finish page, click Finish. \n
    21. \n
  21. Change the priority of the rule to directly follow the rule you created in the previous section. For more information, see Microsoft\u2019s Set the priority of a mail flow rule article.
    22. \n
\n
