You can integrate your XM Cyber Security Control Monitoring (SCM) module into your KSAT console. XM Cyber\u2019s SCM module provides you with insights into the configuration and activation of security controls.
\n
Before You Connect XM Cyber\u2019s SCM to Your KSAT Console\u00a0
\n
Before you integrate SCM into your KSAT console, follow the steps below.
\n
    \n
  1. Log in to your KSAT console as an admin.
    2. \n
  2. Go to Account Settings.
    3. \n
  3. Navigate to the Account Integrations option in the sidebar.
    4. \n
  4. Click the API option.
    5. \n
  5. Select the Enable Reporting API Access toggle.
    6. \n
  6. Click the Reporting API link. \n
    7. \n
  7. Click the Create New API Token button.
    8. \n
  8. The console will create an API key. Please save this API key for the next steps.\u00a0
    9. \n
\n
\n
\nNote:Make sure that you click Save Changes to save your Account Settings updates.
\n
\n
Connecting Cyber XM\u2019s SCM to Your KSAT Console
\n
To connect SCM with your KSAT console, follow the steps below.
\n
    \n
  1. Log in to Cyber XM\u2019s SCM module as an admin.
    2. \n
  2. In the left menu bar, navigate to System > Tools Configuration.
    3. \n
  3. Select Add Tool.
    4. \n
  4. In the Add Tool dialog box, select SSPM, and then complete the fields as follows:
    \n
      \n
    1. \nVendor: Select KnowBe4.
      2. \n
    2. \nType: Select Email protect.
      3. \n
    3. \nProduct Name: Select the Product Name, which is KnowBe4 KMSAT.
      4. \n
    4. \nVersion: Select the product version, which is not applicable.
      5. \n
    5. \nMachine: For a SaaS environment, select the cloud proxy server that XM Cyber gives you. For an on-prem environment, select your cloud proxy server.
      6. \n
    6. \nName: You may use the default name or change the name if desired.
      7. \n
    \n
    5. \n
  5. Select Save. The tool should now appear on the tool list.
    6. \n
  6. Select the tool from the tool list.
    7. \n
  7. Under the General tab, check the System Active check box and select Apply Changes. \n
    8. \n
  8. Navigate to the Datasource tab and select Add DB Model.
    9. \n
  9. Enter this information in the following fields: 
    \n
      \n
    • \nDatabase Type: PostgreSQL
      • \n
    • \nServer: 127.0.0.1
      • \n
    • \nPort: 5432
      • \n
    • \nDatabase: You can use any database name you like as long as it contains only lowercase letters. For example, you can use \u201cknowbe4_kmsat\u201d as your database name.
      • \n
    • \nUser: cyberobserverds
      • \n
    • \nPassword: C0DBLocal! \n
      • \n
    \n
    10. \n
  10. Select the Create Database check box and select Save. You should receive a message that you successfully created a database. \n
    11. \n
  11. To activate plugins, navigate to the Plugins tab and select run.
    12. \n
  12. In each plugin row, select Set Parameters and enter the following information in the fields:\n
      \n
    • \nHost: General URL
      • \n
    • \nEmail: user\u2019s email
      • \n
    • \nPassword: user password
      • \n
    • \nBase URL: General URL
      • \n
    • \nToken: The API token created previously in this article.
      • \n
    \n
    13. \n
  13. In the Plugins tab, select Save. You should receive a message that you have successfully installed the plugin. \n
    14. \n
  14. Select rerun to run this plugin, connect the tool, and collect data. The entry under Last Successful Run will show the date and time the plugin was last run.
    15. \n
\n
