Our ADI Sync Tool is written in C#, which uses environment variables to access proxy information. This means you must first add the appropriate HTTPS_PROXY variable before you can use Active Directory through a proxy.\u00a0
\n
Follow the steps below to add the necessary environment variable:
\n
    \n
  1. From your system\u2019s Control Panel, right-click This PC.
    2. \n
  2. From the drop-down menu that opens, select Properties. \n
    3. \n
  3. From the screen that opens, click on Change settings. \n
    4. \n
  4. In the window that opens, click on the Advanced tab.\u00a0
    5. \n
  5. Click on Environment Variables at the bottom of the window. \n
    6. \n
  6. Under System Variables, click New. \n
    7. \n
  7. Add a system variable named HTTPS_PROXY and set the value to specify your IP address and port number in the following format: http://[IPaddress:port] For example: http://10.1.20.102:3128/
    \n
    \n
    \nNote:  If your proxy server requires a username and password, include the credentials in this syntax: https_proxy = http://username:password@proxy.example.com:8080\n
    \n
    \n
    8. \n
  8. Click OK.
    9. \n
\n
For more information, see this article from Cloud Foundry.\u00a0
