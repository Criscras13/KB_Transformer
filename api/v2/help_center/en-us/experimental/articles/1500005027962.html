\nNote: Due to upcoming changes from Google, we recommend using Direct Message Injection (DMI) as your default whitelisting configuration. For more information about these changes, see Google's Gmail updates.
\n
You can bypass Google Workspace's spam filters by applying a content compliance rule to our IP addresses. This rule will allow our simulated phishing emails to be delivered to your user's inboxes. The Content Compliance whitelisting method involves two steps in Google Workspace: adding KnowBe4's IP addresses to the Email Whitelist, and enabling whitelisting by Content Compliance.
\n
Content compliance is an alternative method to whitelisting by IP address in our Whitelisting by IP Address in Google Workspace article. We recommend using the content compliance method if you have an existing inbound gateway to use the content compliance rule or if you are unable to configure the inbound gateway.
\n
\nNote:This article reflects current best practices for whitelisting with your provider. Please be aware that your mail service provider may make changes to how their systems analyze our emails at any time. If you are experiencing issues whitelisting using the procedure below, contact our support team.\nThis method of whitelisting is a two-part process.\n
\n\n
Add KnowBe4's IP Addresses to Your Google Workspace Whitelist
\n
First, you\u2019ll need to add KnowBe4\u2019s IP addresses to your IP allow list for Google Workspace. Once you add KnowBe4\u2019s IP addresses to your IP allow list, you can enable whitelisting by content compliance.
\n
To add KnowBe4\u2019s IP addresses to your Google Workspace whitelist, follow the steps below:
\n
    \n
  1. Log in to the Google Admin portal and click Apps. \n\n
    2. \n
  2. Click Google Workspace. \n\n
    3. \n
  3. Click Gmail. \n\n
    4. \n
  4. Click Spam, Phishing and Malware. \n\n
    5. \n
  5. Under the Organizational Unit section, highlight your domain. Do not select a sub-organizational unit (OU).\n
    \nNote: Google Workspace does not allow whitelisting by IP address for individual OUs. You can only whitelist by the entire domain.
    \n\n\n
    6. \n
\n
    \n
  1. In the Email whitelist section, enter our IP addresses. Make sure to separate each IP address with a comma. For a current list of our IP addresses, see the IP Addresses, Hostnames, and Header Information section of our Whitelisting Guide article. \n\n
    2. \n
  2. Click Save.
    3. \n
\n\n\n
Enable Whitelisting by Content Compliance
\n
After you have added KnowBe4\u2019s IP addresses to your Google Workspace whitelist, you can enable whitelisting by content compliance.
\n
\nNote:The instructions provided were gathered from Google\u2019s Set up rules for advanced email content filtering article.
\n
To enable whitelisting by content compliance, follow the steps below:
\n
    \n
  1. Log in to your Google Admin portal and select Apps.
    2. \n
  2. Select Google Workspace.
    3. \n
  3. In the Showing status for apps in all organizational units area, click Gmail.
    4. \n
  4. In the Gmail area, click Compliance.
    5. \n
  5. Navigate to the Content compliance section.
    6. \n
  6. Click Add a Rule.\n
    \nNote:If you have previously created a content compliance rule, this option will be called Add Another Rule.
    \n\n\n
    7. \n
\n
After you click Add a Rule or Add Another Rule, configure your content Content compliance by following the steps below:
\n
    \n
  1. In the Email Messages to Affect area, select the Inbound check box.
    2. \n
  2. In the If ANY of the following match the message area, create an expression with the following settings:\n
      \n
    1. From the first drop-down menu, select Metadata match.
      2. \n
    2. From the Attribute drown-down menu, select Source IP.
      3. \n
    3. From the Match type drop-down menu, select Source IP is within the following range.
      4. \n
    4. In the Source IP is within the following range field, enter one of KnowBe4's IP addresses. For the most up-to-date list of our IP addresses, see the IP Addresses, Hostnames, and Header Information section of our Whitelisting Data and Anti-Spam Filtering Information article.
      5. \n
    5. Click Save.
      6. \n
    \n
    3. \n
  3. Repeat the three steps above for each of our IP addresses.
    4. \n
  4. In the If ANY of the following match the message section, add another expression with the settings in the screenshot and list below: \n
      \n
    1. From the first drop-down menu, select Advanced content match.
      2. \n
    2. From the Location drop-down menu, select Full headers.
      3. \n
    3. From the Match type drop-down menu, select Contains text.
      4. \n
    4. In the Content field, enter \"X-PHISHTEST\".
      5. \n
    5. Click Save.
      6. \n
    \n\n
    5. \n
  5. In the If the above expressions match, do the following area, select the check boxes in the screenshot and list below:\n
      \n
    1. Under Spam, select the Bypass spam filter for this message check box.
      2. \n
    2. Under Encryption, select the Require secure transport (TLS) check box.
      3. \n
    \n\n
    6. \n
\n
\nNote:If your organization uses Google Workspace, you\u2019ll also need to disable the return-path header in your KSAT Account Settings before sending phishing tests. For more information, see our How to Change the Return-Path Header in Your Account Settings.
\n
\nTip: To ensure that you have whitelisted correctly, see our How to Verify You Have Whitelisted KnowBe4 Correctly article.
