KnowBe4\u2019s Reporting application programming interface (API) feature, limited to Platinum and Diamond customers, allows you to pull data from the KSAT console for reporting purposes. Currently, this feature is limited to allow requests for phishing, training, user, and group data. When using this API, you must authenticate to the API using an account-specific key.
\n
For full details about the Reporting API, as well as sample requests, visit our Reporting API developer documentation. KnowBe4\u2019s API is a REST API. By using KnowBe4's API, you agree to our Terms of Use.
\n
Creating a Reporting API Token
\n
To create a Reporting API token, follow the steps below:
\n
    \n
  1. From your KSAT console, click your email address at the top-right corner of the page and select Account Settings.
    2. \n
  2. Navigate to Account Integrations > API.
    3. \n
  3. Under Reporting API, select the Enable Report API Access toggle.\n
      
    \n
    4. \n
  4. Click Reporting API. The Reporting API subtab will open.
    5. \n
  5. At the top-right corner of the Reporting API subtab, click + Create New API Token.\n
    \n
    6. \n
  6. Fill out the fields on the Create New API Token page. For more information, see the screenshot and list below:\n
      \n
    1. \nName: In this field, enter a descriptive name for this API token.
      2. \n
    2. \nStatus: Enable or disable the API token.
      3. \n
    \n
    \n
    7. \n
  7. Click Create Token. The Reporting API Token pop-up window will open.
    8. \n
  8. Click the Reporting API token to copy the token. Make sure to save your copied API token somewhere that you can easily access. After you close this window, you will not be able to view this token again.\n
    \n
    9. \n
  9. Click OK.
    10. \n
\n
Once copied, you can use the API token to pull data from the KSAT console for reporting purposes.
\n
Managing Reporting API Tokens
\n
Once you have created a Reporting API token, you can manage the token on the Reporting API subtab. You can access this subtab from the Account Integrations section of your Account Settings page.
\n
\n
\nNote: You can view only the last 10 digits of your Reporting API tokens after they have been created. If you lose a token, you\u2019ll need to create a new one.
\n
\n
To learn more about the options available on this subtab, see below:
\n
\n
    \n
  1. \nKey: This column displays a unique identifier used to authenticate an API request associated with your KSAT console
    2. \n
  2. \nName: This column displays the custom name you assigned to your API token when it was created.
    3. \n
  3. \nGenerated By: This column displays the email address associated with the user that generated the API token.
    4. \n
  4. \nGenerated On: This column displays the date the Reporting API token was generated.
    5. \n
  5. \nStatus: This column displays the API token\u2019s status. The status can be enabled or disabled.
    6. \n
  6. \nActions: Click the drop-down arrow in the rightmost column to view the available actions for a Reporting API token. These actions include:\n
      \n
    • \nEdit: Click Edit to open the Edit Reporting API Token page, which is similar to the Create New API Token page. Edit options as needed, then click Update Token to save your changes.
      • \n
    • \nDelete: Click Delete to permanently delete a Reporting API token.
      • \n
    \n
    7. \n
