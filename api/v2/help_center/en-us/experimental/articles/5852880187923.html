If you would like to test whether your users will enter personal information onto a custom webpage, you can create your own data entry landing page. You can also create a secondary landing page that will display if your users submit information on the first data entry landing page. Then, you can add your landing pages to a phishing template.\u00a0
\n
For general information about data entry landing pages, see our How to Use Data Entry Landing Pages article.\u00a0
\n
Creating a Data Entry Form
\n
To create a data entry form, follow the steps below:
\n
    \n
  1. Log in to your KnowBe4 account and navigate to Phishing > Landing Pages.
    2. \n
  2. Click the + Create Landing Page button to add a new landing page. Or, if you would like to edit an existing landing page, click the name of the landing page.
    3. \n
  3. In the top-left corner of the What You See Is What You Get (WYSIWYG) editor, click the Source button. When you click this button, you will be able to enter HTML or CSS to create your data entry form.
    4. \n
  4. Create your data entry form. For an example of a data entry form, see the code below:\n
    \n  <form action='https://www.example.com' id='loginform' method='post' name='loginform'><input\n  name='username' placeholder='Email Address' required='' type='text' /><br\n  />\n  <input id='password' name='password' placeholder='Password' required='' type='password'\n  /><br />\n  <input type='submit' value='Login' />\n  </form>
    \n
    \n
    \nImportant:We don't record any data that your users enter onto the landing page. No data is stored on our servers, including logs, as long as the form field names follow strict naming conventions. For the form field names, you must use one of the following names: password, password_confirmation, old_password, credit_card, ssn, social_security_number, domain_name, uname, number, verification_value, or brand.
    \n
    \n
    5. \n
  5. To ensure all submitted data is obfuscated correctly, add the following script to the source code:\n
    \n  <script>\n  jQuery(function($) {\n\n  $('#loginform').submit(function() {\n  var txt = $('#password');\n  txt.val(\"*******\");\n  });\n\n  function display(msg) {\n  $('<p/>').html(msg).appendTo(document.body);\n  }\n  });\n  </script>
    \n
    \n
    \nImportant:For the script to work properly, the name and ID for the login form and password input field must match the ID used in the script function. In the code examples above, the name and ID for the login form are highlighted in red. The name and ID for the password input field are highlighted in blue.
    \n
    \n
    6. \n
  6. After you have created your form, click the Save button.
    7. \n
\n
\n
\nNote:After you click the Save button, the landing page will be added to your Drafts category. To use the landing page in a phishing campaign, you must move the landing page from your Drafts category to a separate category. For more information, see our How to Create and Edit Email Templates and Landing Pages article.
\n
\n
Adding a Secondary Landing Page
\n
When you create your data entry landing page, you can also add a secondary landing page. Your users will see this page if they enter and submit data on the data entry landing page. The secondary landing page can be any landing page in your account. For example, you could use another landing page that you\u2019ve created in your account or one of our pre-made landing pages.
\n
To add a secondary landing page to the data entry landing page that you\u2019ve created, follow the steps below:
\n
    \n
  1. Repeat steps 1-3 in the Creating a Data Entry Form section above.
    2. \n
  2. Create your data entry form. In the action tag, enter the URL for the secondary landing page. For an example of where to include the URL for the secondary landing page, see the example code below:\n
    \n  <form action='https://www.example.com\n  <input id='password' name='password' placeholder='Password' required='' type='password'\n  /><br />\n  <input type='submit' value='Login' />\n  </form>
    \n
    3. \n
  3. Click the Save button.
    4. \n
\n
Adding the Landing Page to a Phishing Template
\n
Once you have created your data entry landing page, you can add the landing page to a phishing template.
\n
To add your data entry landing page to a phishing template, follow the steps below:
\n
    \n
  1. Log in to your KnowBe4 account and navigate to Phishing > Email Templates.
    2. \n
  2. Click the name of the phishing template that you would like to add the data entry landing page to.\n
    \n
    \nImportant:To use your data entry landing page, you must select a phishing template that includes at least one link in the body of the email. Phishing templates that include a link will have a (Link) tag in the template name.
    \n
    \n
    3. \n
  3. At the bottom of the phishing template, select your landing page from the Landing Page drop-down menu.
    4. \n
  4. Click the Save button.
    5. \n
\n
\n
After you add your landing page to the phishing template, you can add the phishing template to a phishing campaign. You can use this campaign to test your users and see how likely they are to enter data onto a landing page.
\n
\n
\nImportant:We always recommend that you assign a test campaign to yourself or a limited number of users before you assign the campaign to a large group of users. Test campaigns allow you to ensure that your campaign is working correctly.
\n
