In this article, you will learn how to remove failures from and add user reports to your users\u2019 phishing test results.
\n
The following types of phishing failures can be removed:
\n
    \n
  • Clicks
    • \n
  • Attachment opens
    • \n
  • Replies
    • \n
  • Reply content
    • \n
  • Macros enabled
    • \n
  • Data entry
    • \n
  • QR code scans
    • \n
  • Calling the number and entering the callback code in a callback phishing email
    • \n
  • Entering personal information in a callback phishing call\n
    \n
    \nImportant: We don't record any data that your users enter in a callback phishing call.
    \n
    \n
    • \n
\n
Removing failures cannot be reversed. After a failure is removed, your KSAT account may take up to 15 minutes to update your reports.
\n
Email opens are not considered a failure on a phishing test and do not contribute to a user's Phish-prone Percentage.
\n
There are two methods you can use to remove phishing test failures. You can either remove failures from the phishing campaign's report or from the affected user's timeline.
\n
\n
\nImportant:Within your phishing campaign settings, you can choose to automatically add users who fail the phishing test to a group. If you chose this option when you created the campaign and would like to remove a failure from this campaign, removing the failure will not remove the failing user from the group. You can manually remove the user if you would like. However, if you are using a Smart Group to group your users, the group will update to reflect the failure removal.
\n
\n
Remove a Failure from the Phishing Campaign Report
\n
You can remove phishing test failures from a phishing campaign report individually or in bulk.
\n
    \n
  1. Log in to your KSAT account and navigate to Phishing > Campaigns.\u00a0
    2. \n
  2. From the list of phishing campaigns, click the campaign that you would like to remove a failure from. \n
    3. \n
  3. Select the Users subtab.
    4. \n
  4. Click on the column header for the type of failure you want to remove. For example, if you would like to remove a QR code failure from a phishing test, click the QR Code Scanned column. \n
    5. \n
  5. Once you've clicked the column header, you can remove a failure for a single user or remove failures for multiple users.\n
      \n
    • To remove failures for multiple users at once, click Bulk Update. You will be asked to upload a CSV file that lists those specific users. The CSV file must contain the\u00a0Email\u00a0and\u00a0Update-Value headers. You have the option to send a notification to console administrators once the bulk failure removal is complete. Once you have selected your CSV file and configured your notification settings, click\u00a0Bulk Update. Removing a phishing failure is not reversible. You will be prompted with a deletion confirmation before the failure is removed. \n
      • \n
    • To remove a failure for one user, find the user's name in the user list and click the trash can icon to the right of their name.\n
      \n
      \nNote: For reply-to phishing failures, you can delete the content of the user's reply without deleting the failure. To delete the content of the reply click the red arrow to view the reply content, then click Delete Reply Content.
      \n
      \n\u00a0 \n
      • \n
    \n
    6. \n
  6. When prompted to confirm your action, click either Confirm or Cancel. Remember that removing a failure cannot be reversed.After you click Confirm, the failure is removed. All failure removals will be recorded on the associated user profiles within the user's timeline. \n
    7. \n
\n
Back to top
\n
Remove a Failure from a User's Timeline
\n
You can remove phishing test failures from an individual user's timeline. To use this method, follow the steps below:
\n
    \n
  1. Log in to your KSAT account and navigate to the Users tab.\u00a0
    2. \n
  2. Select the Users\u00a0subtab.
    3. \n
  3. From the list on the Users subtab, find the affected user.\u00a0
    4. \n
  4. Click the user's name or email address to go to their\u00a0User Profile.
    5. \n
  5. Click on their\u00a0User Timeline tab.\u00a0 \n
    6. \n
  6. Find the phishing failure you would like to remove.\n
    \n
    \nImportant:We recommend using the Event Type or\u00a0Date Range drop-down menus to quickly find the event you are looking for.
    \n
    \n
    7. \n
  7. Once you have found the failure you want to remove, click the trash can icon next to the failure. \n
    8. \n
  8. When prompted to confirm your action, click either Confirm or Cancel. Remember that removing a failure cannot be reversed.After you click Confirm, the failure is removed and the removal is recorded on the user's timeline.\n
    9. \n
\n
Back to top
\n
Add User Reports to the Phishing Campaign Report
\n
You can also add user reports in bulk using a CSV file to update Phishing Security Test (PST) reporting. You can set up your CSV file as follows:
\n
    \n
  1. Set up the first line of your CSV file as headers as follows: 
    Email, Update-Value
    2. \n
  2. Set up the second line of your CSV file as follows: 
    Name@domain.com, add-report
    3. \n
  3. In the following lines, add the email addresses of the users, followed by the add report value using the format shown in step 2.
    4. \n
\n
To upload the CSV file to perform bulk PST user reporting, follow the below steps:
\n
    \n
  1. Navigate to the Phishing tab.
    2. \n
  2. Navigate to the Campaigns subtab and select the appropriate Phishing Security Test.
    3. \n
  3. In the Phishing Security Test, select the Users subtab to the right of the Overview subtab.
    4. \n
  4. At the right of the Users page, select Bulk Update.
    5. \n
  5. Upload the CSV file.
    6. \n
\n
Back to top
