\n
\nNote:We\u2019ve added advanced features to phishing templates! All accounts will be opted in to phishing template advanced features in 2025. For more information, see our Phishing Templates Advanced Features Guide.
\n
\n
In your KnowBe4 console, you can create and edit phishing templates and landing pages from the Phishing tab. You can then use these templates and landing pages in phishing campaigns to test your users. To learn more about creating phishing templates and landing pages, read the sections below or watch our Customization of Phishing Templates and Landing Pages video.
\n
Creating and Editing Phishing Templates
\n
In your KnowBe4 console, you can create and edit phishing templates. To create or edit a template, follow the steps below:
\n
    \n
  1. In your KnowBe4 console, navigate to Phishing > Phishing Templates > My Templates.
    2. \n
  2. To create a new phishing template, click the +Create Template button. To edit an existing template, click the name of that template.
    3. \n
\n
When you click the +Create Template button or the name of an existing template, the template editor will open. From here, you can create or edit the template. See the images below for more information on how you can customize your phishing template.
\n
\n
    \n
  1. \nLanguage (Optional): From the drop-down menu, you can select the language for your template. While this is an optional step, templates with a language selected are available for use in localized phishing campaigns.\n
    \n
    \nNote: When copying a system or community template, this field will populate with the language that template uses.
    \n
    \n
    2. \n
  2. \nTemplate Name: Enter a name for your template in this field. The name you choose will not be visible to your users.
    3. \n
  3. \nSender's Email Address: Enter an email address to show who sent the email. You can choose any email address you would like, and users will see this email address in the email.
    4. \n
  4. \nSender's Name (Optional): Enter a sender name to show who sent the email. You can choose any sender name you would like, and users will see this sender's name in the email.
    5. \n
  5. \nReply-To Email Address (Optional): Enter an email address you would like your users to reply to when replying to a simulated phishing template. User replies will be sent to this email address unless reply-to phishing is enabled. When using reply-to phishing, this email address will be overwritten by our system-generated email address so that we can capture replies. For more information on reply-to phishing, see our Reply-To Phishing article.
    6. \n
  6. \nReply-To Name (Optional): Enter a name you would like to have users see when replying to a simulated phishing test. When a user replies to this email, the To field will automatically populate with the name you entered in this field.
    7. \n
  7. \nSubject: Enter the subject line that your users will see in their inboxes. This field will be used as the template\u2019s name if you do not enter a template name.
    8. \n
  8. \nAttachment File Name (Optional): Enter a name for any attachments you would like to use in your email. The name can include placeholders.
    9. \n
  9. \nAttachment Type (Optional): Select the attachment type you would like to use from the drop-down menu. \n
    10. \n
  10. \nSource: Click the Source button to switch to a source code view. If you're familiar with HTML, you can use this view to create the template.
    11. \n
  11. \nImage: Click the image icon to open an image editor. You can only use externally hosted images, which means that you must insert a publicly available URL into the URL field after clicking the button. You can also add alternative text here for accessibility.\n
    \n
    \nImportant: When resizing images, always edit the width and height in the image's style tag. For more information on image resizing and why we only allow publicly available URLs, see the Troubleshooting Template Image Issues article.
    \n
    \n
    12. \n
  12. \nPlaceholder: Placeholders automatically populate the template with specific information. For placeholders to work properly, you must add user information such as user and manager names to your KnowBe4 console ahead of time. For more information on placeholders, see our Placeholders Guide article.
    13. \n
  13. \nPhish Link: Add a phishing link to the email body. For steps on how to create a phishing link, follow these steps:\n
      \n
    • In the text field, enter the text you want to make into a phishing link. Then, highlight the text and click the Phish Link button to turn it into a phishing link. If you create an HTML link in a phishing template, our system will automatically change the link to a phishing link for security purposes. \n
      • \n
    \n
    14. \n
  14. \nQR Code: Click the QR Code button to add a QR code to your phishing template. For more information on QR codes, see our QR Code Phishing Security Tests (PSTs) Guide article.
    15. \n
  15. \nRed Flag: Platinum and Diamond customers can use our Red Flag button to explain what the social engineering indicators were in the phishing template. For more details on how to add red flags and recommended practices, see our Social Engineering Indicators (SEI) article.
    16. \n
  16. \nBody: Enter the text you want to display in your phishing template. The What You See Is What You Get (WSYWIG) editor provides options for customizing your phishing template. You can add tables, lines, and more by using the buttons available in the WYSIWYG editor.
    17. \n
  17. \nLanding Page: Select a landing page from the drop-down menu. This page is where your users will be redirected if they fail a phishing test. You can choose from our system landing pages or your custom landing pages.\u00a0\n
    \n
    \nImportant:If you select a landing page when creating a phishing campaign, it will override the template you created/edited and the default landing page.
    \n
    \n
    18. \n
  18. \nLanding Domain: Select a web address from the drop-down menu that the user will see on the landing page. Changing the landing domain can be helpful if you are also using one of our Data Entry landing pages. You can set a default landing domain on your Account Settings page, which will apply to all email templates you create.
    19. \n
  19. \nDifficulty Rating: You can assign a difficulty rating to your custom phishing templates. The difficulty rating will show how difficult that particular phishing template will be for your users to identify. For more information, see our Template Difficulty Ratings Overview article.
    20. \n
\n
Your new template will be saved in the My Templates section.
\n
Previewing and Testing Templates
\n
After you have created a template, you may find it useful to preview the template so that you can see what it will look like for your users.
\n
To preview a template, navigate to Phishing > Phishing Templates > My Templates and click the Preview icon under the Actions column for that template.
\n
While previewing a template, you can also use the Send Me a Test Email button to send a copy of the template to your inbox as an email. This feature is useful when creating your own phishing templates since it will allow you to see what the email will look like before sending it to your users in a phishing campaign.
\n
We do not recommend using the Send Me a Test Email button to test your whitelisting. The email that is sent using this button will look like a phishing test email but will not have the same functionality as an email sent from a phishing campaign. Instead, we recommend setting up a preliminary test campaign to ensure the most accurate testing results.
\n
Creating and Editing Landing Pages
\n
In your KnowBe4 console, you can create and edit landing pages. To create or edit a landing page, follow the steps below:
\n
    \n
  1. In your KnowBe4 console, navigate to the Phishing tab.
    2. \n
  2. Select the Landing Pages subtab.
    3. \n
  3. To create a new landing page, click the +Create Landing Page button. To edit an existing landing page, click the name of that landing page.
    4. \n
\n
When you click the +Create Landing Page button or the name of an existing landing page, the landing page editor will open. From here, you can create or edit the landing page. See the images below for more information on how you can customize your landing page.
\n
\n
    \n
  1. \nTitle: Enter a title for your landing page here. This title will display in the My Landing Pages section of the Landing Pages subtab and in the Landing Pages drop-down menu when you create a phishing campaign.
    2. \n
  2. \nSource: Click the Source button to switch to a source code view. If you're familiar with HTML, you can use this view to create the landing page.
    3. \n
  3. \nImage: Click the image icon to open the image editor. You must use externally hosted images, which means that you must insert a publicly available URL into the URL field after clicking the button. You can also add alternative text here for accessibility.\n
    \n
    \nImportant: When resizing images, always edit the width and height in the image's style tag. For more information on image resizing and why we only allow publicly available URLs, see the Help! The Images in My Phishing Template Are Not Showing Up Correctly article.
    \n
    \n
    4. \n
  4. \nPlaceholder: Placeholders automatically populate the template with specific information. For placeholders to work properly, you must add user information such as user and manager names to your KnowBe4 console ahead of time. For more information on placeholders, see our How To Use Placeholders article.
    5. \n
  5. \nSEI Placeholders: The Social Engineering Indicators (SEI) feature is available to Platinum and Diamond subscription levels. It allows you to show your users exactly what SEIs, or red flags, they overlooked when they clicked a simulated phishing link. For more information, see our Social Engineering Indicators (SEI) article.
    6. \n
  6. \nBody: Enter the text you want to display on your landing page. The What You See Is What You Get (WSYWIG) editor provides options for customizing your landing page. You can add tables, lines, and more by using the buttons available in the WYSIWYG editor.
    7. \n
\n
\n
\nNote:You may see <a href='#'> in the landing pages' code. This is expected behavior within all landing pages. This syntax indicates where a link would typically be placed, but it is inactive to keep users away from actual websites or possible malicious sites during phishing tests.
\n
\n
Limitations on Uploading Your Own Images
\n
Out of our respect for your intellectual property rights, we take steps to minimize the need for you to reproduce your copyrighted works to display them in your templates. Instead of embedding your images or uploading your images for us to host, using the linking method serves two mutually beneficial purposes. First, we provide a means for you to make more realistic templates that may help preserve the integrity and proportions of the images that you add to your templates. Second, linking your images decreases the amount of content you have to share with us to achieve the same outcome as uploading your images to the console.
