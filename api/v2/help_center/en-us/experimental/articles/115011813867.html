\n
Second Chance is a tool for Outlook and Microsoft 365 that gives your users a warning message before they visit a potentially unsafe or unknown website. Users who click URLs within emails, Microsoft 365 files, or PDF files will receive an alert from Second Chance. Then, the users can select whether they would like to open the link or file.
\n
\n
Second Chance can be installed locally or deployed through your Group Policy (GPO). You can also mark specific URLs as safe to open so that the URLs won\u2019t trigger Second Chance alerts.
\n
\n
\nImportant: Second Chance is not an endpoint protection tool and doesn\u2019t block users from visiting websites. Second Chance helps with user behavior monitoring and security awareness training. If a user clicks a link in an email, a Microsoft 365 file, or a PDF file, the alert they receive will give them a second chance to rethink before they navigate to a website that may be dangerous.
\n

    • Uses Microsoft Outlook 2013 or later.

Prompt Settings
\n
In the Prompt Settings section, you can change the content of your prompts and modify additional prompt settings.
\n
Before installing Second Chance, we recommend that you consider the content you would like to include in the prompt messages that appear when your users click links from emails, Microsoft 365 files, or PDF files.
\n
\n
\nTip:If you're not ready to complete this step before installing Second Chance, you can update the prompt content at any time. Then, the next prompt your users see will match your updated settings.
\n
\n
Second Chance will trigger four different types of prompts. For more information, see the list below:
\n
    \n
  1. \nGeneral Prompts: For general URLs that have not been added to the No Prompt Domain list. For more information, see the General Prompt Text section below.
    2. \n
  2. \nPunycode Prompts: For Punycode links. For more information, see the Punycode Prompt Text section below.
    3. \n
  3. \nAttachment Prompts: For links clicked within Microsoft 365 files or PDF files. For more information, see the Attachment Prompt Text section below.
    4. \n
  4. \nURL Unwinding Prompts: For rewritten or shortened URLs. For more information, see the URL Unwinding Prompt section below.
    5. \n
\n
To edit the content of the prompts listed above, click the Second Chance tab on the top bar, then click the Settings tab.
\n

\n
In the General Prompt section, you can modify the text and buttons that appear in all of your Second Chance prompts. You can modify the text in the Message Banner, Message Prompt, Continue button, and Abort button.

  4. \nContinue Button: This button allows the user to proceed with their action and open the link they clicked. The character limit for this banner is 40 characters.

What is Punycode?
\n
By using Punycode, cybercriminals can imitate the domains of popular websites. Punycode allows characters in a URL to display like regular characters while the computer reads the characters differently. For example, the link your user is clicking may display https://knowbe4.com, but the link may lead to https://xn--80ak6aa92e.com/. For more information, see Wikipedia\u2019s Punycode webpage.

Additional Prompt Settings
\n
In the Additional Prompt Settings section, you can modify the settings for additional prompts that display for specific types of links. These settings are enabled by default.
\n
For more information about these settings, see the list below:
\n
    \n
  • \nDo Not Prompt on Internal Links: You can use this setting to enable or disable prompts on links that appear to be navigating to your local or internal network and are not routable to another network.
    • \n
  • \nDo Not Prompt on KnowBe4 Phishing and Training Links: This setting allows you to enable or disable prompts for KnowBe4's phish domains. When you enable this setting, your users will not see a Second Chance prompt when they click a link in a KnowBe4 simulated phishing email or training notification.\n
    \n
    \nNote:If you select the Use KnowBe4's 'No Prompt Domains' List check box, your users will not receive prompts when they click links in KnowBe4's simulated phishing and training notification emails.
    \n
    \n
    • \n
  • \nUse KnowBe4's \"No Prompt Domains\" List: You can use this setting to enable or disable prompts for domains listed in your No Prompt Domain list.\n
    \n
    \nNote:By default, if you select this check box, prompts for links in KnowBe4 simulated phishing and training notification emails will also be disabled.

No Prompt Domains
\n
From the No Prompt Domains section of the Settings subtab, you can specify URLs that your users won\u2019t receive Second Chance prompts for visiting. From this section, you can see a list of the No Prompt Domains you've already added. If your users visit URLs with ar No Prompt Domain, the users will not be prompted by Second Chance.
\n
No Prompt Domains can be websites that your users need to access frequently or websites that you allow your organization to use. You can add up to 1,000 domains.
\n
\n
\nNote:Adding a website to the No Prompt Domains list will also block prompts for all subdomains under the website\u2019s domain. For example, if you added the website knowbe4.com to the list, the subdomain training.knowbe4.com would be blocked automatically. If a URL containing a subdomain is added to the list, only URLs that contain both the subdomain and the primary domain will not receive prompts. For example, if you added training.knowbe4.com to the list, only URLs that contain training.knowbe4.com will be blocked.
\n
\n
By clicking the blue Import link, you can open up a modal that allows you to enter domains in two different ways. For more information, see the list below:
\n
\n
    \n
  1. \nIndividual Import: You can use this method to enter an individual domain to your No Prompt Domains list. Domains must be entered in the proper domain format. For example, you could enter \u201cknowbe4.com\u201d.
    2. \n
  2. \nCSV Import: You can use this method to import multiple domains by uploading a CSV file. The CSV file you upload must have a Domain heading. In the Domain column, you should add one domain to each cell, line by line.
    3. \n
\n
\n
\nNote:Domains in this CSV file must be entered in the proper domain format. If you select the Replace all check box, the domains in your new CSV file will replace all of your current No Prompt Domains.

Installation Methods
\n
You can install Second Chance from your KSAT Account Settings, by using a command line, or by using your Group Policy. We recommend that you install Second Chance on an individual computer before deploying it to your entire organization.
\n
For more information about the installation methods, see the subsections below.
\n
Install from KSAT
\n
This method allows any user that has admin permissions to install Second Chance to a local computer.
\n
To install Second Chance by using this method, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and click the Second Chance tab.
    2. \n
  2. Click the Overview subtab.
    3. \n
  3. Download the Second Chance Training Device Installer (SecondChanceSetup.exe file).
    4. \n
  4. Open the downloaded SecondChanceSetup.exe file and follow the installation prompts. For more information, see the list below:\n
      \n
    • Accept the Second Chance License Agreement and click Install.
      • \n
    • Enter your license key.
      • \n
    • Click Finish to complete the installation.
      • \n
    • Test the installation by opening a mail client on the computer and clicking a link in an email. You should receive a Second Chance prompt.
      • \n
    \n
    5. \n
\n
If you don\u2019t receive a prompt, see the 8) I'm not receiving Second Chance prompts after installing through Group Policy question in our Second Chance: Frequently Asked Questions article.
\n
Install with Command Line
\n
This method allows you to install Second Chance on one computer by using a command line.
\n
To install Second Chance by using this method, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and click the Second Chance tab.
    2. \n
  2. Click the Overview subtab.
    3. \n
  3. Download and save the Second Chance Training Device Installer (SecondChanceSetup.exe file) located on the Overview tab.
    4. \n
  4. Open an elevated Command Prompt window by running Command Prompt as an administrator.
    5. \n
  5. In the Command Prompt window, navigate to the folder where you saved the Second Chance Training Device Installer.
    6. \n
  6. Enter the command below:\n
    \n\n
    \n
    SecondChanceSetup.exe /q /ComponentArgs \"MainInstaller\":\"LICENSEKEY=''[license key]\"\"\"
    \n
    \n
    \n
    Alternatively, you can install Second Chance with a proxy server by entering the command below:
    \n
    \n\n
    \n
    SecondChanceSetup.exe /q /ComponentArgs \"MainInstaller\":\"LICENSEKEY=''[license key]\"\" PROXYSERVER=''[hostname:port]\"\"\"
    \n
    \n
    \n
    \n
    \nNote:Before you run the command, replace \u201c[hostname:port]\u201d with the hostname or IP address and the TCP port of the HTTPS proxy server. Also, replace \u201c[license key]\u201d with your Second Chance license key. Do not remove any of the quotation marks.
    \n
    \n
    7. \n
  7. Press the Enter key on your keyboard.
    8. \n
  8. Test the installation by opening a mail client on the computer and clicking a link in an email. You should receive a Second Chance prompt.
    9. \n
  9. \n
    Click on a link within an email and make sure you're receiving Second Chance prompts.
    \n
    10. \n
\n
If you don\u2019t receive a prompt, see the 8) I'm not receiving Second Chance prompts after installing through Group Policy question in our Second Chance: Frequently Asked Questions article.

Install with Group Policy
\n
This method allows you to deploy Second Chance to specific organizational units (OUs) or your entire organization using Group Policy (GPO).
\n
\n
\nImportant:We recommend that you test this method on an individual computer before deploying Second Chance to your entire domain. To test this method, you'll need to create and link your GPO to a test OU. This OU should contain the computer that you would like to test Second Chance on.
\n
\n
To install Second Chance to your entire domain, follow the steps below:
\n
    \n
  1. Log in to your server as an administrator.
    2. \n
  2. Create a shared folder on your domain's network.
    3. \n
  3. Right-click the folder and open Properties.
    4. \n
  4. Follow the steps on the Security tab to add domain computers with Read permissions.
    5. \n
  5. Make sure you have Write permissions on the folder and that SYSTEM has Read & Execute permissions.
    6. \n
  6. Download and save the Second Chance Installer (SecondChanceSetup.exe file) to your shared folder.
    7. \n
  7. Create a .bat file in the shared folder by using the following lines:\n
    \n
    \nNote:Edit the fields below to match the location of the shared drive where you saved the SecondChanceSetup.exe installation file as well as your license key. Before you run the command, replace \u201c[hostname:port]\u201d with the hostname or IP address and the TCP port of the HTTPS proxy server. Also, replace \u201c[license key]\u201d with your Second Chance license key. Do not remove any of the quotation marks.
    \n
    \n
    \n\n
    \n
    copy /B /Y \"\\\\NetworkPath\\Name of Shared Folder\\SecondChanceSetup.exe\" \"%TEMP%\\SecondChanceSetup.exe\"\n\"%TEMP%\\SecondChanceSetup.exe\" /q /ComponentArgs \"MainInstaller\":\"LICENSEKEY=''[license key]\"\"\"
    \n
    \n
    \n
    Alternatively, if you have a proxy server filtering outbound traffic, you can specify the proxy server within the .bat file using the following lines:
    \n
    \n\n
    \n
    copy /B /Y \"\\\\NetworkPath\\Name of Shared Folder\\SecondChanceSetup.exe\" \"%TEMP%\\SecondChanceSetup.exe\"\n\"%TEMP%\\SecondChanceSetup.exe\" /q /ComponentArgs \"MainInstaller\":\"LICENSEKEY=''[license key]\"\"\" PROXYSERVER=''[hostname:port]\"\"\"
    \n
    \n
    \n
    \n
    \nNote: Second Chance is proxy-aware and will try to connect to the internet with the proxy server specified above for the local user's account. If your proxy is configured in a way that Second Chance cannot automatically detect, you can manually specify the proxy server Second Chance will use to connect to the internet during installation.
    \n
    \n
    8. \n
  8. Save the .bat file in the shared folder you created. The shared folder should now contain the .exe file and the .bat file.
    9. \n
  9. Create a Group Policy by following the steps below:
    10. \n
      \n
    1. Open the Group Policy Management Tool (gpmc.msc).
      2. \n
    2. Click the Forest and Domains drop-down menu and then right-click your domain. For example, your domain could be \u201cDomain.local\u201d or \u201cDomain.com\u201d.
      3. \n
    3. Click Create a GPO in this domain, and Link it here\u2026.
      4. \n
    4. Follow the options on the screen to name your GPO and click OK.
      5. \n
    5. Right-click your new GPO and select Edit\u2026.
      6. \n
    6. From the Edit\u2026 drop-down menu, select Computer Configuration -> Policies -> Windows Settings.
      7. \n
    7. Click Scripts.
      8. \n
    8. Double-click Startup Scripts.
      9. \n
    9. On the Script tab, click Add... to open the Add a Script window.
      10. \n
    10. Click OK to close the Startup Properties screen. 
      11. \n
    11. Click Browse... and select the .bat file that you created.
      12. \n
    12. Click Open.
      13. \n
    13. Click OK to save the new startup script.
      14. \n
    14. Click OK to close the Startup Properties screen.
      15. \n
    15. Close the Group Policy Management Editor. On the Group Policy Management screen, right-click your new GPO and make sure the Link Enabled check box is selected. Your GPO should now be active.
      16. \n
    \n
  10. Test the installation by opening a mail client on the computer and clicking a link in an email. You should receive a Second Chance prompt.
    11. \n
\n
As long as the GPO is active, the .bat file will be run each time a computer that is included in the GPO starts.
\n
If you don\u2019t receive a prompt, see the 8) I'm not receiving Second Chance prompts after installing through Group Policy question in our Second Chance: Frequently Asked Questions article.

Data and Reports
\n
On the Overview subtab, you can view an overview of user actions over the last 24 hours and since installation. You can also click the Download CSV button to download a CSV file that lists the users who have Second Chance installed on their computers.
\n
The CSV file will download to the folder or location where the Second Chance Training Device Installer file is stored.
\n
\n
The Overview tab also contains two User Action graphs. From these graphs, you can see the User Action Percentages within the last 24 hours or 30 days.
\n
To learn about the URLs, Users, and Devices subtabs, you can view the specific users and devices where Second Chance prompts have occurred.
\n
URLs
\n
From the URLs subtab, you can see the URLs that users have clicked and been alerted by. You can filter the data by the past day, past seven days, past 30 days, or all URLs. You can also sort the URLs alphabetically, by the number of times users have opened a link, and by the last time a user opened a link.
\n
You can also click each of the URLs to view more information about the URL, including all users who were prompted after clicking that URL, the computers they were using, and the action they took after being prompted.
\n
You can also search for specific URLs with the search bar at the top of the subtab.
\n
Users
\n
From the Users subtab, you can view the users that have been prompted by Second Chance, the last computer they were prompted on, the number of actions they\u2019ve taken after being prompted, and the last action date. You can also search for specific users by using the search bar.
\n
Selecting the name of a user will take you to that user\u2019s individual report page, where you can see a list of the URLs that caused a Second Chance prompt to display. For each URL, you can see what action was taken by the user, when they took that action, and what computer the prompt occurred on. You can also select a listed computer to see what other prompts have occurred on the computer.
\n
Devices
\n
From the Devices subtab, you can view information about the computers that have received Second Chance prompts, such as the last user that was prompted on each computer and the version of Second Chance that is installed on each computer.
\n
Selecting a computer will take you to the computer's report page, where you can see a list of all URLs that triggered a Second Chance prompt on that computer. For each URL, you can see the action that the users took after the prompt occurred. You can also click the listed computer to see the other prompts that have occurred on that computer.
\n
You can also select a user from the Devices list to see the user's individual report page, showing the prompts they've received and actions they've taken.
\n
You can also search for specific computers with Search.
\n

\n
    \n
  1. \nControl Panel Uninstall: Open your Control Panel and navigate to the Add/Remove Programs section to uninstall Second Chance. Then, repeat these steps on each computer that has Second Chance.
    2. \n
  2. \nCommand-line Uninstall: Open a Command Prompt as an administrator and enter the command below:\n
    \n\n
    \n
    MsiExec.exe /X{DBEE0453-2CC5-4ECC-905A-0E5EF970B668} /qn /L*v %TEMP%\\scuninstalllog.txt
    \n
    \n
    \n
    3. \n
  3. \nGroup Policy Uninstall: Edit the original .bat file that you used to set up the installation with Group Policy. Replace the content in the file with the line below:\n
    \n\n
    \n
    MsiExec.exe /X{DBEE0453-2CC5-4ECC-905A-0E5EF970B668} /qn /L*v %TEMP%\\scuninstalllog.txt
    \n
    \n
    \n
    4. \n

\n
If you receive errors in Second Chance, see our Second Chance: Frequently Asked Questions article for the troubleshooting steps that we recommend.
\n

For more information, see the subsection below.
\n
Enable Troubleshooting File for Support
\n
To copy a troubleshooting file to send to our support team, follow the steps below:
\n
    \n
  1. Open your Second Chance installation folder.
    2. \n
  2. Double-click the support_enable_logging.bat file.
    3. \n
  3. In the pop-up window that opens, click Yes to allow the Troubleshooter app to make changes to your device.\n
    \n
    \nNote:This step enables verbose debug logging, which prompts your computer to thoroughly record the events that happen as Second Chance runs.
    \n
    \n
    4. \n
  4. When the Command Prompt window displays the Press any key to continue... message, press a key on your keyboard.
    5. \n
  5. Perform the action that caused the error to appear before you enabled verbose debug logging.
    6. \n
  6. When the error message appears again, write down the time and date. The time that the error occurs may help our support team troubleshoot the problem.
    7. \n
  7. Open your Second Chance installation folder again.
    8. \n
  8. Double-click the support_collect_logs.bat file.
    9. \n
  9. In the pop-up window that opens, click Yes to allow the Troubleshooter app to make changes to your device. Two Command Prompt windows will open and begin collecting logs, and a System Information window will open and display a progress bar.
    10. \n
  10. When the Command Prompt window displays the Press any key to continue... message, press a key on your keyboard.
    11. \n
  11. In your Second Chance installation folder, a new directory named TroubleshooterReport will be created. Compress this directory as a zipped file.
    12. \n
  12. Send the TroubleshooterReport zipped file to our support team.
    13. \n
\n
\n
\nNote: Verbose debug logging will automatically be disabled after three days.
