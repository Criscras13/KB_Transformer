In the Compliance Management module, you can export data about your scoped requirements, controls, and tasks. Then, you can download this export as a CSV file to review your data and keep records of data you may need in the future.\u00a0
\n
\n
\nTip: In addition to the exports that are explained in this article, you can also export all of the information included under a scope. To learn more about this scope export, see our How to Export Scope Information article.\u00a0
\n
\n
When you export data from the Compliance Management module, the CSV exports will be available on the Data Exports page. From the Data Exports page, you can download data exports to your device. To learn about all of the data exports available in your account, see our Data Exports Guide.
\n
See the sections below for information about the data exports available in the Compliance Management module. each of these data exports.\u00a0
\n
\n
Exporting Scoped Requirements
\n
You can export a CSV file of all requirements that are included in a scope. This CSV file will include information such as the requirement self-assessment responses and requirement notes.
\n
To export and download a CSV file of scoped requirements, follow the steps below.
\n
    \n
  1. Navigate to Compliance > Scopes from the navigation panel.
    2. \n
  2. Click on the name of the scope that contains the requirements that you would like to export.
    3. \n
  3. Click the Requirements tab.
    4. \n
  4. Click the Export CSV button.\n
    \n
    \nTip: If you apply filters or perform a search in the requirements table before clicking the Export CSV button, only the filtered results will be included in your export.
    \n
    \n\n
    5. \n
  5. In the Name of Export field, enter a name for your data export.
    6. \n
  6. Click the Create Export button.
    7. \n
  7. Navigate to Data Exports from the navigation panel.
    8. \n
  8. Find the data export that you created.\n
    \n
    \nNote: In the Type column of the Data Exports page, scoped requirement exports are labeled Scoped Requirements.
    \n
    \n
    9. \n
  9. In the Actions column, click the Download button. \n
    10. \n
\n
After you download your scoped requirements export, you can open the CSV file in your preferred spreadsheet program. See the table below to learn about the types of data that the scoped requirement export will include.
\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
Column HeaderDescription
Requirement IDThe requirement's ID.\n
\n
\nNote: If you are viewing your CSV file in Microsoft Excel, auto-formatting features may cause certain requirement IDs to display inaccurately. For example, if your requirement ID is \"1 a\", Microsoft Excel may auto-format this as \"1:00 AM\".
\n
\n
NameThe name of the requirement.
DescriptionThe requirement's description.
Self-Assessment ResponseThe response that was selected for the requirement when you completed the Scope Self-Assessment.\n
    \n
  • To learn more about Scope Self-Assessments, see our How to Complete a Scope Self-Assessment\u00a0article.
    • \n
\n
Number of ControlsThe number of controls that are mapped to the requirement.\u00a0
TagsThe tags you have added to the requirement, if applicable.
NotesThe notes that users have left on the requirement. Each note entry includes the first and last name of the user who left the note and the date and time that they left the note. The date and time are in Coordinated Universal Time, or UTC, format.\u00a0\n
\n
\nNote: If your notes for a requirement exceed 2,500 characters, some of the notes will not be included in your export.
\n
\n

\u00a0

Exporting Controls
\n
You can export a CSV file of the controls in your account. This CSV file will include information such as the number of mapped requirements and the number of task schedules for a control.\u00a0
\n
To export and download a CSV file of controls, follow the steps below.\u00a0
\n
    \n
  1. Navigate to Controls from the navigation panel.\u00a0
    2. \n
  2. Click the Export CSV button.\u00a0\n
    \n
    \nTip: If you perform a search in the controls table before clicking the Export CSV button, only the filtered results will be included in your export.
    \n
    \n\n
    3. \n
  3. In the Name of Export field, enter a name for your data export.
    4. \n
  4. Click the Create Export button.
    5. \n
  5. Navigate to Data Exports from the navigation panel.
    6. \n
  6. Find the data export you created.\n
    \n
    \nNote: In the Type column of the Data Exports page, control exports are labeled Controls.
    \n
    \n
    7. \n
  7. In the Actions column, click the Download button. \n
    8. \n
\n
After you download your controls export, you can open the CSV file in your preferred spreadsheet program. See the table below to learn about the types of data that the controls export will include.
\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
Column HeaderDescription
NameThe name of the control.
DescriptionThe control description.
User AssignedThe name of the user who is assigned to the control, if applicable.
Group AssignedThe name of the user group who is assigned to the control, if applicable.\n
    \n
  • To learn more about assigning user groups to controls, see our How to Work with User Groups\u00a0article.
    • \n
\n
Group LeadIf a user group is assigned to the control, this column displays the name of the person who is the group lead for the group.
Approving ManagerThe Approving Manager's name, if one is assigned to the control.\u00a0
Second Level Approving ManagerThe Second-level Approving Manager's name, if one is assigned to the control.
Number of Mapped RisksThe number of risks that are mapped to the control.\n
    \n
  • To learn more about risks, see our Risk Management Module Guide.
    • \n
\n
Number of SchedulesThe number of task schedules that have been created for the control.\n
    \n
  • To learn more about task schedules, see our How to Work with Tasks and Task Schedules for Controls\u00a0article.
    • \n
\n
Number of RequirementsThe number of scoped requirements that are mapped to the control.\n
\n
\nNote: If you are a Scope Administrator who does not have access to all of the scopes in your account, this column shows the number of scoped requirements under your allowed scopes that are mapped to this control.
\n
\n
Control HealthThe control's Control Health percentage.
Tags\u00a0The tags that you have added to this control, if applicable.
Treatment ScoreIf the control's treatment score, if applicable.\n
    \n
  • You can apply a treatment score to a control if the control is mapped to a risk. To learn more, see our Risk Scoring\u00a0article.
    • \n
\n
NotesThe notes that users have left on this control. Each note entry includes the first and last name of the user who left the note and the date and time that they left the note. The date and time will be in Coordinated Universal Time, or UTC, format.\u00a0\n
    \n
  • Click here to view an example of control notes in KCM GRC.
    • \n
\n
\n
\nNote: If your notes for a control exceed 2,500 characters, some of the notes will not be included in your export.\u00a0
\n
\n
Number of One-Time TasksThe number of one-time tasks that have been created for this control.\n
    \n
  • To learn more about one-time tasks, see our How to Work with Tasks and Task Schedules for Controls\u00a0article.
    • \n
\n

Exporting Tasks
\n
You can export a CSV file of the tasks in your platform. This export will include information such as the user assigned, the task status, the date the task was satisfied.
\n
To export and download a CSV file of tasks, follow the steps below.
\n
    \n
  1. Navigate to Tasks from the navigation panel.
    2. \n
  2. Click the Export CSV button.\n
    \n
    \nTip: If you apply filters or perform a search in the tasks table before clicking the Export CSV button, only the filtered results will be included in your export.
    \n
    \n\n
    3. \n
  3. In the Name of Export field, enter a name for your data export.
    4. \n
  4. Click the Create Export button.
    5. \n
  5. Navigate to Data Exports from the navigation panel.
    6. \n
  6. Find the data export you created.\n
    \n
    \nNote: In the Type column of the Data Exports page, task exports are labeled Tasks.
    \n
    \n
    7. \n
  7. In the Actions column, click the Download button. \n
    8. \n
\n
After you download your tasks export, you can open the CSV file in your preferred spreadsheet program. See the table below to learn about the types of data that CSV file will include.
\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
Column HeaderDescription
Task NameThe name of the task.
User AssignedThe first and last name of the user who is assigned to the task.
Approving ManagerThe Approving Manager's name, if one is assigned to the task.
Second-Level Approving ManagerThe Second-level Approving Manager's name, if one is assigned to the task.
StatusThis is the Current Task Status that is shown on the\u00a0Task Status area of the View Task page.\n
    \n
  • The task status can be Active, Satisfied, Past Due, Failed, or Closed Late.
    • \n
\n
StageThis is the Task Stage that is shown on the\u00a0Task Status area of the View Task page.\n
    \n
  • The task stage can be Open, Awaiting Approval, or Approved.
    • \n
\n
Satisfied DateThe date that the User Assigned marked the task as complete.\n
    \n
  • To learn more, see the Tasks section of our KCM GRC: Monitoring and Approving Tasks article.
    • \n
\n
Due DateThe task's due date.\n
    \n
  • To learn more about due dates, see the How Do I Create a Task Schedule (Or a One-time Task)? section of our How to Work with Tasks and Task Schedules for Controls article.
    • \n
\n
NotesThe notes that users have left on this task. Each note entry includes the first and last name of the user who left the note and the date and time that they left the note. The date and time will be in Coordinated Universal Time, or UTC, format.\u00a0\n
    \n
  • Click here to view an example of task notes in KCM GRC.
    • \n
\n
\n
\nNote: If your notes for a task exceed 2,500 characters, some of the notes will not be included in your export.
\n
\n
Control NameThe name of the control that the task is associated with.\u00a0
