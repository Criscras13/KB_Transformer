\n
\nImportant:Some whitelisting configurations may not be available in environments outside of the US.
\n
\n
To whitelist training notifications sent from KnowBe4 in your Microsoft Defender for Office 365 environment, follow the steps below.
\n
\n
\nNote:Before you can whitelist in Microsoft Defender, you\u2019ll need to add a custom email header name and header value in your KnowBe4 console and save them in a place you can access later. For more information, see our KnowBe4 Console Account Settings: Training article.
\n
\n
    \n
  1. Log in to your mail server Admin portal. Then, navigate to Admin centers > Exchange. \n
    2. \n
  2. Navigate to Mail flow > Rules.
    3. \n
  3. Select Add a rule > Create a new rule.\u00a0 \n
    4. \n
  4. On the Set rule conditions page, give the rule a name, such as \u201cWhitelisting Training Notifications\u201d.
    5. \n
  5. In the Apply this rule if fields, select The message headers\u2026 and includes any of these words. \n
    6. \n
  6. Click Enter text, then enter the custom header name you copied earlier.
    7. \n
  7. Click\u00a0Save.\u00a0 \n
    8. \n
  8. Click Enter words, then enter the custom header value you copied earlier.
    9. \n
  9. Click\u00a0Add, then click\u00a0Save.\u00a0 \n
    10. \n
  10. In the Do the following fields, select Modify the message properties and set the spam confidence level (SCL). \n
    11. \n
  11. In the specify SCL window, select Bypass spam filtering. Then, click Save. \n
    12. \n
  12. At the bottom of the page, click Next.
    13. \n
  13. On the Set rule settings page, navigate to the Match sender address in message drop-down menu. Then, select Envelope. \n
    14. \n
  14. Click Next.
    15. \n
  15. Ensure your settings are correct. Then, click Finish. \n
    16. \n
\n
\n
\nTip: To ensure that you have whitelisted correctly, see our How to Verify You Have Whitelisted KnowBe4 Correctly article.
\n
