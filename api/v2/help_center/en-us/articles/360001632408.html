In this article, you will learn how to add the KnowBe4 Phish Alert Button (PAB) to your list of trusted apps in Google Workspace. For more information about whitelisting connected apps in Google Chrome, visit Google's Control which third-party & internal apps access Google Workspace data article.
\n
To whitelist the PAB app for Chrome, follow the steps below:
\n
    \n
  1. Log in to your Google Admin account.
    2. \n
\n
    \n
  1. In the Google Workspace Admin Console, click Security.\n\n
    2. \n
\n
    \n
  1. Click API controls.\n\n
    2. \n
\n
    \n
  1. From the App access control section, click Manage Third-Party App Access.\n
    \n
    2. \n
  2. From the Configured apps table, click Add app. Then, select OAuth App Name Or Client ID. The Configure an OAuth app window will open.\n\n
    3. \n
  3. In the Search for OAuth app name or client ID field, enter the following OAuth2 Client ID:682409154127-0iivv9sj7c06t2niq3cb54l8tshdj0gq.apps.googleusercontent.com\n
    4. \n
  4. Click Select next to the KnowBe4 Phish Alert app name.\n\n
    5. \n
  5. From the OAuth Client IDs list, select the same client ID and click Select.\n\n
    6. \n
  6. From the Scope list, select the organizational units you want to have access to the app. Then, click Continue.\n\n
    7. \n
  7. From the Access to Google Data list, select Trusted. Then, click Continue.\n
    \nNote: If an Allowlist for exemption from API access blocks in context-aware access. Available only for apps added via OAuth clients IDs. option is displayed, select this check box. Then, click Continue.
    \n\n\n
    8. \n
  8. From the Review page, confirm that you have configured the KnowBe4 Phish Alert app correctly for your organization. Then, click Finish.After you've configured the app settings, the KnowBe4 Phish Alert app will be displayed in the Configured apps table.\u00a0\n\n
