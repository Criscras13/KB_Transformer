This guide explains how to deploy the web add-in for your Microsoft 365 users. This solution enhances your organization's security by providing powerful encryption services and delivering real-time guidance when sending emails. It ensures consistent protection throughout your organization, as the add-in will be deployed to all supported devices, including Outlook Web Access (OWA), Mac, and Windows.
\n
Prerequisites
\n
Before you get started, please ensure you have the following:
\n
    \n
  • Global admin rights to your organization's Microsoft 365 admin portal
    • \n
  • Microsoft Outlook JavaScript (JS) API requirement set 1.8 or later
    • \n
  • A gateway must be provisioned for all Web add-in deployments
    • \n
  • We recommend that the gateway be configured with a moderation mailbox. See the Gateway Configuration article for more details.
    • \n
  • Optional: Microsoft 365 or Azure AD group to assign Office JS to a user group
    • \n
\n
Deployment Steps
\n
    \n
  1. Log in to your Microsoft 365 admin portal.\u00a0
    2. \n
  2. Navigate to\u00a0Settings > Integrated apps > Add-ins.\n
      
    \n
    3. \n
  3. Select I have a URL for the manifest file and paste the URL for the Microsoft add-in manifest file into the field.\n
      \n
    • \nUK Manifest: https://www.knowbe4.com/hubfs/prod-uk-manifest.xml\n
      • \n
    • \nUS Manifest: https://www.knowbe4.com/hubfs/prod-us-manifest.xml\n
      • \n
    • \nEU Manifest: https://www.knowbe4.com/hubfs/prod-eu-manifest.xml\n
      • \n
    • \nAU Manifest: https://www.knowbe4.com/hubfs/prod-au-manifest.xml\n
      • \n
    \n
    4. \n
  4. Once you select Upload, you will need to select who you want to deploy the application to and how you want to deploy it:\n
      \n
    • \nAssign Users: Select Specific users/groups and add the chosen users or groups.
      • \n
    • \nDeployment Method: Select Optional.
      • \n
    \n
    5. \n
  5. Click Deploy.
    6. \n
  6. A green tick will display when the add-in is deployed. It may take up to 12 hours before the application appears for users.
    7. \n
\n
