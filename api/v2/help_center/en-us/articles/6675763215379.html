In this article, you will learn how to integrate Gmail, Google Drive, and Google IAM with SecurityCoach. Once you set up this integration, data provided by Google will be available under the SecurityCoach tab of your KSAT console. This data can be viewed in SecurityCoach reports and used to create detection rules for real-time coaching campaigns.
\n
\n
\nTip:SecurityCoach also supports integrating Google as a Splunk vendor. If your organization uses Splunk for security information and event management, we recommend integrating through Splunk instead by following our Splunk Integrations Guide for SecurityCoach.
\n
\n
\n
\n\n
\n
\n
\n\n
\n
For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n
Set Up the Integration in Your Google Cloud Platform
\n
Before you can set up the SecurityCoach integration in your KSAT console, you will need to set up the integration in your Google Cloud platform by creating a project and a service account. You will also need to enable APIs for your project and obtain a JSON file and unique ID from your service account.
\n
To jump to the article subsection for each of these steps, click the links below:
\n
Create a Project
\n
Enable APIs for Your Project
\n
Create a Service Account
\n
Obtain the JSON File and Unique ID of a Service Account
\n
Create a Project
\n
To create a project in your Google Cloud platform, follow the steps below:
\n
    \n
  1. Log in to your Google Cloud platform.
    2. \n
  2. Navigate to the Google Cloud search bar near the top of the page and search for \u201cIAM & Admin Create a Project\u201d. Click Create a Project from the drop-down menu below the search bar.\n
      
    \n
    3. \n
  3. Enter \u201cKB4VendorIntegration\u201d in the Project name field. The Organization and Location fields will automatically populate for you.\n
      
    \n
    4. \n
  4. Click Create.
    5. \n
\n
Enable APIs for Your Project
\n
To enable APIs for the project you created in the Create a Project section above, follow the steps below:
\n
    \n
  1. Navigate to https://console.cloud.google.com.\n
    2. \n
  2. At the top of your Google Cloud platform, click the drop-down arrow and select KB4VendorIntegration as the current project.
    3. \n
  3. Navigate to the Google Cloud search bar near the top of the page and search for \u201cAPIs & Services Library\u201d. Click Library from the drop-down menu below the search bar.\n
      
    \n
    4. \n
  4. Use the API Library search bar to search for Enter \u201cAdmin SDK API\u201d. Click Admin SDK API from the drop-down menu below the search bar.\n
      
    \n
    5. \n
  5. Click Enable.\n
      
    \n
    6. \n
\n
Create a Service Account
\n
To create a service account in your Google Cloud platform, follow the steps below:
\n
    \n
  1. At the top of your Google Cloud platform, click the drop-down arrow and select KB4VendorIntegration as the current project.
    2. \n
  2. Navigate to the Google Cloud search bar near the top of the page and search for \u201cIAM & Admin Service Accounts\u201d. Click Service Accounts from the drop-down menu below the search bar.\n
      
    \n
    3. \n
  3. In the menu bar at the top of the page, click + Create Service Account.\n
      
    \n
    4. \n
  4. Enter a name for your new service account in the Service account name field. We recommend \u201cKB4MyOrganizationGSuiteServiceAcc\u201d, with \u201cMyOrganization\u201d being your organization\u2019s name.
    5. \n
  5. For the Service account description field, enter \u201cThis is the service account for integration with KnowBe4\u201d
    6. \n
  6. Click Create And Continue.\n
      
    \n
    7. \n
  7. Click the Role drop-down menu and select Basic > Viewer.\n
    \n
    8. \n
  8. Click the white Continue button.
    9. \n
  9. Skip the Grant users access to this service account step and click the blue Done button.\n
    \n
    10. \n
\n
Obtain Your Service Account JSON File and Unique ID
\n
After you have created your service account, you will need to obtain your service account JSON file and unique ID. You will need both of these items later in the integration setup process.
\n
To obtain your JSON file and unique ID, follow the steps below:
\n
    \n
  1. At the top of your Google Cloud platform, click the drop-down arrow and select KB4VendorIntegration as the current project.
    2. \n
  2. Navigate to the Google Cloud search bar near the top of the page and search for \u201cIAM & Admin Service Account\u201d. Click Service Accounts from the drop-down menu below the search bar.
    3. \n
  3. Click the service account\u2019s name in the Email column. When you click, the service account\u2019s Details page will open. This page lists the Unique ID.\n
      
    \n
      
    \n
    4. \n
  4. Copy the Unique ID and save it to a place that you can easily access later. You will need it to complete the steps in the Assign Domain-Wide Delegation and Scopes section of this article.
    5. \n
  5. To obtain the JSON file, click Keys in the top menu bar.
    6. \n
  6. Click Add Key drop-down menu and select Create new key from the drop-down menu.\n
    \n
    7. \n
  7. In the pop-up window that opens, select JSON for the Key type and then click Create. The JSON file will automatically download to your device, and the private key will save in the service account's Keys tab.\n
    \n
    8. \n
\n
\n
\nNote:You will need this JSON file when you complete the integration setup process in the Set Up the Integration in Your KSAT Console section below.
\n
\n
Assign Domain-Wide Delegation and Scopes
\n
After you have set up the integration in your Google Cloud Platform and your KSAT console, you can assign domain-wide delegation and scopes.
\n
To assign domain-wide delegation and scopes, follow the steps below:
\n
    \n
  1. Navigate to admin.google.com and enter your administrator login credentials.
    2. \n
  2. Navigate to the Google Admin search bar near the top of the page and search for \u201cSecurity Access and data control API controls\u201d. Click API controls from the drop-down menu below the search bar.
    3. \n
  3. Scroll down in the sidebar on the right side of the page until you see the Domain wide delegation section. In this section, click Manage Domain Wide Delegation.\n
    \n
    4. \n
  4. Click Add new. When you click this button, the Add a new client window will open.
    5. \n
  5. In the Client ID field, enter your service account ID. This is the ID from Step 3 in the Obtain Your Service Account JSON File and Unique ID section of this article.
    6. \n
  6. In the OAuth scopes (comma-delimited) field, enter \u201chttps://www.googleapis.com/auth/admin.reports.audit.readonly\u201d.
    7. \n
  7. Once you have entered your service account ID and scope, click Authorize.\n
    \n
    8. \n
\n
Set Up the Integration in Your KSAT Console
\n
Once you've set up the integration in your Google Cloud platform, you can set up the integration in your KSAT console.
\n
To set up the integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to SecurityCoach > Setup.
    3. \n
  3. In the Available Integrations section, locate the card for the Google integration you want to set up.
    4. \n
  4. At the bottom of the card, click Configure.\n
    \n
    5. \n
  5. To enter the required Google Workspace information, see the screenshot and list below:\n
      
    \n
      \n
    • In the Admin Email field, enter your Google Workspace admin email address.
      • \n
    • In the Credentials File field, click Browse and select the JSON file you downloaded in the Obtain Your Service Account JSON File and Unique ID section of this article.
      • \n
    • Click Connect\n
      • \n
    \n
    6. \n
\n
Map Your Users
\n
After you\u2019ve finished configuring your Google integration, you can map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Map Users in SecurityCoach article.
\n
\n
Once you\u2019ve successfully authorized this integration, you can manage detection rules for your Google integration on the Detection Rules subtab of SecurityCoach. For a full list of available system detection rules for Gmail, Google Drive, and Google IAM, see our System Detection Rules by Vendor article.
