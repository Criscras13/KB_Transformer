In this article, you will learn how to integrate Cisco Secure Email, formerly Cisco Email Security Appliance (ESA), with SecurityCoach. Once you set up this integration, data provided by the Cisco Secure Email will be available under the SecurityCoach tab of your KSAT console. This data can be viewed in SecurityCoach reports and used to create detection rules for real-time coaching campaigns. For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n
\n\n
\n
\nNote:To set up this integration, you will need access to a Cisco administrator account.
\n
\n
Create Your Cisco Secure Email User
\n
Before you can set up this integration in your KSAT console, you will need to create a Cisco integration user role. Then, you will need to create a user with this role.
\n
To jump to the article subsection for each of these steps, click the links below:
\n
    \n
  • Create a Cisco Secure Email Integration User Role
    • \n
  • Create a Cisco Secure Email User
    • \n
\n
Create a Cisco Secure Integration User Role
\n
First, you will need to create an integration user role in your Cisco account. To create your integration user role, follow the steps below:
\n
    \n
  1. Log in to your Cisco administrator account.\n\n
    2. \n
  2. Navigate to System Administration > User Roles.\n\n
    3. \n
  3. Click Add Email User Role\u2026.\n\n
    4. \n
  4. Fill out the fields on the Add Email User Role page. For more information about these fields, see below:\n\n
      \n
    1. \nName: Enter \u201cintegration_role\u201d.
      2. \n
    2. \nEmail Reporting: Select the Access to data by Reporting Group option, then select All Reports from the drop-down menu.
      3. \n
    3. \nMessage Tracking: Select the View Message Tracking option.
      4. \n
    4. \nQuarantines: Select the View Only Access option.
      5. \n
    5. \nLog Subscriptions: Select the Log Subscription Access option.
      6. \n
    \n
    5. \n
  5. Click Submit. Your new user role will display in the User Roles table.
    6. \n
  6. Click Commit Changes at the top-right corner of the page.\n\n
    7. \n
  7. (Optional) Enter a comment about the changes in the Comment (optional) field.\n\n
    8. \n
  8. Click Commit Changes at the bottom-right corner of the pop-up window to create your new user role.
    9. \n
\n
Create a Cisco Secure Email User
\n
After you have created an integration role, you can create a user with this role in your Cisco account. You will need the user\u2019s credentials that you create in this section to complete the integration setup in your KSAT console.
\n
To create your Cisco Secure Email user, follow the steps below:
\n
    \n
  1. Log in to your Cisco administrator account. \n\n
    2. \n
  2. Navigate to System Administration > Users. \n\n
    3. \n
  3. On the Users page, click Add User\u2026. \n\n
    4. \n
  4. Fill out the fields on the Add Local User page. For more information about these fields, see below: \n
      \n
    1. \nUser Name: Enter a username for the user. This username can only include lowercase letters, numbers, and dashes. In the example above, the User Name is kb4-integration.\n
      \n
      \nNote:Make sure to remember this username. You will need it to complete the integration setup in the Set Up the Integration in Your KSAT Console section of this article.
      \n
      \n
      2. \n
    2. \nFull Name: Enter a name for the user. In the example above, the Full Nameis KnowBe4 Security Coach.
      3. \n
    3. \nUser Role: Select the Custom Roles option and then select integration_role from the list.
      4. \n
    4. \nConfirm your Passphrase to make changes: Enter the password for the Cisco administrator account you are currently using.
      5. \n
    5. \nPassphrase: Select either Generate a passphrase or Enter a passphrase of your choice. This will be the passphrase for the user\u2019s account.\n
        \n
      • If you select Generate a passphrase, click Generate to generate a passphrase.
        • \n
      • If you select Enter a passphrase of your choice, create your own passphrase and enter it in the Passphrase and Retype Passphrase fields.\n
        • \n
      \n
      \n
      \nNote:Make sure to remember this passphrase. You will need it to complete the integration setup in the Set Up the Integration in Your KSAT Console section of this article.
      \n
      \n
      6. \n
    \n\n
    5. \n
  5. Click Submit. Your new user will display in the Users table.
    6. \n
  6. Click Commit Changes at the top-right corner of the page. \n\n
    7. \n
  7. (Optional) Enter a comment about the changes in the Comment (optional) field. \n\n
    8. \n
  8. Click Commit Changes at the bottom-right corner of the pop-up window to create your new user.
    9. \n
\n
Locate the API Domain
\n
Before you set up the integration in your KSAT console, you will also need to locate your API domain. This domain is displayed in the URL of your Cisco Secure Email account.
\n
For example, in the image below, the API domain is \u201cdh5802-sma1.iphmx.com\u201d.
\n\n
You will need this API domain to complete the setup process in the Set Up the Integration in Your KSAT Console section below.
\n
Set Up the Integration in Your KSAT Console
\n
Once you have created a user with an integration role in your Cisco account and located your API domain, you can set up the integration in your KSAT console.
\n
To set up the integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and navigate to SecurityCoach > Setup > Security Vendor Integrations.
    2. \n
  2. Locate Cisco Secure Email\u00a0and click Configure.\n\n
    3. \n
  3. In the Username field, enter the username you created in step 4 of the Create Your Cisco Secure Email User section above.
    4. \n
  4. In the Password field, enter the passphrase you generated or created in step 4 of the Create Your Cisco Secure Email User section above.
    5. \n
  5. In the API Domain field, enter your API domain.
    6. \n
  6. Click Authorize. \n\n
    7. \n
\n
Map Your Users
\n
After you\u2019ve finished integrating Cisco Secure Email, you can map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Mapping Users in SecurityCoach article.
\n
Once you\u2019ve successfully authorized this integration, you can manage detection rules for Cisco Secure Email on the Detection Rules subtab of SecurityCoach. For a full list of available system detection rules for this vendor, see our Which Detection Rules Can I Use with My Vendors? article.
