You can view a list of enrolled users and their training progress on either the Users subtab of a training campaign or the User Progress page of a training assignment.
\n
\n
\nTip: To view your users' training progress, you can also use the reports found under the Reports tab in the Training section of your console. Learn more in our Training Reports Guide.
\n
\n
Users Subtab
\n
Follow the steps below to navigate to the Users subtab.
\n
    \n
  1. In your KSAT console, click the Training\u00a0tab.
    2. \n
  2. From the Campaigns tab, click the name of the desired training campaign.
    3. \n
  3. Click the Users subtab.
    4. \n
\n
The Users subtab lists all enrolled users and details about each piece of content assigned to them. For more information, see the screenshot and list below:
\n
a. Subtabs: Click a subtab to view All Users in this training campaign or filter users by completion status. These subtabs also show the number and percentage of users within each completion status
\n
b. Search: Find a specific user by entering their name or email address into the search field.
\n
c. User Table: This table is organized by user first and then by individual training assignments. You can see information such as the date the user was enrolled in the content, whether or not the user has started the assigned content, and how much time the user has spent viewing the content.
\n
    \n
  • The Time Remaining column will display the number of days the user has to complete the training. If the extended due date is greater than 30 days from the current date, the Time Remaining column will round to the closest number of months the user has to complete the assignment.\n
    \n
    \nTip: To see the exact due date, navigate to the user's profile and view Extended to under their Training Details. If you need to see the extended due date for a larger number of users, click Generate CSV to download a CSV file. Then, refer to the Extended Due Date column. Users will see how many days their assignments are overdue when they log in to complete their training.
    \n
    \n
    • \n
\n
d. Notification History: Click the envelope icon next to a user\u2019s name to see which training notifications you have sent to that user. You can see when each notification was sent, whether or not the notification was received, and the name of the notification template sent. However, you can only see the notification history for a manager or admin if they are also enrolled in the campaign. If the manager or admin received notifications regarding their users, those notifications will be included in the notification history.
\n
\n
\nImportant: This feature was released on March 10, 2022. Notifications sent before this date will not be listed in the Notification History table.
\n
\n
e. Include Archived Users: Enable this setting to include all archived users that have been enrolled in this campaign. Because these users are archived, you can't send them manual notifications or change their training progress.
\n
f. Bulk Update: This option allows you to upload a CSV file to pass users, reset users\u2019 training progress, or remove users from a campaign in bulk. For more information, see our Can I Manually Update the Training Progress of a User? article.
\n
g. Generate CSV: Click this button to generate a CSV file. Once generated, you can download the CSV file from your Download Center. See our Download Center Guide for more information.\u00a0
\n
h. + Enroll Users: Click this button to individually enroll additional users in this training campaign. For more information, see our Can I Enroll Users in a Training Campaign That Is in Progress? article.
\n
\n
\nNote:: This button will only display for active training campaigns that are In Progress.
\n
\n
i. Actions: To enable the Actions button, select one or more check boxes next to the content title. Then, select one of the following actions:
\n
    \n
  • \nSend Notification: Manually send the user a notification regarding the assigned content.
    • \n
  • \nMark as Completed: Mark the user's assigned content as completed.
    • \n
  • \nReset Progress: Reset any progress the user made on the assigned content.\u00a0
    • \n
  • \nExtend Selected: This option will be available when you view the Past Due subtab. When viewing the Past Due subtab, the option Extend Selected will also be available. This option allows you to set a new due date for the user's assigned content. See our How to Modify Training Deadlines article for more information.
    • \n
\n
Remove Users: Remove the selected user from the training campaign.
\n
j. Status: This column shows the user's training completion status. The status can be Completed, In Progress, Not Started, or Past Due. This column also includes the following features for completed assignments:
\n
    \n
  • You can hover your cursor over the green Completed badge to view the date and time that the user completed the assigned content.
    • \n
  • If the user has past due training, the red Past Due badge will change to a green\u00a0Past Due badge once they\u2019ve completed the training.
    • \n
  • The Impersonated badge is displayed when an account admin impersonates a user and completes the training for their user.
    • \n
  • You will see the Forced Completion badge when the account admin selects Mark as Completed from the Actions drop-down menu on the Users subtab, or when an admin uses Bulk Update to upload a CSV file to pass users.\n
    \n
    \nNote: This data will be added to a new Admin Action column on CSV files downloaded from the Users subtab. Also, this information is currently only available from this page. It\u2019s not currently available in the Reports tab.
    \n
    \n
    • \n
  • You can click the trophy icon to download a training completion certificate.
    • \n
  • The exclamation point icon\u00a0indicates this training was completed after the original due date.
    • \n
  • The hourglass icon indicates that the training assignment due date has been extended.
    • \n
\n
k. Score: This column displays the user's score on the respective content.
\n
If this campaign includes the Security Awareness Proficiency Assessment (SAPA), the user\u2019s assessment score will also be listed in this column. To learn more about the SAPA, see our What Is the Security Awareness Proficiency Assessment (SAPA)? article.
\n
\n
\nNote: Scores will only display for eligible training content if the Track Scores option was enabled when the campaign was created. See the Individual Campaign Overview section of our Creating and Managing Training Campaigns article for more information.
\n
\n
