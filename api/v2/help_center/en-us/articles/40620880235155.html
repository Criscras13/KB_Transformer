This guide explains how to deploy the web add-in for your Microsoft 365 users. This feature enhances your organization's security by providing powerful encryption services and delivering real-time guidance when sending emails. It ensures consistent protection throughout your organization, as the add-in will be deployed to all supported devices, including Outlook Web Access (OWA), Mac, and Windows.
\n
Prerequisites
\n
Before you get started, please ensure you have the following:
\n
    \n
  • Global admin rights to your organization's Microsoft 365 admin portal
    • \n
  • Microsoft Outlook JavaScript (JS) API requirement set 1.8 or later
    • \n
  • A gateway must be provisioned for all web add-in deployments
    • \n
  • If you are using Prevent with the Web Add-in, you must have Graph API ingestion enabled.
    • \n
\n
Deployment Steps
\n
To deploy the web add-in, follow the steps below:
\n
    \n
  1. Log in to your Microsoft 365 admin portal.
    2. \n
  2. Navigate to Settings > Integrated apps > Upload custom apps.
    3. \n
  3. Use the App type drop-down menu to select Office Add-in.
    4. \n
  4. Select Upload manifest file (XML) from device and select Choose File.
    5. \n
  5. Select the .XML file you downloaded from the deployment center or Prevent admin console. The manifest files are also listed below.\u00a0\n
     
    \n
      \n
    • \nUK Manifest: https://www.knowbe4.com/hubfs/prod-uk-manifest.xml\n
      • \n
    • \nUS Manifest: https://www.knowbe4.com/hubfs/prod-us-manifest.xml\n
      • \n
    • \nEU Manifest: https://www.knowbe4.com/hubfs/prod-eu-manifest.xml\n
      • \n
    • \nAU Manifest: https://www.knowbe4.com/hubfs/prod-au-manifest.xml\n
      • \n
    \n
    6. \n
  6. Select Next.
    7. \n
  7. Under Assign users, select Specific users/groups.
    8. \n
  8. Search for and select the nudge group you created in the deployment center. The default name for this group is Prevent_Nudge_Group.\n
      
    \n
    9. \n
  9. Select Next.
    10. \n
  10. Read the app permissions and capabilities.
    11. \n
  11. In the pop-up window, log in to your Microsoft admin account.
    12. \n
  12. Review the permissions request and select Accept.
    13. \n
  13. Select Next.
    14. \n
  14. Review the deployment summary.
    15. \n
  15. Select Finish deployment.
    16. \n
\n
