In KCM GRC, you can create or import user accounts and assign user roles to each account. By assigning user roles, you can give users access to only the information that they need to perform their responsibilities. To learn about the user roles that KCM GRC offers, see our KCM GRC: User Roles article.
\n
After you create user accounts and assign user roles, you can manage the accounts by updating user information, confirming user accounts, and disabling user accounts.\u00a0\u00a0
\n
Creating User Accounts
\n
To create a user account in your platform, follow the steps below:\u00a0
\n
\n
\nNote: If you're using KCM GRC's Policy Management module to manage internal policy acknowledgments, you will need to use a different method to add end-users to your KCM GRC account. To learn about this method, see the Step Three: Add End Users to the Policy Management Module section of our Policy Management Module Guide.
\n
\n
    \n
  1. Log in to your platform and click the Settings button at the top-right corner of the page.
    2. \n
  2. Select Create New User from the drop-down menu.\u00a0\n\n
    3. \n
  3. On the Create User page, fill out each of the fields. To learn about each field, see the list below:\n
      \n
    1. \nFirst Name: Enter the user's first name. The user will be addressed by their first name in the automated emails sent from your KCM GRC account.
      2. \n
    2. \nLast Name: Enter the user's last name.
      3. \n
    3. \nEmail: Enter the user's business email.\u00a0\n
      \n
      \nImportant: The user must be able to receive emails at this email address in order to confirm their account.
      \n
      \n
      4. \n
    4. \nTelephone: Click the flag drop-down menu and select the telephone number format for your country. Then, enter the user's phone number.
      5. \n
    5. \nUser Roles: Click the drop-down menu and select one or more user roles to assign to the users. Please refer to our User Roles article to learn about the permissions for each user role.\n
        \n
      • \nAllowed Scopes: Click the drop-down menu and select the scopes you'd like the user to have access to. This field is only applicable for Scope Administrator and Auditor roles.
        • \n
      • \nAllowed Campaigns: Click the drop-down menu and select the policy campaigns that you'd like the user to have access to. This field is only applicable to Campaign Administrator and Auditor roles.
        • \n
      \n
      6. \n
    6. \nCreate: After entering all of the user's information, click this button.\u00a0
      7. \n
    \n\n
    4. \n
\n
Importing User Accounts
\n
If you are creating multiple user accounts at once, you can save time by importing users with a CSV file. When you import users into your account, the users are automatically assigned the Contributor user role. After you import your users, you can change their user roles, or you can add additional user roles.
\n
When you create your CSV file, follow the guidelines below:\u00a0
\n
    \n
  • The file should be a valid CSV file with a comma separator.\n
    \n
    \nImportant: If you are using Microsoft Excel, your file must be saved in the CSV UTF-8 (Comma delimited) format.
    \n
    \n
    • \n
  • The following header line is required:\n
    email,first_name,last_name
    \n
    • \n
\n
To import users into your account with a CSV file, follow the steps below:
\n
    \n
  1. Log in to your platform and click the Settings button at the top-right corner of the page.
    2. \n
  2. Select Manage Users from the drop-down menu.
    3. \n
  3. Click the Import button at the top of the page.\n\n
    4. \n
  4. In the Import Items window, click the Click to Upload button. \n\n
    5. \n
  5. Select the CSV file that contains your list of users.
    6. \n
  6. (Optional) Review the users in the list. If you would like to remove a user from the import, click the trashcan icon.\n\n
    7. \n
  7. When you're ready to import these users, click the Import Items button.
    8. \n
\n
After you click the Import Items button, user accounts will be created for all users that you imported. These users will receive an email that prompts them to activate their accounts. For information about account activation, see our How to Activate Your Account, Access Your Account, and Reset Your Password article.
\n
Until they have been assigned to one or more tasks, Contributors are unable to access data in your KCM GRC platform. For more information, see our KCM GRC: User Roles article. If you'd like to change a user's user role, see the Managing Users section of our KCM GRC: User Roles article.
\n
Managing User Accounts
\n
If you're an Account Administrator, you can monitor and manage your users' details from the Manage Users page. To navigate to the Manage Users page, log in to your platform and click the Settings button in the top-right corner of the page. Then, select Manage Users from the drop-down menu. \u00a0
\n\n
For more information about the Manage Users page, see the list below:
\n
    \n
  1. \nFull Name: This column displays the user's full name. If you have created one or more user groups for your Compliance Management module, you can select the check box next to each user that you'd like to add to a group. Then, you can select the desired group from the Select a Group drop-down menu and click Add To Selected Group.\n
      \n
    • To learn about working with user groups in the Compliance Management module, please see our Working with User Groups article.
      • \n
    \n
    2. \n
  2. \nStatus: This column displays the status of the user account. The statuses are Active, Awaiting Confirmation, and Disabled.\n
      \n
    • To learn about account confirmation, see the Confirming User Accounts section below.
      • \n
    \n
    3. \n
  3. \nDate Updated: This column displays the last date that changes were made to the user's account. This includes the user's email address, phone number, or user role.
    4. \n
  4. \nDisable User Icon: Click this icon to disable the user account and transfer their responsibilities to a different user.\n
      \n
    • For more information, see the Disabling User Accounts and Transferring Responsibilities section below.
      • \n
    \n
    5. \n
  5. \nPencil Icon: Click this\u00a0icon to edit the user's account. When you click this icon, you'll be taken to the User Management page.\n
      \n
    • For more information about the User Management page, see the Updating User Details section below.
      • \n
    \n
    6. \n
\n
Updating User Account Details
\n
If you're an Account Administrator, you can update your users' details and account settings. For example, you can change a user's user roles or give the user access to additional scopes.
\n
To view a user's profile, follow the steps below:
\n
    \n
  1. Log in to your platform and click the Settings button at the top-right corner of the page.
    2. \n
  2. From the drop-down menu, select Manage Users.
    3. \n
  3. Click the pencil icon under the Actions column.
    4. \n
\n
\n
\nNote: If you're not an Account Administrator and you'd like to update your own account information, log in to your account and click your name at the top-right corner of the page. Then, click the Profile button.
\n
\n
To learn about the settings that are available on the User Management page, see the details below:
\n\n
    \n
  1. \nSet Up MFA: Click this button to connect your user account with your organization's multi-factor authentication (MFA) application.\n
      \n
    • For details about configuring MFA for your user account, configuring MFA for individual accounts, and configuring MFA for all user accounts, see our Enable and Set Up Multi-factor Authentication article.
      • \n
    \n
    2. \n
  2. \nDisable User:\u00a0Click this button to disable the user's account and transfer their responsibilities to another user.\n
      \n
    • For more information, see the Disabling Users and Transferring Responsibilities section below.
      • \n
    \n
    3. \n
  3. This widget provides an overview of the user's responsibilities in your KCM GRC console.\n
      \n
    • \nScopes: If the user is a Scope Administrator or an Auditor, this shows the number of scopes that the user has been granted access to.
      • \n
    • \nPolicy Campaigns:\u00a0If the user is a Campaign Administrator or an Auditor, this shows the number of policy management campaigns the user has been granted access to.
      • \n
    • \nTasks: When applicable, this shows the number of tasks the user is currently assigned to. For more information, see our Working with Task Schedules for Controls article.\u00a0
      • \n
    • \nCompliance Percentage: When\u00a0applicable, this shows the percentage of this user's assigned tasks that were satisfied on time.
      • \n
    \n
    4. \n
  4. \nTask History: This tab shows all of the tasks the user is currently assigned to and the tasks the user has satisfied in the past. The Task History table lists the following information for all tasks that the user is Responsible for:\n
      \n
    • \nName: The task name. The task may have the same name as the associated control.
      • \n
    • \nDue At: The date the task evidence is due or was previously due.
      • \n
    • \nCompleted On: The date the task was marked as complete.
      • \n
    • \nStatus: The current task status. The statuses are Failed, Past Due, Active, Satisfied, or Closed Late.\u00a0
      • \n
    • \nStage: The current stage of satisfying the task. The stages are Awaiting Approval, Approved, or Open. All failed or unsatisfied tasks will show the Open stage. All closed tasks will show the Approved stage.
      • \n
    \n
    5. \n
  5. \nAudit Trail: Click this tab to view actions that the user has made in your KCM GRC account.\n
    6. \n
  6. \nUser Roles: Click the drop-down menu and select additional user roles that you\u2019d like to add. To remove a user role, click the x button to the right of the role name.\n
    \n
    \nNote: Only Account Administrators can demote other Account Administrators' user roles. However, Account Administrators cannot demote their own user roles.
    \n
    \n
    7. \n
  7. \nAllowed Scopes: Click this drop-down menu to select the scopes you'd like the user to have access to. This field is only available for users with Scope Administrator user roles.
    8. \n
  8. \nAllowed Campaigns: Click the drop-down menu to search for or select the policy campaigns you'd like your users to have access to. This field is only available for users with Campaign Administrator user roles.\u00a0
    9. \n
  9. \nRequire MFA:\u00a0Click the slider button to mandate MFA for this user's account. This button is only available for Account Administrator user roles.\n
      \n
    • Before you enable this setting, ensure the user has installed an authenticator application. The user will be required to set up their MFA application the next time they log in to their KCM GRC account. For more information, see our Enable and Set Up Multi-Factor Authentication article.\n
      • \n
    \n
    10. \n
\n
Confirming User Accounts
\n
Once you've created a new user, they must confirm their account before they can log in. The user will receive an account confirmation email at the email address you provided when you created the user's account. For more information about activating a user account, see our KCM GRC: Account Activation, Account Access, and Resetting Your Password article.
\n
If the user did not receive the account confirmation email, be sure they've checked their Spam or Junk folder. If you need to resend the activation email to your user, see the Resending Account Confirmation Emails subsection below for instructions
\n
Resending Account Confirmation Emails
\n
If you're an Account Administrator and your user has not received an email to confirm their account, follow the steps below to resend their account confirmation email:
\n
    \n
  1. Log in to your platform and click the Settings button at the top-right corner of the page.
    2. \n
  2. From the drop-down menu, select Manage Users.\n\n
    3. \n
  3. From the Manage Users page, find the user who needs the confirmation email and click the update user icon in the far-right column.\n\n
    4. \n
  4. From the User Management page, click the Resend Activation button.\n\n
    5. \n
  5. In the Are you sure you want to reconfirm this user? prompt, select Yes.
    6. \n
\n
If the user does not receive the account confirmation email, we recommend that the user checks their Spam and Junk folders.
\n
To learn about account activation when SSO is enabled for your account, see our How to Activate Your Account, Access Your Account, and Reset Your Password\u00a0article. If you are experiencing difficulty with the account activation process, contact our support team for help.
\n
Disabling User Accounts and Transferring Responsibilities
\n
If you're an Account Administrator, you can disable user accounts. For example, if one of your compliance users with Contributor or Scope Administrator user roles leaves your organization, you would need to disable the user's account. When you disable a user account, you can reassign the user's responsibilities for controls, tasks, and vendors.\u00a0
\n
\n
\nNote: If you are transferring responsibilities from a Scope Administrator to another Scope Administrator, the Scope Administrator who is receiving the responsibilities should have the same Allowed Scopes as the user who you are disabling.\u00a0
\n
\n
To disable a user and transfer the user's responsibilities, follow the steps below:
\n
    \n
  1. Log in to your platform.
    2. \n
  2. Click the Settings button at the top-right corner of the page.
    3. \n
  3. From the drop-down menu, select\u00a0Manage Users.\u00a0\n\n
    4. \n
  4. From the Manage Users page, find the user you need to disable and click the disable user\u00a0icon.\n\n
    5. \n
  5. From the\u00a0Disable User page, select a user or users to transfer the user's responsibilities to. To learn about the responsibilities you can transfer, see the list below:\n
      \n
    1. \nTasks: Transfer the tasks that the user was responsible for approving or completing to the new user. If the user is the User Assigned for tasks, the new user will become the User Assigned for those tasks. If the user is the Approving Manager or Second-level Approving Manager for tasks, the new user will become the Approving Manager of Second-level Approving Manager for those tasks. Only tasks that have not been satisfied will be transferred to the new user. To learn which tasks have been satisfied and which tasks have not been satisfied, see the list below:\n
        \n
      • Tasks that have not been satisfied include the tasks in the list below:\u00a0\n
          \n
        • Tasks that have a task stage of Open and a task status of Active, Past Due, or Failed.
          • \n
        • Tasks that have a task stage of Awaiting Approval.
          • \n
        • All new tasks that will be created in the future for task schedules that the disabled user was assigned to.
          • \n
        \n
        • \n
      • Tasks that have been satisfied include the tasks in the list below:\n
          \n
        • Tasks that have a task status of Satisfied and a task stage of Approved.
          • \n
        • Tasks that have task status of Closed Late and a task stage of Approved.
          • \n
        \n
        • \n
      \n
      2. \n
    2. \nControls: Transfer the controls that the user was responsible for to the new user. You can transfer controls to users who have the Account Administrator, Scope Administrator, or Risk Administrator user roles. If you do not want to transfer the controls to another user, you can select Unassigned.
      3. \n
    3. \nVendors: Transfer the vendors that the user was the Vendor Owner for to the new user. You can transfer vendor ownership to users who have the Account Administrator or Vendor Administrator user roles.
      4. \n
    \n\n
    6. \n
  6. Click the Disable and Transfer button to confirm any responsibility transfers and disable the user account.
    7. \n
\n
\n
\nNote: If you disable a user who is a member of a user group that you've created for controls, the user will be automatically removed from the user group. To learn more, see our Working With User Groups article. However, if the user is a member of a group in the Policy Management module, they will not be removed from that group when they are disabled. To learn more, see the Step Two: Create User Groups for Policy Management Campaigns section of our Policy Management Module Guide.\u00a0\u00a0
