Depending on your system setup, phishing test emails that contain certain types of attachments may be blocked. See below for known mail servers that reject certain attachment types. We recommend using alternate file types when testing your users:
\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
Mail ServerType of Attachment Rejected
Google Suite\n
    \n
  • All attachments with macros (.ppt, .xls, and .doc)
    • \n
  • .pdf attachments
    • \n
  • .xlsx attachments
    • \n
\n
Microsoft 365\n
    \n
  • All attachments with macros (.ppt, .xls, .xlsx, and .doc)
    • \n
\n
\n
This list is not inclusive of all affected attachment types or mail servers and may change over time. Mail client and server rules change often, so your experience with attachment delivery may vary. If you would like to disable attachments with macros from being sent during a phishing campaign, you have the option to do so from the Phishing section of the Account Settings page. For more information, see this article
\n
\n
\nNote: If you use Google, setting up Direct Message Injection (DMI) can allow email attachments in these formats to be delivered. If you use Microsoft 365, setting up advanced delivery policies can allow email attachments in these formats to be delivered.
