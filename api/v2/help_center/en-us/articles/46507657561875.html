You can integrate your Make platform into your KSAT console. Make allows you to visually design and deploy event-driven, real-time automations that respond instantly to user actions, such as failed phishing tests or course completions. Make utilizes KnowBe4\u2019s Graph APIs to enable automated security awareness training workflows and data management in areas including groups, users, phishing, and training.
\n
Before You Connect Make to Your KSAT Console
\n
Before integrating Make into your KSAT console, you must find your API address and create a Product API token. Follow the steps below to do both actions.
\n
\n
\nNote:KnowBe4's Graph APIs are available to Diamond-level customers. For more information about Graph APIs, please see our KnowBe4 Graph API Overview article.
\n
\n
    \n
  1. Your API address depends on where your KnowBe4 account is located. Please see the table below to find the API address for your region.\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n\n
    RegionBase URL
    UShttps://us.api.knowbe4.com
    CAhttps://ca.api.knowbe4.com
    EUhttps://eu.api.knowbe4.com
    UKhttps://uk.api.knowbe4.com
    DEhttps://de.api.knowbe4.com
    \n
    2. \n
  2. Log in to your KSAT console as an admin.
    3. \n
  3. Navigate to Account Settings in the top-right corner under your account\u2019s email address.
    4. \n
  4. In Account Settings, navigate to Account Integrations.
    5. \n
  5. Under Product API, select Product API. The Product API subtab will open.
    6. \n
  6. At the top-right corner of the Product API subtab, select + Create New API Token.
    7. \n
  7. Fill out the fields on the Create New API Token page.
    8. \n
  8. Select Create Token. The Product API Token pop-up window will open.
    9. \n
  9. Select the Product API token to copy the token. Be sure to save your copied API token in a location that you can easily access. After you close this window, you will not be able to view this token again.
    10. \n
\n
Connecting Make to Your KSAT Console
\n
You can connect your Make platform to your KSAT console by following the steps below.
\n
    \n
  1. Log in to your Make platform as an admin.
    2. \n
  2. Select your desired organization.\n
      
    \n
    3. \n
  3. In your chosen organization, select + Create scenario.\n
      
    \n
    4. \n
  4. In the New scenario section, search for KnowBe4 under All Apps. Select the KnowBe4 app.\n
      
    \n
    5. \n
  5. Once you select the KnowBe4 app, you should be able to view 37 actions.\n
      
    \n
    6. \n
  6. Select any of the actions you would like to take.
    7. \n
  7. Select Create a connection.\n
      
    \n
    8. \n
  8. A pop-up window should allow you to enter the Product API token and API address you received in the previous steps. Once you have established a connection, it can be used for other Make modules.\n
      
    \n
    9. \n
\n
Setting Up KnowBe4 Workflows in Make
\n
You can set up KSAT workflows in your Make platform by following the steps below.
\n
    \n
  1. In your Make organization, navigate to Scenarios > All Apps > KnowBe4. Select an action. In this example, we will select Create a User.\n
      
    \n
    2. \n
  2. Provide the required user information in the pop-up screen.\n
      
    \n
    3. \n
  3. Select Save to save the user information.
    4. \n
  4. In the lower-left corner of the page, select Run once.\n
      
    \n
    5. \n
  5. The user\u2019s information should now be visible in KSAT. In your KSAT console, the user\u2019s email address should be visible by navigating to Users > Manage Users > Users.\n
      
    \n
    6. \n
  6. If you would like to run multiple modules in Make, you can also connect them and select Run once.\n
      
    \n
    7. \n
\n
If you experience issues integrating Make with your KSAT console, please contact Make\u2019s technical support.
