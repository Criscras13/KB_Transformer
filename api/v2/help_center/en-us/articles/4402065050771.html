In this article, you\u2019ll learn how to configure SCIM for Okta. Configuring SCIM for Okta allows you to use Okta to manage users in your KSAT console. For information on how to enable SCIM for your KSAT console, see our SCIM Configuration Guide.
\n
The instructions below are for third-party software. If you experience issues with user provisioning in Okta, we recommend reaching out to Okta for specific instructions. You can also contact our support team and we will be happy to assist you.
\n
\n
\nImportant:Some customers may not have the ability to enable SCIM provisioning on custom apps. If the option does not display, you will need to contact Okta support to have them enable this feature for your organization. For more information on this, see Okta's Add SCIM provisioning to app integrations article.
\n
\n
Configuring SCIM
\n
After you have configured your SCIM settings in your KSAT Account Settings, you are now ready to configure SCIM for Okta. To configure SCIM for Okta, follow the steps below:
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click Browse App Catalog.\u00a0
    3. \n
  3. In the search bar, enter \u201cKnowBe4\u201d to filter the results.
    4. \n
  4. Select the KnowBe4 app.
    5. \n
\n
\n
\nNote: If you already have a KnowBe4 SAML app in Okta, you can leave this app as-is.
\n
\n
\n
\nImportant: If you\u2019re adding the provisioning capability to an already existing custom SAML app, you\u2019ll need to provision the existing users before they can log in. To provision existing users, you\u2019ll need to go to the Assignments tab on the app\u2019s configuration page and select Provision User.
\n
\n
    \n
  1. Click Add Integration. \n
    2. \n
  2. Edit the name and settings, if you would like.
    3. \n
  3. Click Done.
    4. \n
  4. Navigate to the Provisioning tab.
    5. \n
  5. Click the Integration subsection.
    6. \n
  6. Click Configure API Integration.
    7. \n
  7. Select the Enable API integration check box.
    8. \n
  8. Paste the Tenant URL from your KSAT Account Settings into the SCIM connector base URL field. To learn how to access your Tenant URL, see the Configuring SCIM section of our SCIM Configuration Guide.
    9. \n
  9. Paste the SCIM token from your KSAT Account Settings into the API Token field. To learn how to access your SCIM Token, see the Configuring SCIM section of our SCIM Configuration Guide.
    10. \n
  10. Click\u00a0Test API Credentials.
    11. \n
  11. A message will display to notify you whether the test succeeded or failed. If the test succeeded, click Save. \n
    12. \n
  12. Now that you have set up the connection between your KSAT console and Okta, you can enable the services that you want to manage through Okta. To get started, click To App.
    13. \n
  13. Click Edit on the right side of the Provisioning to App section.
    14. \n
  14. Select the Enable check box for each feature that you would like to use. \n
    15. \n
\n
\n
\nImportant: Pushing groups in Okta has some limitations that may affect your SCIM operation. For more information on this issue, see Okta\u2019s About Group Push article.
\n
\n
\n
\nImportant: We do not currently support Sync Password. Any passwords that are sent with this setting will not be accepted.
\n
\n
After you have configured SCIM for Okta, you will need to choose which users to sync. To learn more about syncing users through Okta, see the Defining Which Users and Groups to Sync section below.
\n
Defining Which Users and Groups to Sync
\n
After you have followed the steps in the Configuring SCIM section above, you can define which users and groups you would like to sync. Defining which users and groups to sync is required before you can sync users from your identity provider.
\n
\n
\nImportant:If you define groups, we recommend leaving additional group attributes blank. If you leave these additional attributes blank, individual users' attributes will be synced in place of the blank attributes. If you set up a group attribute for the following fields, the group attribute will override any individual user attributes for the users assigned.
\n
\n
To define which users and groups to sync, follow the steps below:
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click the SCIM application that you created in the Configure SCIM section above.
    3. \n
  3. Navigate to the Assignments tab.
    4. \n
  4. Click Assign to select which users you would like to sync.
    5. \n
  5. Click either Assign to People or Assign to Groups, depending on whether you want to define users or groups. \n
    6. \n
  6. Select the users or groups that you would like to sync.
    7. \n
\n
\n
\nImportant: For your first sync, we recommend that you only select a few users or groups.
\n
\n
    \n
  1. Click\u00a0Assign.
    2. \n
  2. After you select the user or group that you would like to sync, click Save and Go Back.
    3. \n
  3. After you\u2019ve added all the users and groups that you would like to include, click Done.
    4. \n
\n
Any users and groups you selected will now display in the Assignments tab.
\n
Defining Which Groups to Sync
\n
To sync groups and group memberships from Okta to your KSAT console, follow the steps below.
\n
\n
\nImportant:We do not currently support the Push now button. Clicking this button may remove memberships from selected groups.
\n
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click the SCIM application that you created in the Configure SCIM section above.
    3. \n
  3. Click Push Groups. \n
    4. \n
\n
\n
\nImportant: Okta does not support using the same groups for assignments and group push. For more information, see Okta\u2019s About Group Push article.
\n
\n
    \n
  1. From the drop-down menu that opens, select Find groups by name.
    2. \n
  2. Enter the names of groups you would like to sync.
    3. \n
  3. Click Save.
    4. \n
\n
Attribute Mappings
\n
In Okta, there are attribute mappings that you can customize in order to define which fields sync between Okta and your KSAT console. To modify these attribute mappings, follow the steps below:
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click the SCIM application that you created in the Configure SCIM section above.
    3. \n
  3. Click Provisioning.
    4. \n
  4. Click To App.
    5. \n
  5. Scroll down to Attribute Mappings.
    6. \n
  6. Make the changes that you would like to make.
    7. \n
  7. Save your changes.
    8. \n
\n
\n
\nImportant:We recommend that you download a CSV file from the Users tab in your KSAT console to have a backup of all user field info in case of an unexpected error.
\n
\n
You may have fields in your KSAT console that you don\u2019t want to update from Okta. As a best practice, we recommend that you remove these attribute mappings so that they aren\u2019t updated during an Okta sync.
\n
For more information about Okta attribute mappings, see the Advanced Configuration Options section below.
\n
Starting Your Sync
\n
After you\u2019ve configured your SCIM settings and added the users and groups that you want to sync, you can start the sync. After you\u2019ve started the first sync, syncs from Okta will occur automatically. You can also manually force a sync from your Okta portal at any time.
\n
\n
\nNote:\u00a0If you have more than several thousand users in your SCIM provisioning application, it\u2019s likely all of your users won't be included in your initial sync. Instead, the users will be synced to your account in stages. We recommend that you keep user provisioning in Test Mode until you see only a few changes between your sync reports. Waiting until you only see a few changes helps prevents users from being archived in your KSAT console. Additionally, syncing group memberships can take longer than syncing users. If you have a larger account, you can expect to see periodic syncs in your KSAT console.
\n
\n
To start your sync, follow the steps below:
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click the SCIM application that you created in the Configure SCIM section above.
    3. \n
  3. Click Provisioning.
    4. \n
  4. Click Force Sync. \n
    5. \n
\n
The sync will be initiated immediately. After your initial sync, syncs will occur automatically when you change user information in Okta.
\n
\n
\nImportant:Once you are satisfied that your users have synced correctly, you\u2019ll need to turn off Test Mode in your KSAT Account Settings. Turning off Test Mode will allow users to be added and archived during the next sync. For more information about Test Mode, see our SCIM Configuration Guide.
\n
\n
Once your sync has started, you can view the sync status and learn about any errors from the Provisioning tab in your KSAT console. To learn more about the Provisioning tab, see our How to Use the Provisioning Tab article.
\n
Advanced Configuration Options
\n
You can customize your Okta configuration by changing default field mappings, adding custom attributes, or mapping custom KnowBe4 fields. For more information about customizing your Okta configuration, see the subsections below.
\n
\n
\nImportant:Email aliases are not currently supported by SCIM provisioning.\u00a0

Mapping Custom Fields
\n
You also have the option to map custom fields to sync with your KSAT console.
\n
These fields are not mapped by default, but you can add them to your Okta platform by following the steps below:
\n
    \n
  1. Log in to your Okta portal and navigate to Applications.
    2. \n
  2. Click the SCIM application that you created in the Configure SCIM section above.
    3. \n
  3. Click Provisioning.
    4. \n
  4. Scroll down and click on Show Unmapped Attributes.
    5. \n
  5. Click the pencil icon next to any of the attributes that you would like to add. \n
    6. \n
  6. From the Attribute value drop-down menu, select the Okta attribute that you would like to map to each custom field. \n
    7. \n
  7. Click\u00a0Save.
    8. \n
\n
\n
\nNote: If you are configuring a Custom Date attribute, the date must be formatted in ISO 8601 format. The format is as follows: YYYY-MM-DD \u201cT\u201d hh:mm:ssZ. For example, 2022-04-04T04:23:30Z.
\n
