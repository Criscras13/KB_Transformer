If you have board members or employees who do not have email addresses at your domain and you need to assign them security awareness training, you can do this within your KSAT console by using dummy email addresses for those users. For example, if you had a board member named John Smith, their dummy email address would be something like John.Smith@example.com.
\n
Keep in mind the domain that you use in this dummy email address must either be your KnowBe4 account's primary domain or a domain that has been allowed for your account. Email addresses of publicly-available email services like Hotmail and Gmail are not allowed.
\n
Adding Your Users
\n
To set up training for your users who do not have email addresses from your domain, you can use two methods: using quick import and importing a CSV file. Both of these methods are discussed in the next two sections of this article.
\n
\n
\nImportant:If you are using Active Directory Integration (ADI), users with dummy email addresses will be removed during your next AD sync unless you prevent those users from being managed by AD. To prevent users from being managed by AD, navigate to each user and click Edit. Then, select click here in the green banner at the top of the page.\u00a0
\n
\n
Adding Users Through Quick Import
\n
In order to add users through quick import, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to Users > Import Users.
    3. \n
  3. Click Quick Import.
    4. \n
  4. In the box below the Quick Import button, paste a list of your dummy email addresses, one per line, with no commas or spaces. Remember, these do not have to be real email addresses from your domain. Think of it more as a login so your users can access training.
    5. \n
  5. Select the Password for Users check box.\n
    \n
    \nNote: You can select the Require users to reset initial password set by admins check box to require users to change their set passwords after their first login.
    \n
    \n
    6. \n
  6. Enter a basic password for your users in the field below the check box.There is no sensitive data on the training module page, so this password does not need to be complex. However, you can advise your users to change their password upon login if you wish.\u00a0\n
    \n
    \nImportant:You must set the password here in order for your users to be able to log in.
    \n
    \n
    7. \n
  7. Click Import Users.\u00a0
    8. \n
  8. After you import the users, include them in a training campaign and instruct them to log into their accounts with the email and password you created.
    9. \n
  9. Send a link to your KnowBe4 training login page. See our KnowBe4's Training Instances article for a full list of login links.\n
      \n
    • If you have Security Assertion Markup Language (SAML) enabled on your account, add the subdirectory /ui/users/login. For example, https://training.knowbe4.com/ui/users/login.
      • \n
    \n
    10. \n
\n
Adding Users by CSV File
\n
To add users with a CSV file, follow the steps below:
\n
    \n
  1. Log in to your KSAT console.
    2. \n
  2. Navigate to Users > Import Users.
    3. \n
  3. Click CSV Import.
    4. \n
  4. In your CSV file, include two fields: \"Email Address\" and \"Password\". Under the Email Address field, include the users' dummy email addresses line by line. Under the Password field, set a basic password for each of your users. The password you set for them does not need to be complex, and it can be the same password for all your users. However, you should advise your users to change their password upon login.\n
    \n
    \nImportant:You must set the password here in order for your users to be able to log in. Set the Provisioning Managed header to False to ensure no accounts are archived. For more information about the Provisioning Managed field, see our How to Import Users with a CSV File article.
    \n
    \n
    \n
    \nNote:You can select the Require users to reset initial password set by admins check box to require users to change their set passwords after their first login.
    \n
    \n
    5. \n
  5. Import your CSV file.
    6. \n
  6. After you import the users, include them in a training campaign and instruct them to log into their accounts with the email and password you created.
    7. \n
  7. Send a link to your KnowBe4 training login page. See our KnowBe4's Training Instances article for a full list of login links.\n
      \n
    • If you have Security Assertion Markup Language (SAML) enabled on your account, add the subdirectory /ui/users/login. For example, https://training.knowbe4.com/ui/users/login.
      • \n
    \n
    8. \n
\n
Preventing Bounces
\n
If you've imported one or more email addresses into your account that don't exist in your mail server and you conduct a phishing test for all users, your test will have multiple bounces. This will result in skewed campaign results for the Phish-prone Percentage since that this number is calculated based on the total number of intended recipients. To prevent bounces and skewed campaign results, you should use groups for your Phishing Security Tests.
\n
We recommend creating a group that contains all of your users with real email addresses. You can name this group something like \u201cPhishable Users\u201d or \u201cReal Emails\u201d.\u00a0 After setting up this group, when you create a Phishing Security Test, select this group rather than All Users.
\n
\n
\nNote: Since training notifications and emails cannot be sent to the users with dummy addresses from the KSAT console, admins will have to set up an alternate method to send the training information to those users.
