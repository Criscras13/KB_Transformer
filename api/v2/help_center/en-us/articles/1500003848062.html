\nNote: We\u2019ve added advanced features to phishing templates! All accounts will be opted in to phishing template advanced features in 2025. For more information, see our Phishing Templates Advanced Features Guide.
When creating a phishing campaign, you have the option to use one of our Automated Template Selections to automatically select the templates used in your campaign. Automated template selections choose templates from topics you select and add variety to recurring phishing campaigns.
The three automated template selection options are AI-selected (Diamond-level customers only), Full Random, and Random. Click on the tabs below to learn more about each option.
\n
\n
AI-selected Phishing Templates
\n
Diamond-level customers have the option to use AI-selected phishing templates. This feature uses data from KnowBe4\u2019s Artificial Intelligence Defense Agents (AIDA) to select the most relevant and challenging template for each user. AI-selected templates are chosen based on a user\u2019s training history, phishing events, and performance metrics, such as their Phish-prone percentage and Security Awareness Proficiency Assessment (SAPA) results. The more data AIDA has, the better it works, so we recommend using these templates for users who have some prior training or phishing history.
\n
To avoid repetition in recurring campaigns, the AI-selected feature remembers the last five emails sent to each user and selects a different template for subsequent tests.
\n
When using AI-selected templates, choose multiple topics to ensure you have a minimum of 20 templates. Providing a large number of templates for AIDA to choose from allows for more variety in your campaign. For best results, we recommend using 50 templates or more.
\n
\n
\n
\n
Full Random (Random Email to Each User)
\n
The Full Random option automatically chooses different templates for each user by pulling from the selected topics. Users enrolled in the campaign will receive a variety of different emails. Using Full Random is a great way to prevent users from telling each other about an incoming phishing test. For an even more effective test, we recommend sending the emails over a period of time.
\n
\nNote: To prevent users from getting the same phishing templates too often, the Full Random option skips the five most recently used phishing templates that were sent to each user.
\n
\n
\n
\n
Random (Same random email to all users)
\n
The Random option will automatically choose one template from the selected topics. All users enrolled in this campaign will receive the same email. To help prevent users from telling each other about this phishing test, we recommend sending all emails when the campaign starts.
\n
To avoid repetition in recurring campaigns, this feature remembers the last five emails sent and selects a different template for subsequent tests.
\n
\n
\n
For more information about phishing campaigns, see our Creating and Managing Phishing Campaigns article.
