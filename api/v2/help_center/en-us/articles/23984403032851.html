In this article, you\u2019ll learn how to integrate COZYROC SSIS+ with your KSAT console. This integration pulls campaign and user data available via our reporting API into the COZYROC platform to migrate data from one place to another.
\n
\nNote:Before you set up this integration, you'll need to copy the Reporting API Token, User Event API Token, and Product API Token from your KSAT Account Settings page. For more information, see the API section of our KSAT Account Settings: Account Integrations article.
\n\n\n
To learn how to create a connection to KnowBe4\u2019s REST API using COZYROC's REST Connection Manager, follow the steps below.
\n
    \n
  1. On your Windows computer, navigate to Start > Visual Studio.
    2. \n
  2. Right-click Visual Studio, then select More > Run as administrator.
    3. \n
  3. Right-click on the Connection Managers tab and select New Connection.
    4. \n
  4. In the Add SSIS Connection Manager pop-up window, scroll down and select Connection manager for REST connections.
    5. \n
  5. Click Add\u2026.\n\n
    6. \n
  6. In the KnowBe4 Connection Manager pop-up window, select KnowBe4 from the Configuration drop-down menu.
    7. \n
  7. In the Server field, enter the server URL for your KnowBe4 instance using the following format: \u201chttps://[server].api.knowbe4.com\u201d. For example, accounts on our EU server located at eu.knowbe4.com, must use the https://eu.api.knowbe4.com server URL.\n
    \nNote:For more information about KnowBe4\u2019s API usage and URLs, see our Developer Documentation.
    \n\n
    8. \n
  8. Based on the APIs you would like to connect to, provide the values for the Events_Token, Reporting_Token, or PhishER_Token fields. For more information about these tokens, see the list below:\n
      \n
    • \nEvents_Token: Use this parameter for creating custom events for users in KnowBe4. If you want to use this parameter, paste your User Event API Token in this field.
      • \n
    • \nReporting_Token: Use this parameter for pulling reporting data out of KnowBe4. If you want to use this parameter, paste your Reporting API Token in this field.
      • \n
    • \nPhishER_Token: Use this parameter for pulling messages from KnowBe4\u2019s PhishER SOAR. If you want to use this parameter, paste your Product API Token in this field.
      • \n
    \n
    9. \n
  9. Click Test Connection to test your API connection.\n\n
    10. \n
  10. When you\u2019re satisfied with your connection, click OK.
    11. \n
\n
You\u2019ve successfully established an API connection to your KnowBe4 instance.
\n\n\n\n
To learn how to read your API data from your COZYROC and KnowBe4 connection, follow the steps below.
\n
    \n
  1. On your Windows computer, navigate to Start > Visual Studio.
    2. \n
  2. Right-click Visual Studio, then select More > Run as administrator.
    3. \n
  3. Select Data Flow, then click the link to create a new Data Flow task.\n\n
    4. \n
  4. Select SSIS Toolbox, then drag the REST Source component onto the Data Flow page.\n\n
    5. \n
  5. Double-click on the REST Source component, then select KnowBe4 from the Connection drop-down menu.
    6. \n
  6. From the Resource drop-down menu, select one of the API parameters that you provided information for earlier.\n\n
    7. \n
  7. Click Columns, then select the KnowBe4 resource columns you want to read.\n\n
    8. \n
  8. Select Preview to ensure all necessary resource columns are included.\n\n
    9. \n
  9. Click\u00a0OK.\u00a0
    10. \n
\n
\nNote:For more information about this connection, see COZYROC\u2019s KnowBe4 Connection article.
