In this article, you'll learn how to configure SCIM with Microsoft Entra ID (formerly Azure Active Directory). Configuring SCIM for Microsoft Entra ID will allow you to add and manage users and groups in your KSAT console using Microsoft Entra ID.
\n
The instructions in this article are for third-party software. If you experience issues with user provisioning in Microsoft Entra ID, we recommend reaching out to Microsoft Entra for specific instructions. You can also contact our support team and we will be happy to assist you.
\n
\n
\nNote:To sync users and groups with SCIM, you must have a Microsoft Entra subscription. For more information about syncing users and groups through Microsoft Entra, see Microsoft's Assign users and groups to an application article.
\n
\n
Configuring SCIM
\n
In this section, you'll learn how to configure your SCIM settings with Microsoft Entra. Please note that you should configure these steps after you've configured your settings in your KSAT console. For more information about configuring SCIM in your KSAT console, see our SCIM Configuration Guide.
\n
To configure your SCIM settings with Microsoft Entra, follow the steps below:
\n
    \n
  1. Log in to your Azure Portal at https://portal.azure.com/.\n
    2. \n
  2. Select the Manage Microsoft Entra ID tile\u2019s View button from the Azure Portal\u2019s home page.\n
      
    \n
    3. \n
  3. From the + Add drop-down menu, select Enterprise application.
    4. \n
  4. In the Microsoft Entra Gallery\u2019s search bar, enter \u201cKnowBe4\u201d to filter your results.\n
      
    \n
    5. \n
  5. Select the KnowBe4 Security Awareness Training tile.
    6. \n
  6. Select Create from the pop-up window that appears on the right side of the screen.\n
      
    \n
    7. \n
  7. After you click Create, you'll be redirected to the Overview page for the application that you created.\n
      
    \n
      \n
    1. If you are not directed to the Overview page, you'll need to open the application from the list of Enterprise applications. You can find Enterprise applications by clicking on All services in the left-most panel, then Recents from the accordion menu that pops up, then Enterprise applications.\n
        
      \n
      2. \n
    \n
    8. \n
  8. Select the Get started link in the Provision User Accounts tile underneath the Getting Started header.\n
      
    \n
    9. \n
  9. Select + New configuration.\n
      
    \n
    10. \n
  10. \n
    Next, you'll need to enter the information from your Account Settings page. Read more about where you can find this information in our SCIM Configuration Guide. In the Tenant URL field, enter the Tenant URL. In the Secret Token field, enter the SCIM Token.
    \n
    \n
    \nImportant:This feature does not currently work with on-demand provisioning.

    11. \n
  11. After you\u2019ve entered your information, select the Test Connection button. Clicking this button will allow you to ensure that you entered the correct information. If the connection is successful, a success banner will display at the top-right corner of your screen.\n
    \n
    12. \n
  12. Select the Save button at the top of the screen.
    13. \n

Defining Which Users and Groups to Sync from Microsoft Entra
\n
\n
\nNote:The instructions in this section define how specific users and groups should sync. If you would like to sync all your users and groups from Microsoft Entra ID, see the Frequently Asked Questions (FAQ) section of this article.
\n
\n
After completing the steps in the Configuring SCIM section above, you can decide which users and groups you would like to sync. This configuration is required in order to sync users and groups from your identity provider (IdP).
\n
\n
\nImportant:Nested groups are not currently supported by SCIM and Microsoft Entra ID provisioning. For more information, see the Scoping section of Microsoft\u2019s How Application Provisioning works in Azure Active Directory article.
\n
\n
To define which users and groups you would like to sync from Microsoft Entra ID, follow the steps below:
\n
    \n
  1. From your Microsoft Entra ID, navigate to Enterprise applications.
    2. \n
  2. Select the application you created for your KnowBe4 connection.
    3. \n
  3. Select the Assign users and groups from the Assign users and groups tile.\n
      
    \n
    4. \n
  4. \n
    Select + Add user/group to select the users or groups that you would like to sync.
    \n
    5. \n
  5. Select None Selected to search for users or groups that you would like to include in your sync. To add a user or group, click on the name of the user or group. They will now display in the Selected side panel.\n
      
    \n
    \n
    \nNote:We recommend that you only include a few users when you first configure your settings. Starting with a few users allows you to ensure that the connection works properly before adding all the users and groups you want to include.
    \n
    \n
      
    \n
    6. \n
  6. After you\u2019ve added the users and groups you want to include to the Selected category, click Select.
    7. \n
  7. Select Assign.
    8. \n
\n
The users and groups that you selected will now display in the table.
\n
Starting Your Sync
\n
After you have configured SCIM and have added the users and groups that you want to sync, you'll need to start the sync. Once you start the sync, the system will automatically check for changes to your users and groups in Microsoft Entra ID every 40 minutes and will initiate a sync if changes were made.
\n
\n
\nNote:If you have more than several thousand users in your SCIM provisioning application, it\u2019s likely all of your users won't be included in your initial sync. Instead, the users will be synced to your account in stages. We recommend that you keep user provisioning in Test Mode until you see only a few changes between your sync reports. Waiting until you only see a few changes helps prevents users from being archived in your KSAT console. Additionally, syncing group memberships can take longer than syncing users. If you have a larger account, you can expect to see periodic syncs in your KSAT console.
\n
\n
To start your sync, follow the steps below:
\n
    \n
  1. From your Microsoft Entra ID, navigate to Enterprise applications.
    2. \n
  2. Select the application that you created for your KnowBe4 connection.
    3. \n
  3. From the menu on the left side of the page, select Provisioning.
    4. \n
  4. Click Start provisioning.\n
    \n
    5. \n
\n
The sync will be initiated immediately. After your initial sync, the system will check for changes to your Microsoft Entra ID every 40 minutes and will initiate a sync if changes were made.
\n
\n
\nImportant:Once you are satisfied that your users have synced correctly, you\u2019ll need to turn off Test Mode in your KSAT Account Settings. Turning off Test Mode will allow users to be added and archived during the next sync. For more information about Test Mode, see our SCIM Configuration Guide.
\n
\n
To see the status of these syncs as well as any errors and additional information about your syncs, navigate to Users Provisioning in your KSAT console.
\n
\n
Advanced Configuration Options
\n
When enabling SCIM synchronization with Entra ID, existing user data in KnowBe4 will be overwritten by the SCIM identity provider. Before proceeding, verify that groups and user statuses are correctly configured in Entra ID to prevent unintended data loss. Users not included in the synchronization will be automatically archived, preserving their data and maintaining historical records.
\n
\n
\nImportant: Email aliases are not currently supported by SCIM provisioning.
\n

Changing the Default Mappings
\n
You can change the default mappings to customize the user information that syncs between Microsoft Entra and your KSAT console.
\n
To change the default mappings, follow the steps below:
\n
    \n
  1. From your Microsoft Entra ID, navigate to Enterprise applications.
    2. \n
  2. Select the application you created for your KnowBe4 connection.
    3. \n
  3. From the menu on the left side of the page, select Provisioning.\n
    \n
    4. \n
  4. From the Provisioning window, click Edit attribute mappings under Manage provisioning.\n
    \n
    5. \n
  5. Click the Mappings drop-down arrow to expand the Mappings tab.\n
    \n
    6. \n
  6. Click Provision Azure Active Directory Users.
    7. \n
  7. Scroll down to the Attribute Mappings section. From this section, you'll see a list of all the attributes that have been mapped. The Azure Active Directory Attribute column displays the name of the attribute in Microsoft Entra. The KnowBe4 Attribute column displays the SCIM standard name for this attribute.\n
    \n
    8. \n
  8. Select the attribute you would like to edit.
    9. \n
  9. In the Edit Attribute side pane, customize the attribute. For details about the customization options, see the list below:\n
      \n
    1. \nMapping type: Select Direct from the drop-down menu.
      2. \n
    2. \n
      Source attribute: Select the Azure field that you want to map to this custom field.
      \n
      \n
      \nNote:If you're using SSO for Microsoft Entra ID, this attribute should be the same as the SSO Source attribute. By default, the SSO Source attribute is user.userprincipalname. For more information, see Add the KnowBe4 Application to Azure AD section of our How Do I Configure SSO/SAML with Azure Active Directory (AD)? article.
      \n
      \n
      3. \n
    3. \nDefault value if null: This field is optional, and we recommend that you leave it blank.
      4. \n
    4. \nTarget attribute: Select the custom field that you want to map to the Azure field you selected.
      5. \n
    5. \nMatch objects using this attribute: We recommend you select No.
      6. \n
    6. \n
      Apply this mapping: We recommend you select Always.
      \n
      \n
      \nNote:If there is an attribute you don\u2019t want to sync, you can click the Delete button next to that attribute to disable syncing. This action will only remove the connection between this attribute and the corresponding field in your KSAT console. No data will be deleted from Azure.
      \n
      \n
      \n
      7. \n
    \n
    10. \n
  10. \n
    Once you have made the changes you would like to make, click Ok.
    \n
    \n
    \nNote:We recommend that you only change the Source attribute field. Changing the other settings on the attribute may break the connection between Microsoft Entra and your KSAT console.

Frequently Asked Questions (FAQs)
\n
Below is a list of frequently asked questions about using SCIM with Microsoft Entra ID.
\n
How often do syncs occur?
\n
The system will check for updates to the users and groups in your Microsoft Entra ID every 40 minutes. If changes are found, a sync will begin automatically. However, you can force a sync at any time by clicking the Force Sync Now button in the SCIM Settings section of your KSAT Account Settings.
\n
How do you restore the default mappings?
\n
You can restore the default mapping at any time by following the steps below:
\n
    \n
  1. Navigate to Enterprise applications.
    2. \n
  2. Select the application you created for your KnowBe4 connection.
    3. \n
  3. From the menu on the left side of the page, select Provisioning.
    4. \n
  4. Click Edit attribute mapping under Manage provisioning.
    5. \n
  5. Click the Mappings drop-down arrow to expand the Mappings drop-down menu.
    6. \n
  6. Select Restore default mappings.
    7. \n
  7. Click Save at the top of the screen.
    8. \n
\n
How do I sync all my users and groups?
\n
If you would like to sync all users and groups from your Microsoft Entra ID, follow the steps below:
\n
    \n
  1. Navigate to the application you set up for your SCIM connection.
    2. \n
  2. Navigate to Provisioning.
    3. \n
  3. Select Edit provisioning at the top of the screen or select Add scoping filters under Manage provisioning.
    4. \n
  4. Click the Settings drop-down menu.
    5. \n
  5. From the Scope drop-down menu, select Sync all users and groups.
    6. \n
  6. Click Save at the top of the page.
    7. \n
\n
I don\u2019t have the ability to assign users to an application by group. How can I limit the users being synced to my KSAT console?
\n
To limit the users being synced to your KSAT console, you can set up a scoping filter. For more information about making a scoping filter, see Microsoft's Attribute-based application provisioning with scoping filters article.
