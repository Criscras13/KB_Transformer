\u00a0A USB drive test allows you to test your users on how they will react to finding an unknown removable storage device, such as a thumb drive.
\n
If you navigate to Physical Tests >\u00a0USB in your KSAT console, you can download test files and copy them to universal serial bus (USB) drives. Once the USB drives are set up, you can place the USB drives in planned locations. Then, you can use your KSAT console to see if your users are opening the test files. You can also see information such as where your users are opening the test files.
\n
Alternatively, you can watch our USB Drive Test video.
\n\n
USB Drive Tests Page
\n
From the USB Drive Tests page of your KSAT console, you can view and manage all existing USB drive tests.
\n\n
    \n
  1. \n+ Create New USB Drive Test: You can click this button to go to the Create New USB Drive Test page and add details for a new test.
    2. \n
  2. \nDownload CSV Report: You can click this button to download a report that shows details for all of your USB drive tests. This report will display each time a test file was opened, when macros were enabled, the IP address location for each failure, and more.
    3. \n
  3. \nCampaign Name: This column displays each USB drive test. To view more details about a test, click the test's name.
    4. \n
  4. \nUSB Location, Total File Opens, Total Macros Enabled, Started on, and Completed on: These columns display the number of times a file was opened, the number of times macros were enabled, when the test was started, and when the test was completed.
    5. \n
  5. \nStatus: This column displays the status of each USB drive test. For more information about the test statuses, see the list below:\n
      \n
    • \nCreated: This status displays when the test has been created but has not started yet.
      • \n
    • \nStarted: This status displays when the test has started and is in progress.
      • \n
    • \nCompleted: This status displays when the test has ended.
      • \n
    \n
    6. \n
  6. \nActions: This column displays a drop-down menu with more actions to manage your USB drive test. For more information about the test statuses, see the list below:\n
      \n
    • \nEdit: Click this button to change information about your test, such as the name or location.
      • \n
    • \nDelete: Click this button to delete your test.

\n\n\n
Creating a USB Drive Test
\n
To create a USB drive test, you'll need one or more USB drives and the ability to reach the KnowBe4 USB server domain.
\n
Keep in mind that you may not be able to physically reclaim the USB drive again once your test begins.
\n
The KnowBe4 USB server domain is the instance where your console is hosted, followed by the subdirectory /usb_campaigns/report/. For example, the United States USB server is https://training.knowbe4.com/usb_campaigns/report/. See our KnowBe4's Training Instances article for more information.
\n
\nImportant:If you do not configure the necessary connection to the above page, then data sent from the test files may be blocked and will not report any results to your KSAT console.
\n
When you're ready to start a USB drive test, navigate to Physical Tests >\u00a0USB in your KSAT console. Then, click + Create New USB Drive Test to create a new test.
\n
Give your test a name, specify the location where you will leave the device, and add any notes that could be helpful for you to track. Click Create USB Drive Test once all details are entered.
\n\n
    \n
  1. \nName: Enter a name for your USB Drive Test.
    2. \n
  2. \nLocation: Enter the location where you will leave the USB drive.
    3. \n
  3. \nNotes: Enter any details about this test, such as whether the test will be started before or after users have received training on USB drive safety.
    4. \n
  4. \nCreate USB Drive Test: Once you have added all the information you want, click\u00a0Create USB Drive Test.
    5. \n
\n\n\n\n
Starting a USB Drive Test
\n
Once you have created a USB drive test, you\u2019ll be redirected to the test Overview page. You can also access this page by navigating to Physical Tests >\u00a0USB in your KSAT console and selecting the test\u2019s name. This page lists four steps to start your USB drive test.\u00a0
\n
See the sections below for more information about these steps.
\n
    \n
  1. From the Download Test File column, click the name of a test file to download it. You can download and use multiple test files in a single USB drive test.\n\n
    2. \n
  2. Copy the files to a USB drive. You can copy test files to multiple USB drives in a single USB drive test.\n
    \nImportant:We encourage you to rename these files before saving them to your USB drive. We recommend using filenames that entice your users to open them, such as \u201cSalaries2022.xlsx\u201d or \u201cBoardMeetingNotes.docx\u201d.
    \n
    3. \n
  3. At the bottom of the page, click Start Test.\n\n
    4. \n
  4. Place the USB drives in your desired location.
    5. \n
\n
\nNote:Users must have the appropriate programs installed on their computers to open and run the embedded scripts in the test files. Without a program to open the files, the test data cannot be sent back to your console.
\n\n\n
Monitoring a USB Drive Test
\n
To check the details of a USB drive test, navigate to Physical Tests > USB in your KSAT console. Then, click the name of the test you'd like to monitor. The Overview page displays the number of times a file has been opened, the number of times macros have been enabled, and a map of IP address locations where the failures occurred.
\n\n
On the Details subtab, you can view a table that\u00a0displays each time a test file was opened, when macros were enabled, and the IP address location for each failure. If macros were enabled by a user, the table will also display the user's username, display name, and computer name.
\n\n\n\n
Ending a USB Drive Test
\n
There is no specified duration for how long a USB drive test will run. When you're ready to stop monitoring the failures from a test, click End Test on the test overview page.Once you have ended a test, the test will not be updated with new data on opened files or enabled macros. If you would like to continue a test, click\u00a0Restart Test\u00a0on the test overview page.
