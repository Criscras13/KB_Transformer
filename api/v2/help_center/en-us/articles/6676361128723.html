In this article, you will learn how to integrate SonicWall Capture Client with SecurityCoach. Once you set up this integration, data provided by SonicWall will be available under the SecurityCoach tab of your KSAT console. This data can be viewed in SecurityCoach reports and used to create detection rules for real-time coaching campaigns.
\n
\n\n
For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n
Create a SonicWall Capture Client Admin Account
\n
Before you can set up the integration in your KSAT console, you will need to create a SonicWall Capture Client admin account. To create your admin account, follow the steps below:
\n
\n
\nNote:If your organization already has a SonicWall Capture Client admin account, you can skip the steps in this section and follow the steps in the Obtain Your SentinelOne API Key\u00a0section of this article.
\n
\n
    \n
  1. From your browser, navigate to your SonicWall Capture Client console and click Login with MySonicWall. \n
    2. \n
  2. Navigate to Management > Administrators. \n
    3. \n
  3. Click the + sign on the right to open the Administrators window. \n
    4. \n
  4. Enter the desired email, name, and password for your new admin account.
    5. \n
  5. Click Create. \n
    6. \n
\n
Obtain Your SentinelOne API Key
\n
Before you set up the integration in your KSAT console, you will need to obtain your SentinelOne API key. To obtain your API key, create a support ticket from your SonicWall console requesting your SentinelOne API key from the SonicWall support team.
\n
Set Up the Integration in Your KSAT Console
\n
Once you've created your SonicWall Capture Client admin account and obtained your SentinelOne API key, you can set up the integration in your KSAT console. To set up the integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and navigate to SecurityCoach > Setup.
    2. \n
  2. In the Available Integrations section, locate the SonicWall Capture Client card.
    3. \n
  3. At the bottom of the card, click Configure. \n
    4. \n
  4. In the User Name field, enter the email address from the admin account you created in the Create a SonicWall Capture Client Admin Account section of this article.
    5. \n
  5. In the Password field, enter the password from the admin account you created in the Create a SonicWall Capture Client Admin Account section of this article.
    6. \n
  6. In the URL field, enter your SonicWall Capture Client console URL. Your SonicWall Capture Client console URL will be similar to \u201chttps://captureclient-01.sonicwall.com\u201d.
    7. \n
  7. In the API Key field, enter your SentinelOne API key.
    8. \n
  8. Click Authorize. \n
    9. \n
\n
Map Your Users
\n
After you\u2019ve finished integrating SonicWall Capture Client, you can map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Mapping Users in SecurityCoach article.
\n
Once you\u2019ve successfully authorized this integration, you can manage detection rules for SonicWall Capture Client on the Detection Rules subtab of SecurityCoach. For a full list of available system detection rules for this vendor, see our Which Detection Rules Can I Use with My Vendors? article.
