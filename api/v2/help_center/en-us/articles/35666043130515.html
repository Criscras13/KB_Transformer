Defend rewrites URLs from inbound emails to scan linked content for security risks using linguistic and contextual analysis.
\n
When visiting a URL from a Defend-protected email, you will be redirected via the Defend portal first. URL links in emails will present as \u201clinks.defend.egress.com/\u2026\u201d. This feature is a configurable option. Domains you do not wish to be rewritten can have an exception.
\n
\n
\nNote:When a URL rewriting exception is configured, the original URL will be displayed to users, but link scanning will still occur. This includes checking for character obfuscation, suspicious domains, domain age, and potential impersonation of trusted organizations.
\n
\n
To add a URL exception, follow the steps below:
\n
    \n
  1. Navigate to URL Rewriting / Decode > URL Rewriting.
    2. \n
  2. Select Add Exception.
    3. \n
  3. Enter the URL hostname in the Hostname field.
    4. \n
  4. Select the Match Subdomains option if required.
    5. \n
  5. Use the Action drop-down menu to select Disable Link Rewriting or Disable Link Scanning.
    6. \n
  6. Add an optional comment.
    7. \n
  7. Select Save.
    8. \n
\n
\n
Defend will no longer rewrite links from the specified URL.
