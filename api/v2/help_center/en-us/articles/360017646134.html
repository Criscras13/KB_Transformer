The Active Directory Integration (ADI) tool allows you to transfer users and groups from your Active Directory (AD) into the KnowBe4 training console.\u00a0
\n
Organizational Units (OU) and security groups within AD can use any character set, but some of these character sets require special accommodation.\u00a0
\n
Click the drop-down arrow next to the title of each column in the table for more information about the table.\u00a0
\n
\n
\nNote:\u00a0All of the information in this article has been validated as of ADI Sync version 1.0.20.\n
\n
\n
Table Information
\n
    \n
  • \nCharacter: This column lists the characters used within an OU. If a character isn\u2019t listed, it is understood to be anywhere within the name of the OU.
    • \n
  • \nEscape Sequence Syntax: This column lists the sequences required to escape the character in an OU.
    • \n
  • \nExample: This column provides an example for reference.
    • \n
\n
Character Escape Table
\n
Special Characters
\n
Add two backslashes (\\\\) before the group name if the group name begins with any of the following special characters:
\n
    \n
  • Hashtag (#)
    • \n
  • Angle brackets (<>)
    • \n
  • Comma (,)
    • \n
  • Equal sign (=)
    • \n
  • Forward slash (/)
    • \n
  • Semicolon (;)
    • \n
\n
Example:
\n
    \n
  • Original: \u201c#GroupName\u201d
    • \n
  • Escape group name: \u201c\\\\#GroupName\u201d
    • \n
\n
Backslash\u00a0
\n
If a group name starts with a backslash, add an additional backslash before the group name.
\n
Example:
\n
    \n
  • Original: \u201c\\GroupName\u201d
    • \n
  • Escape group name: \u201c\\\\GroupName\u201d
    • \n
