The Documents tab of your KCM GRC platform is a central repository of all of the documents, files, or links that have been added or uploaded to your account. This page contains files and links from your Compliance Management module, Policy Management module, and Vendor Risk Management module.\u00a0
\n
\n
\nTip: Your auditors can see the files and links that are in the Documents tab in a limited capacity. For more information, see our Guide for Auditors article.
\n
\n
See the sections below to learn how to view files and links from the subtabs of the Documents tab.
\n
\n
\nNote: You will not be able to open links to local files by clicking the file name or link icon. Instead, you will need to right-click the file name and copy the link address. Then, open a new window in your browser and paste the link into the address bar.
\n
\n
All Documents
\n
When you navigate to the\u00a0Documents tab, you'll be taken to the All subtab.
\n
To learn about this page, see the screenshot and list below:
\n
    \n
  1. \nName: This column displays the name that the user assigned to the file or link when they added it to your platform. For vendor documents, this column displays the same name as the file that was attached to a questionnaire.\n
    \n
    \nTip: If you hover over the name, you can see the URL for links and the extension for files.
    \n
    \n
    2. \n
  2. \nDate Created: This column displays the date that the file or link was added to your platform.\u00a0
    3. \n
  3. \nActions: This column displays the following icons:\n
      \n
    1. If the item is a link, this column will show the link icon. Click this icon to open the web page in a new browser tab.
      2. \n
    2. If the item is a file, this column will show the download icon. Click this icon to download and view the file.
      3. \n
    \n
    4. \n
\n
Evidence
\n
The Evidence subtab is a repository of supporting evidence that your users have provided for the controls that you've implemented in your account. Users submit evidence to the platform by uploading a file or by providing a link to externally-hosted evidence. To learn more about submitting evidence, see our How to Monitor and Complete Tasks article.
\n
To learn about this page, see the screenshot and list below:
\n\n
    \n
  1. \nRight arrow icon: Clicking on this icon expands the row and see any requirements that are applicable to the evidence item. Expand the following menu for more information.\n
      \n
    • \n
        \n
      • If a requirement is applicable to an evidence item, this means that the evidence was submitted to a task for a control that is mapped to the requirement.
        • \n
      • The expanded row will show the following information for the applicable requirements:\n
          \n
        1. \nRequirement ID: This column displays an ID that determines the order of requirements within a scope.
          2. \n
        2. \nRequirement Name: This column displays the name assigned to the requirement.
          3. \n
        3. \nRequirement Description: This column displays a description of the requirement.
          4. \n
        \n\n
        • \n
      \n
      • \n
    \n
    2. \n
  2. \nName: This column displays the name that the user assigned to the evidence item when they added it to your platform.\n
    \n
    \nTip: If you hover over a name you can see the URL for links and the extension for files.
    \n
    \n
    3. \n
  3. \nControl Name: This column displays the name of the control that is associated with the evidence item. You can click on the control name to open the control.\u00a0
    4. \n
  4. \nTask: Click a Task button in this column to open the control task that the evidence was submitted under.
    5. \n
  5. \nDate Created: This column displays the date that the evidence item was added to your platform.
    6. \n
  6. \nActions: This column displays the following icons:\n
      \n
    1. If the evidence item is a link, this column will show the link icon. You can click this icon to open the web page in a new browser tab.
      2. \n
    2. If the evidence item is a file, this column will show the download icon. You can click this icon to download and view the file.
      3. \n
    3. You can click the pencil icon to update the evidence.\n
      \n
      \nNote: The pencil icon will not be available if the associated task is closed. Once a task is closed you cannot update the task evidence.
      \n
      \n
      4. \n

Policies\u00a0
\n
The Policies subtab contains the internal policies, terms, or agreements that you have added to your platform for the purpose of user acknowledgment. Users add policies to the platform by uploading a file or by providing a link to an externally-hosted policy. For more information about the Policy Management module, see our Policy Management Module Guide.
\n
To learn about this page, see the screenshot and list below:
\n\n
    \n
  1. \nName: This column displays the name that the user assigned to the policy item when they added it to your platform.\n
    \n
    \nTip: If you hover over a name you can see the URL for links and the extension for files.
    \n
    \n
    2. \n
  2. \nVersion: This column displays the version that the user assigned to the policy item when they added it to your platform.
    3. \n
  3. \nCampaigns: This column displays the name of the campaign that you have created for users to acknowledge the policy. If the policy is assigned under multiple campaigns, the policy item is listed multiple times on this page.\u00a0
    4. \n
  4. \nLast Updated: This column displays the date that the policy item was last updated.\u00a0
    5. \n
  5. \nActions: This column will display the following icons:\n
      \n
    1. If the item is a link, this column will show the link icon. You can click this icon to open the web page in a new browser tab.
      2. \n
    2. If the item is a file, this column will show the download icon. You can click this icon to download and view the file.
