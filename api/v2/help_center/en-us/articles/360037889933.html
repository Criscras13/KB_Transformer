In KCM GRC, administrators can create tasks and assign them to users. Then, users can satisfy their assigned tasks by submitting evidence as a document or a link. For more information about using evidence in your KCM GRC platform, see our Glossary of Compliance Terms article.
\n
Administrators can adjust the settings for the types or evidence that users can submit. They can limit the types of evidence to only documents or only links. Administrators can also prevent users from submitting either type of evidence.\u00a0
\n
The sections below contain three different methods for adjusting the settings for the evidence types that users can submit.\u00a0
\n
\n
Option 1: Account Settings
\n
\n
\nImportant:Changing your account settings will affect the default setting for existing scopes and new scopes. However, you can update this setting for existing scopes and new scopes, which will override your account settings.
\n
\n
If you want to change the default settings for your account, use the account settings to enable or disable an evidence type. For example, if your organization stores evidence on an internal server, you can disable document uploads for your account.\u00a0
\n
Follow the instructions below to change your default account setting for uploading links and documents.
\n
    \n
  1. Navigate to Settings > Account Settings in the top-right corner of the page. \n
    2. \n
  2. On the View\u00a0Account page, click the Account Settings\u00a0tab.\u00a0 \n
    3. \n
  3. Toggle the Document Upload Allowed and\u00a0Link Allowed settings to enable or disable them.\u00a0 \n
    4. \n
  4. Click the Save button in the bottom-right corner.
    5. \n
\n
\n
Option 2: Scope Settings
\n
\n
\nImportant:Scope settings override the account settings for enabling and disabling documents and links.
\n
\n
When you create new scopes or update existing scopes, the scope settings for submitting evidence default to match the account settings. If you want a scope's settings to differ from the account settings, you can change the scope's settings by using the instructions below.\u00a0 
\n
\n
Change the Scope Settings When Creating a New Scope
\n
    \n
  1. Navigate to Compliance >\u00a0Scopes\u00a0from the left navigation panel.
    2. \n
  2. Click the Create Scope button. \n
    3. \n
  3. Use the drop-down menus under Link Allowed and\u00a0Document Upload Allowed to select your preferred settings.\u00a0 \n
    4. \n
  4. Click the Create button in the bottom-right corner.
    5. \n
\n
For more information about creating scopes, see our Getting Started with the Compliance Management Module article.
\n
\n
Change the Scope Settings When Updating an Existing Scope
\n
    \n
  1. Navigate to Compliance > Scopes\u00a0from the left navigation panel.\u00a0
    2. \n
  2. Select a scope from the\u00a0View All Scopes\u00a0page.
    3. \n
  3. Click the\u00a0Update\u00a0button.\u00a0 \n
    4. \n
  4. Use the drop-down menus under Link Allowed and\u00a0Document Upload Allowed to select your preferred settings.\u00a0 \n
    5. \n
  5. Click the Save button in the bottom-right corner.
    6. \n
\n
\n
Option 3: Task Settings
\n
When you create a task, you can limit the types of evidence that a user can submit to satisfy a control. Follow the instructions below to change a control's task settings for submitting evidence.
\n
\n
\nTip: The steps below describe how to navigate to a control task from the applicable scope. As an alternative, you can create tasks for controls in the Controls Library.
\n
\n
    \n
  1. Navigate to Compliance > Scopes\u00a0from the left navigation panel.\u00a0
    2. \n
  2. Select a scope from the\u00a0View All Scopes\u00a0page.
    3. \n
  3. From the Controls tab, select a control.\u00a0 \n
    4. \n
  4. From the\u00a0Task Schedule tab on the\u00a0View Control\u00a0screen, click the\u00a0Create Schedule button. \n
    5. \n
  5. Under Evidence Requirements, toggle the Require Link and Require Document Upload settings to enable or disable them.\u00a0 \n
    6. \n
  6. Click the Schedule button.
    7. \n
\n
For more information about creating tasks, see our How to Work with Tasks for Controls article.
\n
\n
Frequently Asked Questions (FAQs)
\n
Below are some FAQs for limiting the types of evidence that users can submit. If you have additional questions, contact\u00a0our support team.\u00a0
\n
Question:\u00a0Do scope settings override account settings?
\n
Answer:\u00a0Yes. For example, if you allow document uploads in your Account Settings, but disable document uploads in a scope's settings, users cannot upload documents as evidence for tasks that are associated with that scope.
\n
Question:\u00a0Do task settings override scope settings?\u00a0
\n
Answer: No. If you limit the types of evidence that users can submit in a scope's settings, you cannot change this setting when you create a task. For example, if you disable document uploads for a scope, you cannot enable document uploads when you create a task. However, if you do not limit the types of evidence that users can upload in a scope's settings, you can limit the types of evidence that users can submit when you create a task.
\n
Question: If a control is mapped to two scopes that have conflicting settings, what happens?
\n
Answer: The enabled setting will override the disabled setting. For example, if one scope enables uploading documents and the other scope disables uploading documents, users can upload documents for that control.\u00a0
