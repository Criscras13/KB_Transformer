\n
\nNote: Due to upcoming changes from Google, we recommend using Direct Message Injection (DMI) as your default whitelisting configuration. For more information about these changes, see Google's Gmail updates.
\n
\n
In this article, you can learn how to whitelist KnowBe4's emails by IP address in Google Workspace. Whitelisting can help you ensure that your users receive our simulated phishing tests and training notifications.
\n
\n
\nImportant: To ensure that your user opens are being tracked properly, you may need to add our phish link domains to your Google Workspace\u2019s Image URL proxy allowlist. For more information, see Google\u2019s Set up an image URL proxy allowlist article.
\n
\n
We only recommend whitelisting by IP address if you don't have a cloud-based spam filter. If you have a cloud-based spam filter, we recommend whitelisting by email header instead.\u00a0 For more information, see our Whitelisting by Header in Google Workspace or Whitelisting Guide articles.
\n
\n
\nNote:This article contains our recommendations for whitelisting in Google Workspace, but Google Workspace may make changes to its features at any time. If you\u2019re experiencing issues with whitelisting by using the instructions below, please contact our support team.
\n
\n
Add KnowBe4's IP Addresses to Email Whitelist
\n
To whitelist our IP addresses, you'll need to add our IP addresses to your email whitelist in Google Workspace.
\n
To add our IP addresses to your email whitelist, follow the steps below.
\n
    \n
  1. Log in to your Google Workspace Admin console and click Apps\u00a0>\u00a0Google Workspace >\u00a0Gmail. \n
    2. \n
  2. Click Spam, Phishing and Malware.\n
    \n
    \nTip: If you use an older version of Google Workspace, you may need to click the Advanced settings button to see this option.
    \n
    \n\n
    3. \n
  3. In the Organizational Unit section of the page, select your domain.\n
    \n
    \nNote:Google Workspace only allows whitelisting by IP address for an entire domain, so you're unable to whitelist by IP Address for individual organizational units (OUs).
    \n
    \n\n
    4. \n
  4. In the Email whitelist section, enter our IP addresses separated by commas. For the most up-to-date list of our IP addresses, see the IP Addresses, Hostnames, and Header Information section of our Whitelisting Guide article. \n
    5. \n
  5. Click Save.
    6. \n
\n
Add KnowBe4's IP Addresses as Inbound Gateways
\n
When your users receive a simulated phishing email from KnowBe4, banners may display in Gmail to say \"This message seems dangerous\" or \"Be careful with this message\". To prevent these banners from displaying, we recommend that add our IP addresses as inbound gateways.\u00a0
\n
To add our IP addresses as inbound gateways, follow the steps below:
\n
\n
\nImportant:While we've found that these steps help to prevent Google banners from displaying, these steps aren't documented as a whitelisting recommendation by Google.
\n
\n
    \n
  1. Log in to your Google Workspace Admin console.
    2. \n
  2. Follow step 2 through step 4 in the Add KnowBe4's IP addresses to Email Whitelist section above. These steps will take you to your Spam, Phishing and Malware settings.
    3. \n
  3. Configure the Inbound gateway.
    4. \n
  4. Fill out your information to match the screenshot below:\n
      \n
    1. \nIP addresses/ranges: Enter KnowBe4's IP addresses. For the most up-to-date list of our IP addresses, see the IP Addresses, Hostnames, and Header Information section of our Whitelisting Guide article.
      2. \n
    2. Ensure the Reject all mail not from gateway IPs check box isn't selected.\u00a0\n
      \n
      \nNote:The Automatically detect external IP setting may interfere with whitelisting for KnowBe4. Unless you use other IP addresses that require you to enable this setting, we recommend that you don't select the Automatically detect external IP check box. For more information, see Google's Set up an inbound mail gateway article.
      \n
      \n
      3. \n
    3. Select the Require TLS for connections from the email gateways listed above check box.
      4. \n
    4. Select the Message is considered spam if the following header regexp matches check box. Then, enter a spam header tag that is unlikely to be found in a Phishing Security Test email. For example, you could enter random letters such as \"kzndsfgklinjvsdnfioasm\".
      5. \n
    5. Select the Disable Gmail spam evaluation on mail from this gateway; only use header value check box.
      6. \n
    \n\n
    5. \n
  5. Click Save. This setting may take up to an hour to deploy to all of your users.
    6. \n
\n
\n
\nNote:Before creating simulated phishing tests, you should also disable the return-path header in your KSAT Account Settings. For more information, see our How to Change the Return-Path Header in Your Account Settings article.
\n
\n
\n
\nTip: To ensure that you have whitelisted correctly, see our How to Verify You Have Whitelisted KnowBe4 Correctly article.
\n
