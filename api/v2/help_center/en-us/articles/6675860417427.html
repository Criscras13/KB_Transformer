In this article, you will learn how to integrate Okta with SecurityCoach. Once you set up this integration, data provided by Okta will be available under the SecurityCoach tab of your KSAT console. This data can be viewed in SecurityCoach reports and used to create detection rules for real-time coaching campaigns.
\n
\n
\n\n
\n
For general information about SecurityCoach, see our SecurityCoach Product Manual.
\n
\n
\nTip:SecurityCoach also supports integrating Okta as a Splunk vendor. If your organization uses Splunk for security information and event management, we recommend integrating through Splunk instead by following our Splunk Integrations Guide for SecurityCoach.
\n
\n
Create Your Okta API Token
\n
Before you can set up this integration in your KSAT console, you will need to create a read-only administrator account. Then, you will need to create an API token using this new account.
\n
To jump to the article subsection for each of these steps, click the links below:
\n
    \n
  • Create a Read-Only Administrator Account
    • \n
  • Create Your API Token
    • \n
\n
Create a Read-Only Administrator Account
\n
First, you will need to create a read-only administrator account for Okta. To create the account, follow the steps below:
\n
    \n
  1. Log in to your Okta console.
    2. \n
  2. Navigate to Directory > People.
    3. \n
  3. Click Add person.
    4. \n
  4. In the Add Person pop-up window that opens, enter the details for the new user.
    5. \n
  5. Click Save to create the new user.
    6. \n
  6. Navigate to Security > Administrators.
    7. \n
  7. Click + Add administrator.
    8. \n
  8. Fill out the fields on the Administrator assignment by admin page. For more information about these fields, see below:\n
      \n
    1. \nAdmin: Select the user you created in step 5 above.
      2. \n
    2. \nRole: Select Read-only Administrator.
      3. \n
    \n
    9. \n
  9. Click Save Changes to create the new administrator account.
    10. \n
\n
Create Your API Token
\n
After you have created a read-only administrator account, you can create your API token.
\n
To create your API token, follow the steps below:
\n
    \n
  1. Log in to your Okta console with the read-only administrator account you created in the Create a Read-Only Administrator Account section above.
    2. \n
  2. Navigate to Security > API.\n
    \n
    3. \n
  3. Select the Tokens tab. \n
    4. \n
  4. Click Create Token.\n
    \n
    5. \n
  5. Enter a name for the token.\n
    \n
    6. \n
  6. Click Create Token. Once you click this button, your token will display.\n
    \n
    \nImportant:Make sure to copy this token and save it in a place that you can easily access later. You will need this token during the setup process in the Set Up the Integration in KSAT section below.
    \n
    \n
    7. \n
  7. Once you have saved your token, click the OK, got it button to close the window.\n
    \n
    8. \n
\n
Once you have your API token, you can continue the setup process in your KSAT console.
\n
Set Up the Integration in Your KSAT Console
\n
Once you've created your Okta API token, you can set up the integration in your KSAT console. To set up the integration in your KSAT console, follow the steps below:
\n
    \n
  1. Log in to your KSAT console and navigate to SecurityCoach > Setup.\n
    \n
    2. \n
  2. In the Available Integrations section, locate the Okta card and click Configure.\n
    \n
    3. \n
  3. Paste your API token in the API Token field. This is the API token that you created in the Create Your API Token section above.\n
    \n
    4. \n
  4. In the API Domain field, enter your organization's unique Okta domain.\n
    \n
    \nNote: If you do not know your Okta domain, you can locate it by following the steps in Okta's Find your Okta Domain article. If you need to access your Okta account to locate your domain, please open a separate window to log in to your Okta account.
    \n
    \n
    5. \n
  5. Click Authorize.
    6. \n
\n
Map Your Users
\n
After you\u2019ve finished integrating Okta, you can map your users either through mapping rules (recommended) or through a CSV file upload. For more information about user mapping, see our Mapping Users in SecurityCoach article.
\n
\n
Once you\u2019ve successfully authorized this integration, you can manage detection rules for Okta on the Detection Rules subtab of SecurityCoach. For a full list of available system detection rules for this vendor, see our Which Detection Rules Can I Use with My Vendors? article.
